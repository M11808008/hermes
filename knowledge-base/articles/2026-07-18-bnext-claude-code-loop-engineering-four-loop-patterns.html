Claude Code Loop Engineering：4 種迴圈設計法，重點不是一直跑而是停止條件

來源：數位時代 BusinessNext 文章〈Claude Code官方解密！一次看懂4種「迴圈」設計法，讓AI Agent自動搞定複雜任務〉，整理 Claude Code 團隊在官方 blog 提出的 loop engineering 方法論：代理重複執行工作循環，直到滿足停止條件。本文以 Anthropic 官方 blog 與 Claude Code 文件交叉驗證。

一句話摘要

Loop engineering 不是叫 AI「一直改到好」，而是設計觸發條件、停止條件、驗證方法與權限邊界；Claude Code 官方把它拆成回合制、目標式、時間式與主動式四類，對欒帥的 Hermes / 課程設計最有價值的是：先定義「怎樣算完成」，再決定要不要升級成 /goal、/loop、/schedule 或多代理 workflow。

文章重點

數位時代引用 Claude 官方 blog「Loop engineering: Getting started with loops」，將 loop 定義為 agent 重複執行工作循環直到停止條件成立。文章提醒：如果沒有先設計好觸發方式、停止方式、工具指令與任務型態，所謂自動化很容易變成「一直問你要不要繼續」或「無限制消耗 token」。

4 種迴圈類型

| 類型 | 你交給 AI 的東西 | 適合任務 | 對應工具 / 指令 | 風險 |

|---|---|---|---|---|

| 回合制迴圈 | 單次 prompt 與驗證清單 | 還在探索、需要你逐步把關的任務 | 一般 Claude Code 對話、SKILL.md 驗收清單 | 仍需人手動檢查，來回輪數可能變多。 |

| 目標式迴圈 | 明確停止條件 | 有可量化完成標準的任務，例如測試通過、Lighthouse 分數達標 | `/goal` | 停止條件寫太模糊會讓 agent 自己瞎繞；評估需要可見證據。 |

| 時間式迴圈 | 觸發時間與固定 prompt | 例行檢查、PR / CI 狀態、定期摘要 | `/loop`、`/schedule` | 間隔太密會浪費 token；本機 session、7 天到期、雲端 routine 條件需確認。 |

| 主動式迴圈 | 持續發生的工作流與治理規則 | issue 分類、bug 回報、依賴升級、長期 migration | schedule/routines + auto mode + dynamic workflows / subagents | 權限、安全、成本與審查機制要先設計好。 |

官方查證與修正

  • Claude 官方 blog 確認 loop engineering 的核心分類：turn-based loops、goal loops、time-based loops、proactive loops。
  • Claude Code `/goal` 文件確認可讓 Claude 持續嘗試直到目標達成或達到回合上限，適合有清楚驗收條件的任務。
  • Claude Code scheduled tasks 文件確認 `/loop` 可在 CLI session 內按時間重跑 prompt；這類任務依賴目前 session 與本機環境。
  • Claude Code routines 文件確認 `/schedule` / routines 是雲端常駐排程能力，但有登入、組織政策、研究預覽與觸發條件限制。
  • Claude Code best practices 與 commands 文件支持「給 Claude 可讀的 pass/fail 訊號」、「用 hooks / skills / commands 管理工作流」這個方向。

欒帥可直接採用的教學框架

  • 先問：這個任務需要人一步步看嗎？如果需要，就留在回合制迴圈。
  • 如果「完成」可以被測試或量化，才升級成 `/goal`。
  • 如果輸入資料會定時變化，例如每日摘要、PR 狀態、CI 結果，才用 `/loop` 或 `/schedule`。
  • 如果任務是長期流入的工作串流，才設計主動式迴圈，並加入權限、審查、停止與成本監控。
  • 把驗收標準寫成 SKILL.md / CLAUDE.md / hooks，而不是每次口頭提醒。

實務注意事項

  • 迴圈的品質取決於「驗收標準」，不是單靠模型聰明。
  • 能用腳本穩定處理的重複工作，不要每次都讓 AI 重新推理。
  • 排程間隔要對應外部事件變化頻率；不要因為能每 5 分鐘跑就真的每 5 分鐘跑。
  • Auto mode 與主動式 workflow 不等於安全無人值守；涉及刪檔、金流、發布、個資或外部訊息仍應加人工 gate。
  • 如果一次 loop 的結果失敗，應把教訓寫回技能檔、測試或 hooks，讓下一輪系統變好，而不是只修單一輸出。

Why it matters

這篇文章把「Claude Code 進階玩法」從技巧列表提升成工程方法論：真正重要的是設計一個可驗證、可停止、可觀測、可控成本的 agent loop。對 AI 學習雷達與課程來說，它很適合作為「從 prompt engineering 走向 harness / loop engineering」的核心教材。
