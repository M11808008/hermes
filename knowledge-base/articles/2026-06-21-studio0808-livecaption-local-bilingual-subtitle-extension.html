Studio0808 LiveCaption｜本機即時雙語字幕 Chrome 外掛與無障礙學習工作流

來源為 Generative AI 技術交流中心中李佳恩分享的 Facebook 社團貼文。貼文發布一款以 Antigravity + Gemini 3.5 Flash 開發的「Studio0808 LiveCaption」多國語言版：用 Chrome 外掛擷取網頁影音分頁音訊，再由本機 Python 後端以 SenseVoice / Sherpa-ONNX 做語音辨識，並可接 Ollama 或 DeepSeek 產生雙語字幕。

一句話摘要

Studio0808 LiveCaption 是一個很適合 AI 學習與無障礙教育場景的開源工具：它把「看國外直播或無字幕影片」變成可即時閱讀、可翻譯、可保存的本機字幕工作流，但目前仍屬早期專案，需注意安裝門檻、模型下載、API 金鑰與翻譯準確性驗證。

技術架構拆解

| 模組 | 作用 | 技術線索 |

|---|---|---|

| Chrome Extension | 擷取當前分頁音訊、顯示字幕懸浮框 | Manifest V3、`tabCapture`、`offscreen`、`activeTab`、content script |

| Python Backend | WebSocket 接收音訊、VAD 斷句、ASR 辨識、翻譯 | FastAPI、Uvicorn、websockets、Sherpa-ONNX、OpenCC、httpx |

| VAD | 偵測語音片段與斷句 | Silero VAD ONNX；後端設定 threshold 0.4、min speech 0.15s |

| ASR | 多語音訊轉文字 | SenseVoice-Small ONNX int8，支援 zh/en/ja/ko/yue |

| 翻譯 | 字幕翻譯與雙語對照 | Ollama 本地模型、DeepSeek API；程式註解也提到 Google Translate fallback |

| UI / UX | 毛玻璃字幕、字體大小、歷史行數、滑鼠穿透 | 適合直播或影片不中斷觀看 |

對欒帥的教學與工作流價值

  • **AI 學習雷達的「直播吸收器」**：可用來看 OpenAI、Anthropic、Google、NVIDIA、AI 研究者的國外直播，把字幕變成第一層理解輔助。
  • **無障礙教育案例**：很適合放進「AI for accessibility」或「AI 教育平權」單元，因為它直接把聽覺障礙、語言障礙與同步學習串在一起。
  • **本機 AI 架構示範**：Chrome Extension + local FastAPI + ONNX ASR + optional LLM translation，是很清楚的端側 AI pipeline 範例。
  • **可延伸成會議紀錄工具**：若 Markdown 存檔功能穩定，可接到 Obsidian / NotebookLM / AI 學習雷達做課程與會議筆記。
  • **適合作為 Antigravity / Gemini 實作案例**：Facebook 原文強調用 Antigravity + Gemini 3.5 Flash 開發，可作為「AI coding agent 產出實用工具」的案例。

實測前建議檢查清單

| 檢查項 | 原因 |

|---|---|

| Chrome 權限 | 外掛需要 `tabCapture` 與 `<all_urls>`，應確認只在可信瀏覽器環境使用 |

| 模型下載來源 | `download_models.py` 從 sherpa-onnx release 下載 SenseVoice / Silero VAD 模型 |

| API key 安全 | 若使用 DeepSeek，金鑰會存於外掛設定；避免在共用電腦使用 |

| 翻譯品質 | Facebook 留言有人詢問翻譯正確性；正式教學前應用同一段英文直播做抽樣評估 |

| 隱私範圍 | 本機 ASR + Ollama 才是完整離線；使用 DeepSeek 就會把文字送到雲端翻譯 API |

| 依賴完整性 | `backend/main.py` 使用 `opencc`，但目前 `requirements.txt` 片段中未列出 opencc 相關套件；若安裝失敗需補查依賴 |

| 授權 | README 有 MIT badge，但 GitHub API 未偵測 license；商用或再散布前應確認 LICENSE 檔 |

可做的小型驗證實驗

  • 選一段 5 分鐘英文 AI 發表會影片。
  • 分別測試：僅原文、Ollama 本地翻譯、DeepSeek API 翻譯。
  • 紀錄延遲、首字漏字率、專有名詞準確率、雙語對照可讀性。
  • 測試中英日韓與粵語各一段，確認 SenseVoice 多語能力是否符合教學需求。
  • 將輸出的字幕/Markdown 接到 AI 學習雷達，評估是否能形成「直播 → 字幕 → 摘要 → 知識庫」流程。

本文根據 Facebook 社團貼文、GitHub API / README、官方 GitHub Pages 手冊與原始碼片段整理；尚未在本機實際安裝執行 LiveCaption，因此延遲、準確率、穩定性、GPU/CPU 資源占用與 Markdown 存檔可靠度仍需實測。GitHub 指標為 2026-06-21 查詢快照，會隨時間變動。Facebook 原文提到「100% 本地端」與「100% 離線隱私」時，應理解為使用本機 ASR + Ollama 翻譯的模式；若切換 DeepSeek API，翻譯文字會送往雲端服務。
