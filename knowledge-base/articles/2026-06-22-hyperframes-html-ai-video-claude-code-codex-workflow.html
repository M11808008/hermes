HyperFrames｜用 HTML、Claude Code 與 Codex 生成可重現動畫影片的開源工作流

來源是 Threads@kufutw 轉述 Peter Yang 對 HyperFrames 的介紹：HyperFrames 團隊展示了一套直接在 Claude Code 和 Codex 裡，以 HTML 製作高品質動畫影片的工作流。重點不是單純「AI 生成影片」，而是把影片變成可 lint、可 preview、可 render、可由 coding agent 迭代的工程化產物。

一句話摘要

HyperFrames 把「影片」改寫成 HTML + CSS + media + seekable animations：AI Agent 先規劃 storyboard、frame.md 與畫面元素，再用 CLI 預覽與輸出 MP4；這讓產品 Demo、課程宣傳、社群短片與 AI 學習雷達摘要影片，有機會走向可重現、可版本控管的 video-as-code 流程。

Threads / X 貼文重點

Threads 貼文指出：

  • HyperFrames 可直接在 Claude Code 與 Codex 裡，用 HTML 製作動畫影片。
  • 流程不依賴傳統剪輯軟體，而是透過 AI 生成 `frame.md`、Storyboard 分鏡腳本與視覺元素。
  • 工具目前 100% 免費且開源。
  • 要做出精緻效果，關鍵在完整工作流：分鏡規劃、`frame.md` 結構、以及如何引導 AI 生成一致性的畫面與動畫。
  • 應用情境包含簡報動畫、產品 Demo、社群短影片。

Peter Yang 的 X 貼文也補充：他原本不相信可以用 HTML 做出這類影片，直到 HyperFrames 團隊展示如何直接在 Codex 與 Claude Code 裡完成；他特別提醒，真正好看的結果仰賴正確工作流，而不只是把 prompt 丟給工具。

README 明確說明，HyperFrames 可透過 CLI 本機使用，也可由 AI coding agents 透過 skills 操作。官方 quickstart 建議：

```bash

npx skills add heygen-com/hyperframes

```

或手動建立專案：

```bash

npx hyperframes init my-video

cd my-video

npx hyperframes preview

npx hyperframes render

```

HyperFrames 的核心概念

1. HTML 成為影片格式

HyperFrames 不是讓 AI 直接吐出一段不可控影片，而是定義一個 HTML composition。畫面、文字、影片、音訊、時間、軌道與動畫，都放在可讀的 HTML / CSS / JS 結構中。

官方 README 範例使用：

  • `data-composition-id`
  • `data-start`
  • `data-width` / `data-height`
  • `data-duration`
  • `data-track-index`
  • GSAP timeline
  • media elements

最後 renderer 會在 headless Chrome 中逐 frame seek，再用 FFmpeg 編碼，所以同一份輸入能產生較可重現的輸出。

2. Agent skills 管理製作 loop

官方說明 HyperFrames skills 會教 AI Agent 一套 production loop：

  • plan the video；
  • write valid HTML；
  • wire seekable animations；
  • add media；
  • lint；
  • preview；
  • render。

這些 skills 可搭配 Claude Code、Cursor、Gemini CLI、Codex、Google Antigravity、GitHub Copilot CLI 等支援 skills 的 coding agents。

3. frame.md 是設計系統到影格的翻譯層

官方 `frame.md` 定位很關鍵：一般 `design.md` 描述品牌、tokens、字體、色彩與 UI 規範，但不是為鏡頭與時間軸設計。`frame.md` 則把同一套設計規則轉為適合 16:9 或其他影片畫面的 composition 指令，讓 Agent 不必猜 scale、dwell、motion 與視覺節奏。

換句話說：

  • `design.md`：品牌與介面設計語言。
  • `frame.md`：影片構圖、節奏、畫面比例與 motion direction。
  • Storyboard：敘事順序與每幕重點。
  • HyperFrames HTML：可預覽、可渲染、可版本控管的影片原始碼。

適合欒帥的應用場景

1. AI 學習雷達短影音

可把每週知識庫重點轉成 15–30 秒短片：

  • 3–5 個重點卡片；
  • 每則工具／文章一個 scene；
  • 加上標題、來源、關鍵數字與 CTA；
  • 用 GitHub Pages 文章連結作為延伸閱讀。

這比手動剪片更容易規模化，也能把已歸檔的 Markdown 變成影片素材來源。

2. 課程與工作坊宣傳

AI 課程、n8n 工作坊、Physical AI / Agentic Workflow 主題都可以轉成：

  • 開場概念影片；
  • 課程單元預告；
  • 報名頁 hero animation；
  • 章節轉場素材；
  • 產品 demo-style explainer。

3. 技術產品 Demo

對工具型產品或內部自動化，可用 HTML 生成：

  • 流程圖動畫；
  • before / after 對比；
  • feature launch；
  • dashboard 數據動畫；
  • PR walkthrough / code diff 動畫。

建議實驗路線

先不要直接追求 60 秒大片。建議用低風險任務驗證：

  • 選一篇 AI 學習雷達文章。
  • 把摘要拆成 5 個 scenes。
  • 寫一份簡短 `frame.md`：色彩、字級、節奏、畫面比例、動效風格。
  • 讓 Claude Code 或 Codex 產生 HyperFrames HTML。
  • 跑 `npx hyperframes lint`。
  • 跑 `npx hyperframes preview` 看節奏。
  • 微調 ease、duration、stagger、字幕可讀性。
  • 最後 `npx hyperframes render` 匯出 MP4。

可用 Prompt 骨架

```text

Using /hyperframes, create a 20-second 16:9 explainer video from this article.

First produce a storyboard and frame.md plan.

Use a premium dark editorial style, strong Traditional Chinese typography, high contrast, and readable mobile-safe captions.

Do not render yet. After the storyboard is approved, write valid HyperFrames HTML, run lint, then preview before final render.

```

風險與限制

  • HyperFrames 仍需要 Node.js 22+ 與 FFmpeg，並不是純瀏覽器一鍵產片。
  • 影片品質高度依賴 storyboard、frame.md、素材、字體、節奏與人工審稿；不能把「開源工具」誤解成自動產出高品質。
  • Agent 產生的 HTML / JS 仍需 lint、preview 與人工檢查，尤其是中文字級、留白、節奏、音樂版權與來源標示。
  • 官方 repo 成長快速，stars / forks / CLI 指令與 skills 清單屬時間敏感資訊。
  • 本次尚未在本機實際 render MP4；本文定位為工具與工作流評估筆記，不是實測報告。
