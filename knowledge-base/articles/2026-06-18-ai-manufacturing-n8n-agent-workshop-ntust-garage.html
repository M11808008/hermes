課程內容重點

| 層次 | 工作坊主題 | 技術意義 |

|---|---|---|

| AI Skill | 企業知識結構化 | 把 SOP、經驗、規格、設備維護知識轉成可被 AI 調用的能力庫，類似企業內部 RAG / Skill Library |

對應的製造業痛點

| 痛點 | AI / n8n 對應解法 |

|---|---|

| 人力短缺 | 用 agent 承接重複查詢、報表整理、文件比對、通知與追蹤任務 |

| 經驗傳承困難 | 把老員工經驗、SOP、常見異常處置與維修知識結構化，形成可搜尋、可調用、可更新的知識庫 |

| SOP 實施難 | 將 SOP 轉成 checklist、任務流程、審核節點與自動通知，而不是只停留在 PDF 文件 |

| 報表製作耗時 | 用 n8n 串接資料來源，自動產生日報、週報、異常摘要與 KPI 追蹤 |

| 系統資料分散 | 以 workflow orchestration 方式整合 ERP、MES、試算表、Email、LINE、資料庫與內部 API |

技術判讀：為什麼值得追

  • **它不是純 AI 工具介紹**：主軸是把 AI Agent 放進製造流程，而不是只展示聊天機器人或生成式 AI demo。
  • **n8n 很適合作為落地層**：相較直接用 LangChain / AutoGen 做完整系統，n8n 對企業內部串接、審核、通知與可視化流程更直覺。
  • **AI Skill 是製造業真正的核心**：製造業的競爭力常在 SOP、設備 know-how、異常處置與客製流程；若沒有知識結構化，agent 很難穩定落地。
  • **適合 ERP / MES / 報表 PoC**：最小可行場景可從報表自動化、異常通知、SOP 查詢、工單摘要與維修紀錄整理開始。
  • **和欒帥現有方向高度貼合**：可連到 AI 學習雷達、企業知識庫、multi-agent workflow、Hermes skills、n8n orchestration 與教育課程設計。

企業級 AI Agent System 設計藍圖

| 模組 | 功能 | 製造業例子 |

|---|---|---|

| Knowledge Layer | SOP、設備手冊、製程參數、FAQ、維修紀錄、報表欄位說明 | 「請查這台設備 E-23 異常碼的處置步驟」 |

| Skill Layer | 將常用任務封裝成可調用技能 | 查庫存、產生報表、比對工單、通知主管、建立維修單 |

| Agent Layer | 根據任務目標選擇技能並協調多步驟流程 | 「每天 9:00 產出昨日產線異常摘要並標出需人工追蹤項目」 |

| Workflow Layer | n8n 串接資料、排程、通知、審核與外部 API | Google Sheets / ERP / Email / LINE / Slack / DB / Webhook |

| Governance Layer | 權限、日誌、人工核准、版本控制、資料邊界 | 高風險修改需人工 approval；每次報表產出保留來源與 log |

| Feedback Layer | 使用者修正、失敗案例、SOP 更新 | 將現場主管修正回寫到知識庫與 prompt / skill 規則 |

欒帥可用的導入提問清單

如果要把這場工作坊內容轉成企業 PoC，可先問製造業客戶：

  • 哪些報表每週最耗時？資料來源在哪裡？
  • 哪些 SOP 最常被問、最常執行錯誤？
  • 哪些經驗只存在資深員工腦中？是否能訪談轉成知識庫？
  • 哪些系統之間需要人工複製貼上？
  • 哪些任務可以先從「只讀摘要」開始，而不是一開始就自動寫回系統？
  • 哪些節點必須保留人工審核？
  • 如何驗證 agent 的輸出正確：來源引用、欄位比對、報表數字、主管簽核或測試資料集？
