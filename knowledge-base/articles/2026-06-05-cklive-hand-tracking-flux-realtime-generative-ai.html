CKLive 手勢追蹤接入 Flux｜用手指位置與 pinch gesture 即時控制生成影像

來源：Instagram Reels `@_cklive_`，貼文內容提到 CKLive 新增 hand tracking，並接入 Flux 即時生成流程。

一句話摘要

CKLive 把手部追蹤變成生成式影像 pipeline 的即時控制訊號：系統會連續追蹤手指位置，並用 pinch gesture 觸發可自訂的動作，讓創作者能用手勢直接控制 Flux 生成流程。

原貼文重點

貼文原文指出：

  • CKLive 已新增 hand tracking，並 wired into Flux。
  • Finger positions are tracked continuously：手指位置會被連續追蹤。
  • Pinch gestures trigger actions：捏合手勢可以觸發行動。
  • Actions can be mapped to whatever you want：手勢觸發的動作可自行映射。
  • Runs in real time on an RTX 5090：展示環境在 RTX 5090 上即時運作。
  • Tracking data is just a live input：追蹤資料本質上是一組 live input，可路由到 pipeline 的任意部分。

為什麼值得觀察

1. 生成式 AI 從「輸入提示詞」走向「即時表演介面」

這個案例的關鍵不只是 Flux 生成影像，而是把手部姿態、手指位置與捏合手勢變成控制層。對 live art、舞台視覺、互動展覽、VJ、教學演示來說，prompt 不再是唯一入口，身體動作也可以成為生成式 AI 的高頻輸入。

2. Live input 可路由到整個 pipeline

貼文強調 tracking data 可以 route into any part of the pipeline。這代表手勢不只可用來切換開關，也能控制 seed、strength、mask、prompt weight、camera movement、style blend、區域生成、音畫同步或其他節點參數。真正重要的是「把感測資料轉成可組合的控制訊號」。

3. 即時生成正在接近創作工具而非單次出圖工具

若手勢追蹤、Flux、GPU 即時推論與節點式 workflow 串起來，使用者操作會更接近樂器或舞台控制台：可以連續調整、即時回饋、現場表演。這與一般文生圖「輸入 prompt、等待結果」的互動模型完全不同。

欒帥可延伸應用

  • AI 課程展示：用手勢控制生成圖像，讓學生立即理解「感測輸入 → AI pipeline → 視覺輸出」的概念。
  • 互動展覽：把手指位置映射到風格、色彩、構圖、局部變形或粒子視覺。
  • AI 學習雷達 demo：未來可把「手勢控制生成」整理成一個案例包，連結 computer vision、real-time generation、edge GPU 與創意科技。
  • Hermes / LINE 助理概念：手勢或影像感測不一定只用在藝術，也可作為多模態 agent 的觸發方式。

技術觀察與可能組成

貼文本身沒有公開完整 workflow，但從描述可推測核心模組包含：

  • 手部追蹤：持續輸出手指位置與 gesture state。
  • Gesture mapping：將 pinch gesture 映射為 pipeline action。
  • Flux 即時生成：以 RTX 5090 提供低延遲生成能力。
  • Live routing：把 tracking data 作為 pipeline 參數輸入，而不是單純前處理。

留言中也有人提到可用 stream diffusion、TouchDesigner 或社群現成外掛做類似方向；這可視為延伸探索線索，但仍需以 CKLive 官方釋出的 workflow 為準。

需要保留的限制

  • 貼文未提供完整技術棧、延遲數字、解析度、Flux 版本或 workflow 檔案。
  • RTX 5090 是高階硬體，不能直接推論一般筆電或低階 GPU 也能達到同等即時效果。
  • Reels 展示偏產品/創作 demo，若要落地課程或展演，需另外測試穩定性、散熱、長時間運作與互動延遲。
