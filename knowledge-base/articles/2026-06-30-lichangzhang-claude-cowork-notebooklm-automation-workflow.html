工作流架構

1. NotebookLM 是內容生成與資料理解層

NotebookLM 在這個工作流中的角色不是聊天工具，而是資料工作台：

  • 匯入文件、逐字稿、研究資料與網頁來源。
  • 生成資訊圖、幻燈片、音頻、思維導圖與摘要。
  • 將分散資料整理成可展示、可聽、可簡報的輸出。

它的限制是：多數操作仍需手動，且不容易直接跟日常任務排程、郵件、資料夾或瀏覽器流程整合。

2. Claude Cowork 是自動化與操作層

Claude Cowork 在影片中的角色是「會操作工具的助理」：

  • 讀取指定資料夾。
  • 開啟 NotebookLM。
  • 上傳檔案或逐字稿。
  • 填寫生成要求。
  • 觸發資訊圖、簡報、報告或音頻輸出。
  • 可搭配定時任務進行每週或每日更新。

對欒帥的課程語境來說，這是「AI 從回答問題，升級成操作流程」的好案例。

3. Skill 是兩者之間的橋樑

作者提供的 NotebookLM Skill 讓使用者可以在 Claude Cowork 中用斜槓命令，或直接說「在 NotebookLM 裡做某件事」來觸發自動化。Skill 內容包含：

  • 可執行的 NotebookLM 功能清單。
  • 觸發詞與使用情境。
  • 透過 Claude Chrome 插件操作 NotebookLM 的步驟。
  • 任務完成後如何停止，避免無謂 token 消耗。

這呼應近期 AI Agent 設計的關鍵：不是每次重新 prompt，而是把可重複流程做成可維護的 Skill。

權限與隱私設定

影片中最重要的實務提醒，是 Claude in Chrome 的權限設定。

作者建議不要直接選擇 Allow extension，因為那代表 Claude 預設可以打開瀏覽器訪問任何網站。比較安全的做法是：

  • 選擇 Block extension。
  • 只把必要網站加入允許清單。
  • 例如只允許：

- google.com

- notebooklm.google.com

  • 網址不要加 https 開頭，也不要在結尾加反斜線，否則 Cowork 可能無法辨識。

對企業、課程與個人知識庫來說，這個設計很關鍵：Agent 自動化一定要採取最小權限原則，只開放任務必要的網站與資料。

A. 文件夾資料 → NotebookLM 資訊圖

作者示範把一個包含 Codex 核心功能研究資料的資料夾交給 Claude Cowork，資料類型包含 Word、Markdown、Excel、PPT 等。Cowork 會操作 NotebookLM：

  • 建立新筆記本。
  • 將文件拖曳進 NotebookLM。
  • 讀取資料後打開自訂資訊圖功能。
  • 填入生成描述。
  • 等待 NotebookLM 產生完整資訊圖。

這個任務花了約七分鐘，但資訊完整度不錯。它的啟發是：只要有可調用工具的 Skill，就可以在單一平台中分配任務，讓 Agent 操作外部工具完成產出。

B. YouTube 逐字稿 → NotebookLM 幻燈片

作者建立了一個名為「YouTube NotebookLM 幻燈片」的 Claude Cowork Project，並在 project instructions 中寫清楚：

  • 使用固定的 NotebookLM 筆記本，避免每次都新建筆記本造成混亂。
  • 任務開始前先刪除舊資料，再加入新逐字稿。
  • 要使用指定品牌色與簡報要求。
  • 完成後可下載成 PDF 或 PowerPoint，也可在 NotebookLM 中修改細節。

這對欒帥很有價值：可以把 AI 學習雷達中的影片、課程逐字稿或講座紀錄轉成簡報初稿，再由人校稿與美化。

作者示範建立定時任務：每週六晚上五點，自動搜尋過去一週 Claude Cowork 相關消息，把網站資料加入 NotebookLM 筆記本，再生成報告。

同一架構也可以延伸成：

  • 每天早上自動調研指定主題。
  • 檢查郵件與日曆。
  • 生成晨間簡報。
  • 把摘要做成音頻，讓使用者用聽的掌握一天安排。

這與欒帥的 AI 學習雷達高度相關：未來可以讓 Agent 定期整理 AI 工具新聞、課程素材、社群貼文或研究趨勢，再輸出成週報、簡報或音頻版摘要。

Token 成本控制

影片中特別提到 Claude Cowork 的 token 消耗問題。作者的建議是：

  • 不要讓 Skill 一直等待 NotebookLM 生成完整結果。
  • 在 Skill 裡設定：只要點擊開始工作，並確認任務已經啟動，Claude Cowork 就可以結束操作。
  • 後續生成由 NotebookLM 自己完成，不需要 Cowork 持續消耗 token。
  • 如果手機有 NotebookLM App，完成後可收到通知。
  • 對這類工具調用任務，Sonnet 通常已足夠，不一定需要 Opus。

這是很實用的 Agent 工作流設計原則：Agent 只負責啟動與交接任務，不一定要全程等待外部工具完成。

風險與限制

  • 影片中展示的 NotebookLM Skill 是作者提供的免費資源，使用前仍需檢查內容、權限與觸發邏輯。
  • Claude in Chrome 若開放過大權限，可能帶來隱私與資料外洩風險。
  • NotebookLM 生成資訊圖或簡報的品質仍需人工校稿，尤其是品牌一致性、事實查核與版面細節。
  • 若將郵件、日曆或公司資料交給 Agent 處理，必須先確認資料權限、個資保護與組織政策。
  • 定時任務應避免「自動發送」或「自動公開發布」，建議先輸出草稿與報告，由人確認後再發布。

可落地的最小實驗

  • 選一個不含敏感資料的主題，例如「近期 Claude Code 教學影片」。
  • 建立一個 NotebookLM 筆記本。
  • 在 Claude Cowork Project instructions 中放入筆記本連結、輸出要求與品牌色。
  • 上傳或貼上逐字稿。
  • 讓 Cowork 啟動 NotebookLM 幻燈片生成。
  • 設定 Skill 在「確認任務啟動」後停止，避免 token 浪費。
  • 人工檢查簡報內容與版面，再決定是否作為課程素材。
