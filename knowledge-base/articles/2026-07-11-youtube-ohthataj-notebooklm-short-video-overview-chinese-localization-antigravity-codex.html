歐那個 AJ YouTube｜NotebookLM Short Video Overview：把資料變短影音，再用 Antigravity / Codex 一條龍中文化

一句話摘要

這支影片把 NotebookLM 的短影音生成、Google Antigravity / Codex 代理式工具、Gemini 語音 API 與 FFMPEG 串成一條「資料 → 英文短影音 → 繁中字幕與中文旁白影片」的工作流。

核心流程

影片示範的流程可整理為六段：

  • 在 NotebookLM 建立新筆記本，匯入資料來源。
  • 切換 Google 帳戶語言到 English / United States，讓 NotebookLM 介面顯示尚未在中文介面開放的 Short Video Overview 選項。
  • 在 Video Overview / Short 格式中使用 Custom Topic 或提示詞，讓 NotebookLM 根據資料生成約一分鐘的英文短影音。
  • 下載 NotebookLM 產生的英文影片；影片本身已含壓入畫面的英文字幕。
  • 把影片交給 Antigravity 或 Codex，由 agent 規劃並執行在地化工作流：抽音訊、辨識英文、翻成繁中、生成中文旁白、用 FFMPEG 合成。
  • 產出含繁體中文字幕與中文 AI 旁白的新影片。

章節與重點

0:00-0:56｜NotebookLM 開始做短影音

AJ 開場指出 NotebookLM 新增 Video Overview 裡的短影音生成，能把資料轉成高質感短影片；目前關鍵限制是輸出仍以英文為主，所以後段重點會放在如何一條龍轉為中文語音與字幕版本。

0:59-2:24｜啟用 Short Video Overview

流程包括：

  • 進入 NotebookLM。
  • 建立或開啟筆記本。
  • 匯入資料來源，影片示例使用 Google 的 Gemma 模型資料。
  • 如果中文介面看不到短影音功能，切到 Google 帳戶個人資訊中的語言設定，新增 English / United States 並置頂。
  • 回到筆記本後，Video Overview 內會出現 Short 相關選項。

這是一個時效性技巧；日後 NotebookLM 若正式支援中文介面或中文版短影音，這段切語言流程可能不再需要。

2:24-3:39｜用 Custom Topic 生成短影音

NotebookLM 會根據匯入資料提出主題候選，也可自行在 Custom Topic 填入提示詞。AJ 建議：

  • 提示詞用英文效果較穩。
  • 不必寫得過度複雜。
  • 生成後可得到約一分鐘、含動畫與字幕的短影音。

但目前下載下來的影片字幕是壓死在畫面上的英文字幕，因此後續中文化不是單純換字幕檔，而是要處理整段影片的在地化版本。

3:39-5:08｜Antigravity 與 Codex 作為 agentic media 工具

AJ 將 Antigravity 形容為「Google 版 Codex」：

  • Codex 接 GPT 系列能力。
  • Antigravity 接 Google Gemini 能力。
  • 兩者都可作為代理式工具，在本機執行任務、操作檔案與工具鏈。
  • 若公司使用 Google Workspace，Antigravity 可能更順；若使用 ChatGPT Business，Codex 可能更自然。
  • 個人使用者可兩者都試，因為都有一定免費額度。

這段的重點不是兩者誰勝出，而是把「影片中文化」交給會操作本機工具的 agent。

5:08-7:02｜把任務交給 Agent：中英轉換工作流

影片中的任務目標是：

  • 將 NotebookLM 產生的英文 Video Overview 短影音轉成中文版短影音。
  • 產生自然口語的繁體中文字幕。
  • 產生中文 AI 旁白。
  • 盡量對齊原影片時間軸。
  • 由 agent 自行整理 1-7 步流程並執行。

AJ 提醒：若在 Windows 環境缺 Python、FFMPEG、Node 等工具，AI 不一定能自動裝好；本機工具鏈仍是成功關鍵。

7:09-8:27｜Gemini API Key 與 AI Studio

Agent 會判斷需要 Gemini API Key。AJ 示範到 Google AI Studio 建立 API Key，並提醒：

  • Google AI Studio 可建立 API Key。
  • 影片示範使用免費帳號產生 Key。
  • 這類 Key 不應公開貼到影片、前端或公開 repo。

這段很適合延伸成課程中的「API Key 安全」教學點。

9:09-9:46｜產出三個結果檔

影片最後得到三類成果：

  • SRT 字幕檔：先辨識英文語音，再翻譯成中文。
  • 中文語音檔：透過 Gemini TTS 生成中文旁白。
  • 合成後影片：用 FFMPEG 把新聲音與影片合成。

這就是「NotebookLM 產出英文短影音 → Agent 轉成本地化短影音」的完整閉環。

對 AI 學習雷達的價值

這支影片值得歸檔，因為它展示的是一種很實用的新型態工作流：

  • NotebookLM 負責把來源資料摘要成影片素材。
  • Agentic coding 工具負責本機自動化與媒體處理。
  • Gemini API 負責語音生成。
  • FFMPEG 負責音訊/影片合成。
  • 人類負責提供資料、設定目標、檢查字幕與成品品質。

這代表知識工作者可以把「讀資料、做摘要、產影片、翻譯配音、合成輸出」拆成可由 agent 接手的管線。

適合應用場景

  • 教師將講義、論文、課程資料轉成一分鐘複習短片。
  • 企業內訓把 SOP、產品文件、FAQ 轉成中文短影音教材。
  • 自媒體把英文研究報告轉成繁中科普短片。
  • AI 課程示範從 NotebookLM 到 Codex / Antigravity 的跨工具工作流。
  • 團隊建立半自動「資料庫 → 短影音 → 在地化」內容生產線。

風險與注意事項

  • 來源授權：NotebookLM 匯入的資料與產出的影片若要公開發布，仍需確認原始素材與文字授權。
  • API Key 安全：Gemini API Key 不可放到公開影片、公開 repo、前端程式或截圖中。
  • 字幕品質：自動辨識與翻譯可能錯詞，特別是模型名、產品名、數字與專有名詞。
  • 語音同步：TTS 中文語速與原片時間軸可能不完全一致，需要人工微調。
  • 免費額度：Google AI Studio、NotebookLM、Antigravity / Codex 額度會變動，不應把「免費」當成永久承諾。
  • 工具鏈依賴：本機缺 Python、FFMPEG、Node 或權限不足時，agent 可能卡住。

可複用工作流筆記

最小可行流程：

  • 用 NotebookLM 建立筆記本並匯入資料。
  • 用 Short Video Overview 產生英文短影音。
  • 下載影片。
  • 在 Antigravity 或 Codex 中給任務：將影片轉為繁中字幕與中文旁白版，需保留原時間軸。
  • 提供 Gemini API Key，並要求 agent 使用安全的環境變數或本機設定，不要寫進程式碼。
  • 檢查產出的 SRT、中文旁白音檔與合成影片。
  • 人工修正專有名詞與字幕斷句後再發布。
