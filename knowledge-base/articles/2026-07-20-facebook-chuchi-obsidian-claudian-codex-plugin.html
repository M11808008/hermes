Claudian 把 Claude Code/Codex 帶進 Obsidian：邊讀筆記邊呼叫本地 coding agent

來源是朱騏的 Facebook photo post。公開可見貼文表示：他終於把 Obsidian 裡的 Claudian 接上 Codex；Claudian 是一款 Obsidian 插件，可在 Obsidian 介面中使用 Claude Code/Codex，方便「邊讀筆記邊和 AI 對話」。貼文也提到設定過程中有 4 個容易踩坑的地方，第一點是「Codex CLI path 不是填 API Key」。Facebook 登出頁只公開到第一個踩坑標題，後續細節未完整展開。

一句話摘要

Claudian 的價值在於把 Obsidian vault 直接變成本地 coding agent 的 working directory：你可以在筆記脈絡中呼叫 Claude Code、Codex、Opencode 等代理讀寫檔案、搜尋、bash 與多步驟工作，而不是在外部 CLI/桌面 app 和筆記之間來回複製上下文。

欒帥可用的實務理解

  • 對 AI 學習雷達很關鍵：它把 Obsidian 從「筆記庫」推向「agent workspace」，可讓筆記、素材、prompt、課程草稿直接被本地代理操作。
  • 對課程示範很直觀：學生可以看到 AI 不是只在 chat window 回答，而是能在一個知識庫內讀、改、整理、重構內容。
  • 對 Codex 使用者有用：若已登入 Codex CLI，Claudian 走的是本機 CLI path / provider runtime，而不是在 Obsidian 裡貼 API Key。
  • 對工作流設計有啟發：Obsidian vault 可以承載 notes、skills、prompt templates、MCP references、會議紀錄與研究素材；agent 則負責跨檔案整理與產出。

安裝與設定注意事項

  • 建議先從 Obsidian Community Plugins 安裝：Settings → Community plugins → Browse → 搜尋 Claudian → Install / Enable。
  • 若使用 Codex，要填的是 Codex CLI path 或讓外掛找到 CLI，而不是 OpenAI API Key。Codex 的登入、模型與額度管理仍依 Codex CLI 本身。
  • README 也提醒 Claude CLI 找不到時，常見原因是 GUI app 抓不到 shell 的 PATH，尤其 macOS 上 nvm/fnm/volta 或 npm-installed CLI 路徑不同。
  • 外掛會讓 agent 以 vault 為工作目錄讀寫檔案；因此要先確認 vault 備份、git 版本控制、敏感筆記隔離與工具 permission 設定。
  • README 的 privacy section 表示 provider subprocesses 會繼承 Obsidian process environment 加上外掛設定的 environment variables；API/CLI 認證與 secrets 管理要特別小心。

Caveats

  • Facebook 貼文後半部與完整 4 個踩坑項目未公開可見，本篇不補寫未取得的細節。
  • 本篇只做文件層與公開來源查核，未在欒帥本機 Obsidian vault 實測安裝 Claudian。
  • Claudian 是 desktop-only Obsidian plugin，不適用 mobile Obsidian。
  • 讓 agent 直接操作 vault 很強，但也代表誤刪、批次改寫、secret exposure 的風險升高；正式使用前應建立 git/備份與敏感資料分區。

Why it matters

這則貼文延續「AI Vault / agentic knowledge base」的方向：真正有用的 AI 筆記系統不是只把內容丟進 RAG，而是讓 agent 站在知識庫內部工作。Claudian 把 Claude Code/Codex 這類 coding agent 放進 Obsidian，等於把筆記、檔案系統、CLI、MCP 與多步驟任務接到同一個操作面。對欒帥的課程與 AI 學習雷達而言，它是一個很好的展示案例：個人知識庫可以從「搜尋與閱讀」進化到「可委派、可修改、可審核的 agent workspace」。
