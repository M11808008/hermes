AI Engineering from Scratch｜503 課、20 階段的開源 AI 工程自學課綱

來源是 欒帥分享的 GitHub repo。原始分享 URL 帶有 `fbclid` 追蹤參數；本文已將 canonical source 正規化為 GitHub repo：`rohitg00/ai-engineering-from-scratch`。

一句話摘要

`AI Engineering from Scratch` 是一套大型開源 AI 工程課綱：從數學與 ML 基礎一路走到 LLM、MCP、Agent Engineering、多代理、production infrastructure、alignment 與 capstone projects；它的特色不是只看影片或只呼叫 API，而是每課都要求「從原理手刻 → 用正式框架重做 → 產出可重用 artifact」。

這份課綱的核心設計

README 的定位很明確：

  • 20 phases。
  • README 標示 503 lessons。
  • 約 320 小時；ROADMAP 目前寫約 314 小時。
  • 每課產出一個可重用 artifact：prompt、skill、agent 或 MCP server。
  • 每個演算法先從 raw math / no-framework 版本開始，再接 production library，例如 PyTorch / sklearn。
  • 不只是「學 AI」，而是能解釋底層、能建、能 ship。

這對欒帥很有價值，因為它剛好接在「AI 工具使用」與「AI 工程能力」之間：適合當作 AI 課程設計、企業內訓、學生自學地圖，或 Hermes / Codex / Claude Code agent skill 生態的參考素材。

每課的固定節奏

README 描述每課都有六段式：

  • **Motto**：一句話抓住核心。
  • **Problem**：先定義真實痛點。
  • **Concept**：用圖解與直覺理解概念。
  • **Build It**：從 raw math / no-framework 手刻。
  • **Use It**：用 PyTorch、sklearn 或正式框架重做。
  • **Ship It**：產出 prompt、skill、agent 或 MCP server。

這個設計很適合拿來示範「AI engineering 不是只會叫 API」：學員必須理解機制、能重現最小版、能用 production 工具，最後還能把成果接成可用工具。

內建 Agent Skills

Repo 內含 `.claude/skills`，README 特別列出兩個 skill：

| Skill | 用途 |

|---|---|

| `/find-your-level` | 10 題 placement quiz，評估學習者應從哪個 phase 開始，並產生個人路徑與時數估計 |

| `/check-understanding <phase>` | 針對某個 phase 產生 8 題測驗，檢查是否準備好進下一階段 |

這點對 Hermes 特別有參考價值：它把 curriculum 本身變成可被 agent 操作的學習系統，而不只是靜態教材。對欒帥的課程或內訓，可以借鑑這種「教材 + skill + quiz + personalized path」的設計。

與 AI 學習雷達 / 課程設計的用法

1. 作為 AI 工程學習路線圖

適合整理成：

  • 初學者：Phase 0–3，先補工具、數學、ML、DL。
  • AI 應用開發者：Phase 10–14，LLM、MCP、Agent Engineering。
  • 企業導入 / 工程主管：Phase 13–18，tools/protocols、agents、production、ethics。
  • 高階專題：Phase 19 capstone projects。

2. 作為「課程頁」或「內訓地圖」參考

可以借用它的階層設計：

```text

Level 1：AI 使用者素養

Level 2：AI 工具與 prompt workflow

Level 3：RAG / LLM App

Level 4：MCP / Tool use / Agent loop

Level 5：Production / Evaluation / Safety

```

3. 作為 Hermes skill 生態案例

這個 repo 的 `.claude/skills` 顯示一個重要方向：未來教材不只是 PDF / 網站，而是能被 agent 直接讀取、提問、測驗與生成個人路徑的活教材。

適合誰，不適合誰

| 對象 | 適合度 | 原因 |

|---|---|---|

| 想從使用 AI 進階到 AI 工程的人 | 高 | 課綱涵蓋底層、框架、agent 與 production |

| 資工 / 資管 / AI 課程設計者 | 高 | 可參考 phase 與 lesson contract 設計課程 |

| 已會 ML/DL、想補 LLM/Agent/MCP 的人 | 高 | 可直接跳 Phase 10–14 |

| 只想快速做 no-code automation 的人 | 中低 | 這套偏硬核工程與底層理解 |

| 想 1 週內完成完整 AI 全棧的人 | 低 | 官方估計約 314–320 小時，不是短期速成 |

導入建議

  • **不要從頭硬啃 503 課。** 先用 `/find-your-level` 或自評方式定位起點。
  • **以專案反推課程。** 例如要做 RAG agent，就優先看 LLM Engineering、Tools & Protocols、Agent Engineering。
  • **把 artifact 當成果。** 每課最後的 prompt / skill / agent / MCP server 比單純筆記更重要。
  • **配合 Hermes / Codex 實作。** 可讓 coding agent 讀 lesson、跑 code、補測試，再轉成自己的教學範例。
  • **注意內容仍在快速變動。** README、ROADMAP、AGENTS.md 對 lesson 數有不一致處，顯示 repo 正在演進，引用時需以當日 GitHub 為準。

判讀限制

  • 本文是 repo inspection 與 README / ROADMAP / AGENTS.md 摘要，尚未 clone 全 repo 或逐課執行測試。
  • README 顯示 503 lessons；AGENTS.md 仍有「435 lessons」文字；`.claude/skills/find-your-level` 的描述也出現「260+ lessons」。這可能是不同階段文件尚未完全同步，正式引用應以 README / ROADMAP 最新版為主。
  • Stars、forks、issues、30 天讀者與 page views 都是時間敏感數字，需視 GitHub / 官方網站最新資料更新。
  • 若要用於正式課程，建議先抽樣執行 Phase 0、Phase 1、Phase 14 的 lesson code / tests，確認教材品質與環境需求。
