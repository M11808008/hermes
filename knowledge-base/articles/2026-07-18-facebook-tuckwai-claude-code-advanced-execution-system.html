Claude Code 不只是寫程式：15 個進階功能其實是在打造「個人執行系統」

來源：TuckWai Ctw 的 Facebook Reel，主張「Claude Code 不是只能寫 code」，而是能自動跑任務、排程、檢查、並行處理，甚至用手機接手電腦裡的工作；貼文列出 `/schedule`、`/loop`、`/batch`、hooks、`/voice`、remote control 等進階功能，並提供中文 PDF。

一句話摘要

這則貼文的方向值得歸檔：Claude Code 正從「程式助理」變成一套可被命令、排程、鉤子、非互動模式與遠端觸發串起來的 agent 執行環境；但各 slash command 是否可用會受 Claude Code 版本、方案、平台與 bundled skills 影響，不能把每個社群列出的指令都視為所有帳號固定可用。

欒帥可用的整理框架

把這則貼文當成「Claude Code 進階能力地圖」會比單純收一份指令表更有價值：

  • Session 控制：`/clear`、`/compact`、`/context`、`/status` 類命令負責整理對話與工作狀態。
  • Workflow 命令：`/plan`、`/permissions`、`/model`、`/effort` 等負責控制 agent 的行為邊界。
  • 自動化入口：headless `claude -p`、Agent SDK、GitHub Actions，適合 CI / 批次任務。
  • 事件鉤子：hooks 用於格式化、測試、通知、阻擋危險命令與記錄審計。
  • 多代理與技能：skills、subagents、MCP 讓 Claude Code 從單一助手變成可擴充工作台。
  • GUI / 遠端操作：computer use 與 remote-control 類能力讓 agent 能跨出純 terminal，但要更重視權限與安全。

實務注意事項

  • 先在本機 Claude Code 執行 `/` 或查看 `claude --help`，確認實際可用命令；社群 PDF 可能比官方版本快或慢。
  • 排程與循環任務要加上停止條件、輸出上限與錯誤通知，避免 agent 無限執行或重複花費。
  • hooks 適合 deterministic guardrail；需要判斷的地方才用 prompt / agent hook。
  • remote control / computer use 類能力需避免讓 agent 接觸個資、金流、未授權網站或無法回復的 destructive action。
  • 若要教學，可設計「從 slash commands → hooks → headless → GitHub Actions」四層示範，讓學員理解它是執行系統而非單一聊天工具。

Why it matters

這則 Reel 的價值在於提醒：AI Coding Agent 的競爭不只在模型智商，而在「可被可靠調度」。當 Claude Code 能透過 commands、hooks、headless、GitHub Actions、computer use 與 remote-control 串接任務時，它就從寫程式工具轉成個人或團隊的 execution layer。
