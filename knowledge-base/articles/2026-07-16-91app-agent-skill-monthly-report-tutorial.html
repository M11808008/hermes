91APP 教學｜第一個 Agent Skill 從每月行銷報表開始

一句話摘要

91APP 建議第一支 Agent Skill 不要從大型自動化開始，而是從每月重複、格式固定、判斷標準明確的「行銷月報」下手：把 KPI、算法、資料來源、輸出格式與人工覆核點寫進 `SKILL.md`，讓 AI 每月依同一套標準整理報表，人再負責檢查與策略判斷。

原文重點

這篇 91APP Marketing Hub 文章把 Agent Skill 轉成行銷人容易落地的情境：每月行銷報表。

  • 第一支 Skill 應挑「高頻、規則明確、輸出固定」的任務；月報比一次性市場分析更適合。
  • 動手前要先定義四件事：固定看的指標、數字口徑、資料來源、輸出長相。
  • 最小可行 Skill 至少包含：Skill 資料夾、`SKILL.md`、範例輸入或理想輸出。
  • `SKILL.md` 的 frontmatter 需包含 `name` 與 `description`；description 要同時說明做什麼、何時啟用。
  • 月報 Skill 的流程是：整理規則 → 寫 description → 用 skill-creator 或支援工具建立 → 用測試資料試跑 → 補規則與範例後再交給團隊。
  • 上線前要檢查指標是否寫死、口徑是否定義、禁忌是否交代、是否有範例輸入、是否保留人工覆核點。

---

官方脈絡與交叉驗證

Brave Search 找到的官方與高可信來源顯示，91APP 對 Skill 結構的描述與主流 Agent Skills 文件一致：

  • Anthropic Agent Skills 文件與工程文章都把 Skill 描述為一個資料夾，核心入口是含 YAML frontmatter 的 `SKILL.md`，可附加 scripts、templates、references 等資源。
  • DeepLearning.AI 與 Anthropic 合作的 Agent Skills 課程強調 skill folder、`SKILL.md`、progressive disclosure，以及把通用 agent 轉成特定任務專家的做法。
  • Claude Code Skills 文件也延續 `SKILL.md` / frontmatter / supporting files 的格式，但實際安裝位置、工具支援與 plugin 行為要依平台文件為準。
  • 「開放格式 / 跨平台採用」是方向，但完整可攜性仍取決於各工具是否支援相同欄位、腳本能力、資源目錄與安全限制。

---

對 AI 學習雷達的保存價值

這篇值得保存，因為它補上了中文企業情境中的 Skill authoring 入門：不是單純教語法，而是教使用者先把「腦中的工作標準」顯性化。

對課程與團隊導入來說，可把這篇當成練習模板：

  • 從每月 / 每週重複任務開始，而不是追求一次到位的全自動系統。
  • 把資料口徑與輸出格式寫成可審查文件，避免每次 prompt 都遺漏規則。
  • 用去識別化資料試跑，先讓格式與口徑穩定，再逐步擴大資料來源。
  • 把 AI 定位成整理與初稿生成者，人仍負責數字檢查、異常判讀與策略決策。

---

行銷月報 Skill 的實作啟示

最小可行版本不需要複雜工具鏈，反而需要清楚規格：

  • 指標：sessions、轉換率、新會員、回購會員、廣告花費、營收、ROAS 等。
  • 口徑：ROAS = 營收 / 廣告花費；回購率 = 當月再次購買會員數 / 曾購會員數。
  • 結構：本月摘要、指標表格、異常變化、下月建議。
  • 禁忌：沒有資料支撐時不得寫成結論，只能列為待驗證假設。
  • 覆核：營收、會員分群、結論與建議必須由人確認。

若接上 91APP CDMP 或品牌會員資料，最重要的是不要把「工具模板」直接套用到所有品牌，而要把自家會員分群、標籤邏輯、比較基準與敏感資料規範寫進 Skill。

---

導入注意事項

  • 不要把會員個資、訂單明細、內部 token 或完整營收敏感資料寫進可分享的 Skill。
  • Skill 應保存規則與範例格式，不應保存每月原始資料本身。
  • 團隊共用前要建立 review 流程，確認指標、算法、資料來源與權限。
  • 若使用 Claude、ChatGPT、Codex 或其他支援 Skills 的工具，需分別確認目前帳號方案、管理員權限與安全掃描規則。
  • 月報 Skill 的價值在於穩定重複與減少交代成本，不代表可以省略人工判讀。
