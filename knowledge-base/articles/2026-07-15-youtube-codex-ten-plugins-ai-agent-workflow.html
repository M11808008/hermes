10 個插件 / 能力清單

以下清單根據影片描述、Bilibili mirror、CSDN/Jina mirror，以及官方來源交叉整理；其中有些是 OpenAI 官方 repo / docs 可確認的 plugin bundle 或能力面，有些則是作者推薦的第三方工作流：

  • Chrome：瀏覽器自動化、搜尋、填表、截圖、頁面測試；官方文件有 Chrome extension / Browser / Playwright 類能力，但登入與 cookie 權限要謹慎。
  • Computer Use：桌面級操作能力；OpenAI Codex 文件有 Computer Use 頁面，適合跨應用流程但需 sandbox 與人工確認。
  • GitHub：repo、issue、PR、review；openai/plugins 內有 github plugin，官方 MCP 文件也列 GitHub。
  • Build Web Apps：用自然語言生成與部署 web app / MVP；openai/plugins 內有 build-web-apps，OpenAI Codex Sites 方向相符。
  • Figma：讀取設計稿並轉前端；openai/plugins 內有 figma，官方 use case 也有 Figma designs to code。
  • IPFoxy Proxy Manager：代理與地區測試；需標為第三方/作者推薦，未在 openai/plugins 官方 repo 直接確認。
  • Presentations：簡報生成；OpenAI Codex features 文件列 Presentations artifact，但具體插件可用性依帳號/Workspace 而異。
  • Spreadsheets：表格分析與資料清理；OpenAI Codex features 文件列 Spreadsheets artifact，具體插件可用性依帳號/Workspace 而異。
  • HyperFrames：HTML/CSS/JS/媒體到影片的 agent workflow；openai/plugins 內有 hyperframes，HeyGen repo 也有 .codex-plugin。
  • Remotion：React/程式化影片生成；openai/plugins 內有 remotion，適合資料驅動與大量模板影片。

需要保留的 caveat

  • 「10 個必裝」是作者推薦排序，不是 OpenAI 官方最佳實踐；不同帳號、方案、地區、workspace 與 plugin marketplace 可能看到不同可用項目。
  • IPFoxy Proxy Manager 類代理工具需特別注意法規、平台條款與帳號風險；不應被一般化為所有人必裝。
  • Chrome / Computer Use 涉及瀏覽器 session、登入狀態、cookie、桌面權限與本機檔案，課堂或企業導入時應使用 sandbox profile、demo 帳號與人工 approval。
  • Build Web Apps / Sites 適合 prototype、landing page、內部工具與 demo；正式上線仍需檢查網域、資安、CI/CD、資料庫、監控與維運。
  • HyperFrames / Remotion 很適合 code-first video，但仍需處理素材授權、音訊授權、品牌規範與渲染品質。

對 AI 學習雷達的意義

這支影片可作為「Codex plugin 生態」入門索引。它提醒我們：AI Agent 的競爭已不只在模型本身，而在於是否能把工具、權限、工作流、artifact 產出與驗證流程整合成穩定系統。

對教學設計來說，可以拆成四類能力：

  • 操作型：Chrome、Computer Use，處理瀏覽器與桌面任務。
  • 工程型：GitHub、Build Web Apps、Figma-to-code，處理 repo、MVP 與設計交付。
  • 生產力型：Presentations、Spreadsheets，處理簡報、報告與資料分析。
  • 媒體型：HyperFrames、Remotion，處理可程式化影片、產品 demo 與大量內容變體。

最適合的課堂練習不是「一次裝滿 10 個」，而是讓學生選一個具體任務，列出：需要哪些 plugin、每個 plugin 要什麼權限、輸出如何驗證、有哪些資料不能交給 agent。

---

可直接使用的課堂作業

任務：用 Codex plugins 做一個「產品上市小型 campaign」。

  • Chrome / Browser：蒐集競品 landing page 與定價資訊。
  • Spreadsheets：整理競品功能與價格矩陣。
  • Presentations：產生 6 頁 pitch deck。
  • Build Web Apps 或 Sites：做一頁活動 landing page prototype。
  • Figma：若有設計稿，轉成前端 component。
  • HyperFrames / Remotion：產生 10 秒產品介紹 motion graphic。
  • GitHub：把產物提交到 repo，留下 diff 與 review 記錄。

評估標準：不只看產物好不好看，也看來源是否保存、權限是否合理、輸出是否可重現、是否有人工驗證與風險說明。

保存重點：Codex plugins 正在把「AI 寫程式」擴展成「AI 交付工作成果」；但越靠近瀏覽器、桌面、帳號與外部服務，越需要 sandbox、權限治理與人工檢查。
