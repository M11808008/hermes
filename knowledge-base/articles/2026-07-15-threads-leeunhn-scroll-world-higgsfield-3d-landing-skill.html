---

它實際做的是什麼

`scroll-world` 的核心不是「讓瀏覽器即時算 3D」，而是「用生成式影片 + scroll scrub 偽裝成連續鏡頭飛行」。官方 SKILL.md 說明的 pipeline 是：

  • 訪談使用者：產業／品牌、brand kit、藝術方向、旅程段落與 CTA。
  • 用 Higgsfield 生成每個場景的 cohesive isometric diorama stills。
  • 為每個場景生成「dive-in」鏡頭影片。
  • 為相鄰場景生成 connector clips，讓上一段最後一幀與下一段第一幀接近 frame-identical，避免切換時跳動。
  • 用 portable vanilla JS scrub engine 將影片時間軸綁到 scroll position。
  • 輸出可放進 plain HTML、Next.js、Vue 或其他前端框架的互動頁面。

官方特別強調：成敗關鍵是 seams 必須 frame-identical；如果 connector 沒做好，捲動時會看到明顯 pop。

---

為什麼這件事重要

這個專案代表 Agent Skill 正在從「指令模板」進化成「完整創意製作流程」。它不是只給一句 prompt，而是把製作沉浸式 landing page 所需的專業知識包進 workflow：

  • 如何訪談品牌與故事段落；
  • 如何保持每個場景的 visual style 一致；
  • 如何規劃鏡頭進出與段落銜接；
  • 如何處理影片素材、frame extraction、encoding；
  • 如何把生成素材封裝成可嵌入網站的 runtime。

對 AI 學習雷達來說，這是「creative agent as production pipeline」的好案例：AI 不是取代單一工具，而是把多工具流程、決策順序與品質檢查變成可重複執行的技能。

---

實務提醒與限制

  • 這不是完全免費的一鍵工具：README 明確要求 Higgsfield CLI、登入認證與 credits。
  • 它依賴預渲染影片，不是可互動探索的真 3D 場景；適合品牌敘事 landing page，不適合需要自由操作的 3D 應用。
  • 生成時間可能很長：SKILL.md 提到 Higgsfield generation 可能每個 3–8 分鐘，需用背景任務與輪詢處理。
  • 手機版仍是 beta：官方 skill 要求明確詢問是否產出 mobile-optimized version，且提醒 16:9 影片在手機直式裁切可能失焦或卡頓。
  • 專案價值不只在視覺酷炫，而在「prompt → assets → scroll runtime → QA」的完整封裝方式。
