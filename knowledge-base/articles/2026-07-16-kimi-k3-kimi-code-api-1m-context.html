Kimi K3 正式開放｜1M Context、Kimi Code 與 API 的旗艦模型更新

一句話摘要

Kimi K3 已在 Kimi Code 與 Kimi API 官方頁面正式出現，定位為 Kimi 目前最強旗艦模型，支援最高 1M token context，面向大型程式專案、知識工作、深度推理與 agentic coding；Kimi Code 同時提供 K3、K2.7 Code 標準版與 K2.7 Code HighSpeed 三種模型 ID。

---

Kimi Code 三個模型 ID

官方 Kimi Code Docs 對三個模型 ID 的定位如下：

  • `k3`

- 模型：Kimi K3

- 定位：Kimi 目前最強旗艦模型

- 強項：coding、games / 3D、knowledge tasks

- Context：最高 1M

- Reasoning：支援 `thinking.effort:max` / `reasoning_effort:max`，低 / 高 effort 後續支援

  • `kimi-for-coding`

- 模型：Kimi K2.7 Code

- 定位：成熟穩定的 coding 模型

- Context：256K

- Reasoning：Thinking ON

  • `kimi-for-coding-highspeed`

- 模型：Kimi K2.7 Code HighSpeed

- 定位：K2.7 Code 高速版，能力相同但輸出速度約 6 倍

- Context：256K

- 使用限制：官方頁標示需 Allegretto 或更高方案

---

方案與 Context 限制

Kimi Code Docs 對 K3 可用性與 context 上限的說明：

  • Andante：不支援 K3。
  • Moderato：K3 最高 256K context。
  • Allegretto 及以上：K3 最高 1M context。

因此「K3 支援 1M context」是模型與高階方案能力，不代表所有 Kimi Code 使用者都可直接使用 1M。

對 AI 學習雷達的保存價值

Kimi K3 的重點不只是新模型，而是「長上下文 + coding agent + 第三方工具」的產品化：

  • 1M context 讓大型 repo、長文件、多檔案任務更容易被放進同一工作階段。
  • Kimi Code 與第三方 coding agents 的支援，使 Kimi 進入 Cursor、Cline、Roo Code 等 agentic coding 工作流。
  • K3 對 games / 3D、前後端工程與大型知識任務的定位，與近期 AI coding 從單檔補全走向 repo-level agent 的趨勢一致。
  • 對 Hermes / Codex / Claude Code 工作流來說，K3 是值得測試的「長上下文程式代理模型」候選。

---

導入注意事項

  • 1M context 不是免費午餐：長上下文會增加成本、延遲與 cache 行為的不確定性，應先用代表性 repo 做 benchmark。
  • Kimi Code docs 提到切換模型後，既有 context cache 不再命中新模型，剛切換時用量可能上升；建議新模型用新 session。
  • HighSpeed 版本雖快，但官方標示會消耗約 3 倍 quota，需看任務是否真的需要高速輸出。
  • 社群影片與測評可作為參考，但正式導入仍應以 Kimi 官方 docs、API 實測、價格與方案限制為準。
  • 若用於第三方 coding agent，需確認 API key 權限、資料外流邊界與 repo 私密性。
