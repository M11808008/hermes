欒帥可用的整理框架

  • 先看官方 cheatsheet：把常見詞彙與命令建立成共通語言。
  • 再看本機 `/` 清單：Claude Code 的 slash commands、skills、plugins 會因版本、設定與帳號而異。
  • 對照 permissions/settings：任何自動執行 shell、讀寫檔案、連外 API 的技巧，都要先知道權限邊界。
  • 分離 productivity tips 與 unsafe hacks：提升效率可以教；繞過政策、竊取資料、規避付費或濫用他人系統不可教。
  • 把社群 cheat sheet 轉成課堂 checklist：每一項都標註「官方文件確認 / 本機驗證 / 社群未驗證」。

實務注意事項

  • 不要因為標題寫「illegal」就複製高風險做法；Claude 與 Claude Code 的安全模型仍以 permissions、tool prompts、審批流程與使用政策為邊界。
  • 社群 cheat sheet 常混合官方功能、個人偏好、舊版命令、未公開技巧與行銷話術；需逐項查證。
  • 若要整理給學員，建議把「可穩定示範」與「需現場確認」分欄，不要承諾每個帳號都有同樣功能。
  • 對 MCP、hooks、subagents、skills 這類高槓桿功能，務必加入資料外洩、命令注入、錯誤自動化與成本失控的防護說明。

Why it matters

Claude 技巧短影音正在從「prompt tips」轉向「agent operations tips」。真正值得學的不是某個神秘技巧，而是如何把碎片化技巧轉成可驗證的操作手冊：先依官方 cheatsheet 建立概念，再用本機環境驗證，最後加上權限與安全邊界。這樣才能把社群靈感變成可教、可複製、可維護的 AI 工作流。
