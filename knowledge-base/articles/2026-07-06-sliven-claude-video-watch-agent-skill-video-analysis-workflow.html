一句話摘要

`claude-video` 的重點不是「幫你摘要影片」而已，而是把影片拆成影格、場景、字幕與時間戳，讓 Claude Code 能像看過影片一樣回答問題；這讓影片內容可以被 agent 轉成可分析、可學習、可重用的創作方法。

Threads 貼文重點

sliven0722 在 Threads 分享 GitHub 熱門專案 `claude-video`，指出它目前已累積超過 3,400 顆 star。貼文強調，這個工具的重點不是單純摘要影片，而是讓 Claude / Claude Code 更完整理解影片內容。

使用方式是提供 YouTube、TikTok 或其他影片連結，工具會自動：

  • 下載影片。
  • 擷取關鍵影格並分析場景變化。
  • 轉錄音訊內容。
  • 將視覺與語音資訊整理成 Claude 可以直接理解的內容。

貼文最值得保存的觀點是：這提供了一種新的影片分析工作流。使用者可以分析爆紅影片，研究：

  • 開頭幾秒如何吸引觀眾。
  • 鏡頭切換與節奏如何安排。
  • 畫面、字幕與旁白如何互相配合。
  • 哪些內容可能提升觀看完成率。

接著再交給 AI：「根據這支影片的節奏與敘事方式，重新規劃我們產品的影片腳本。」這不是複製別人的內容，而是把影片拆解成可分析、可學習、可重複運用的創作方法。

它如何運作

README 描述的流程如下：

  • 使用者貼上影片 URL 或本機影片路徑，並提出問題。
  • `yt-dlp` 先檢查影片是否有字幕。若只需要 transcript 且有字幕，就不下載完整影片。
  • 需要視覺資訊時，`ffmpeg` 依 detail mode 擷取影格：可用 keyframes、scene-change frames 或 uniform fallback。
  • 轉錄來源優先用平台原生字幕；若沒有字幕，則抽取音訊並用 Whisper API fallback。Groq Whisper 是偏好選項，OpenAI Whisper 是替代選項。
  • 工具輸出 timestamped transcript 與帶時間戳的 frame paths，Claude 再讀取影格圖像並結合逐字稿回答。
  • 工作目錄最後可清理；若使用者可能追問可保留。

這個設計讓 agent 的回答能同時 grounded in：

  • 影片畫面：每個 timestamp 的截圖。
  • 音訊內容：字幕或 Whisper 逐字稿。
  • 使用者問題：例如某個時間點、開場 hook、視覺節奏、UI bug、影片摘要等。

安裝方式

README 提供幾種安裝面：

Claude Code：

```text

/plugin marketplace add bradautomates/claude-video

/plugin install watch@claude-video

```

Codex、Cursor、Copilot、Gemini CLI 或其他 Agent Skills hosts：

```bash

npx skills add bradautomates/claude-video -g

```

claude.ai web：下載 release 裡的 `watch.skill`，到 Settings → Capabilities → Skills 上傳。

第一次執行時，skill 會檢查 `yt-dlp` 與 `ffmpeg`。macOS 可透過 Homebrew 自動安裝；Linux / Windows 會印出對應安裝指令。Whisper API key 不是必須，但影片沒有字幕時需要 Groq 或 OpenAI key 才能轉錄音訊。

Detail modes 與 token 成本

`/watch` 有多種 detail mode，這是它很值得學習的設計：

| Mode | 用途 | 成本特性 |

|---|---|---|

| transcript | 只取字幕或逐字稿，不取影格 | 最省 token，適合純內容摘要。 |

| efficient | 快速 keyframe pass，最多約 50 frames | 速度快，適合初步掃描。 |

| balanced | 預設，scene-aware frames，最多約 100 frames | 品質與成本折衷。 |

| token-burner | scene-aware uncapped | 最完整但高 token 成本。 |

README 特別提醒，影格是主要 token 成本來源。長影片若超過 10 分鐘，預設模式會變成稀疏掃描；若問題集中在某段，應使用 `--start` / `--end` 聚焦，讓同樣 token 花在更密集的時間區間。

為什麼這很重要

1. 影片從「黑盒內容」變成可檢索素材

過去 AI 對影片常只能讀標題、描述或字幕。`claude-video` 讓 agent 同時讀到畫面與語音，影片就可以被拆成：

  • 開場 hook。
  • 場景切換。
  • 字幕與旁白搭配。
  • 畫面上的 UI / code / slide。
  • 節奏、段落與敘事結構。

這對創作者、課程設計、產品行銷、競品研究都很有價值。

2. 它示範了「multimodal skill」的標準寫法

這個 repo 的重點不只是能看影片，而是把多媒體處理封裝成 agent skill：

  • skill contract 寫在 `skills/watch/SKILL.md`。
  • runtime script 在同一個 self-contained skill folder 內。
  • 支援 Claude Code plugin、Codex / Cursor / Copilot Agent Skills CLI、claude.ai web bundle。
  • 有 setup preflight、dependency installer、detail mode、token budget、cleanup 與安全說明。

對 AI 學習雷達來說，這是很好的「Agent Skill 工程化」案例。

3. 影片創作可以從模仿變成結構學習

Threads 貼文的判讀很關鍵：使用這種工具不是要抄別人的內容，而是把影片拆解成可遷移的方法。例如：

  • 為什麼前 3 秒有吸引力。
  • 每幾秒切一次鏡頭。
  • 字幕出現的節奏。
  • 旁白如何鋪陳問題與轉折。
  • 視覺示例如何支撐文字主張。

這些可以轉成自己的產品影片、課程短影音、教學腳本或社群內容設計。

可轉成 Luarn-Pin 工作流的地方

  • AI 課程影片研究：把 YouTube / TikTok / Reels 教學影片拆成開場、段落、視覺設計與 CTA 模板。
  • 產品短影音腳本：先分析爆紅案例的節奏，再讓 agent 依 Luarn-Pin 的課程或工具重新規劃腳本。
  • Bug reproduction：學生或使用者傳螢幕錄影時，可讓 agent 直接定位畫面哪裡出錯。
  • 課程知識庫：把長影片拆成 timestamped notes，加入 Obsidian / AI 學習雷達私密區，而不是只留影片連結。
  • Agent skill 設計教材：用 `/watch` 做示範，說明好的 skill 應該包含 setup、依賴、成本模式、失敗處理、安全界線與跨工具安裝面。

風險與限制

  • 長影片預設 frame cap 會造成稀疏掃描；分析細節時應聚焦片段。
  • 原生字幕可能不準，Whisper fallback 需要 API key 並會上傳抽取音訊。
  • 工具會在本機下載影片與產生影格，需注意版權、平台條款與本機暫存資料清理。
  • 影片分析不能取代人類對創意、品牌調性與倫理的判斷。
  • 對他人爆紅影片做結構分析可以學習方法，但不應複製內容、畫面或腳本。

後續可追蹤

  • Agent Skills CLI 與 Claude Code plugin 生態是否會讓這類 multimodal skills 普及。
  • `/watch` 是否能與 Obsidian / AI Vault 直接串接，將分析輸出變成結構化筆記。
  • 是否能加入「影片腳本重寫模板」或「短影音節奏評分」等上層 skill。
  • 對中文影片與繁中字幕的支援品質，以及 Groq / OpenAI Whisper 的成本與準確度差異。
