OpenAI Codex Record & Replay｜示範一次 Mac 工作流，自動變成可重用 Skill

來源是 Threads 帳號「碩」分享 OpenAI Codex 新功能 Record & Replay，並用 OpenAI Developers 官方文件補強。核心是：使用者在 Mac 上示範一次工作流程，Codex 觀察操作與視窗內容後，將流程整理成可重複使用、可編輯的 Skill。

一句話摘要

Record & Replay 是 Codex 從「你用 prompt 教它」走向「你做一次給它看」的功能：適合重複、個人偏好強、很難用文字說清楚的流程，例如上字幕、請假申請、整理資料、壓縮影片、報銷或下載固定報表。

Threads 串文重點

  • 使用者只要示範一次操作流程，Codex 會把過程記錄下來。
  • Codex 會把整個過程轉成可重複使用的 Skill，之後可直接要求它執行。
  • 原貼舉例：官方展示 YouTube 放字幕、請假申請、整理資料。
  • 作者補充目前翻文件看起來似乎只開放 macOS，並在 Codex 外掛安裝。
  • 作者實測本地影片壓縮流程：示範影片放哪裡、用什麼腳本，Codex 可成功複製並執行。
  • 回覆補充：呼叫 record replay 工具後會跳出小彈窗並開始錄影；做完後停止錄影，Codex 會自動轉成 skill。

OpenAI 官方文件確認資訊

  • Record & Replay is available on macOS。
  • 初始可用地區不包含 EEA、United Kingdom、Switzerland。
  • 需要 Computer Use 可用且已啟用。
  • 適合已知如何完成、步驟穩定、成功標準清楚的 workflow。
  • 開始方式：在 Codex app 開啟 Plugins → + menu → Record a skill → 補充 context → 批准錄製 → 在 Mac 上示範流程 → 停止錄製。
  • 停止後，Codex 會檢查捕捉到的 workflow 並草擬 skill，內容包含何時使用、需要哪些 inputs、步驟與驗證方法。
  • replay 時可在新 thread 要 Codex 使用 generated skill，並提供本次變動的值，例如檔案、issue、日期區間。

2026-06-21 補充：張維峰 Facebook 貼文與 OpenAI Developers X 公告

欒帥後續分享張維峰的 Facebook 公開貼文，內容把 Record & Replay 解釋成「把螢幕操作變成 skill」：示範一次流程給 Codex 看，Codex 會產生可檢查、可編輯、可重複使用的 skill。貼文補充的價值在於把官方文件轉成更清楚的中文使用情境：報帳、預約停車位、建立正確設定的 issue、發布影片、下載定期報告、提交休假申請，都是「做一遍比寫 prompt 快」的典型流程。

同貼文留言也附上 OpenAI Developers X 官方公告。官方 X 文案確認三個要點：Show Codex a workflow once；reuse it as a skill；Codex turns that demo into an inspectable, editable skill。這與 OpenAI Developers 文件相互印證；因此本文維持官方文件為 canonical source，並把 Facebook / X 作為補充來源與中文教學轉譯材料。

2026-06-23 補充：Prompt Case Threads 與 Skill Creator 入口

欒帥後續分享 Prompt Case 的 Threads 貼文。這則貼文重點是把 Record & Replay 說成「不會寫 Skill 也能先錄一遍」：先在 Codex Plugins 裡加入 Record & Replay，輸入簡短任務目標，接著在錄影視窗示範實際操作，Codex 會呼叫內建 Skill Creator，把示範流程封裝成可複用的技能模組。

本次補充不另開新條目，原因是核心 canonical 仍是 OpenAI Developers 的 Record & Replay 文件；新貼文的價值在於補強「Skill 製作門檻下降」與「computer use / 介面操控流程最適合用錄製示範」兩個教學角度。Brave 搜尋同時確認 OpenAI Developers 的 Agent Skills / Plugins 文件已明確寫到：Record & Replay 會記錄 workflow、檢查步驟，並從示範草擬 reusable skill；Plugins 文件也將 Record & Replay 定位為「Show Codex a workflow once and turn it into a reusable skill」。

可補充成三個教學重點：

  • 不是把影片當巨集重播，而是把示範萃取成 inspectable / editable skill。
  • 最適合「人知道怎麼做，但用文字描述很麻煩」的 GUI / browser / computer-use 流程。
  • 錄完後仍要人工審查 skill 的觸發條件、輸入、驗證步驟與敏感資料邊界。

對欒帥的實用判讀

  • 這是 AI workflow automation 的關鍵門檻降低：不用先把 SOP 寫成完美 prompt，而是用示範建立第一版 skill。
  • 對內容營運很有用：影片壓縮、字幕處理、上傳平台、報表下載、素材整理都屬於高重複、偏好明確的流程。
  • 它不像傳統 RPA 只記錄座標；官方描述重點是把示範流程轉成 skill context，再用 Computer Use、browser actions、plugins 執行。
  • 對企業而言，最佳做法是先從低風險、無敏感資料的流程開始，錄完再人工 review 生成的 SKILL.md。
  • 它與 Hermes skills 概念高度相通：一次成功的操作經驗應該被沉澱成可重用程序，而不是留在一次性對話裡。

錄製品質建議

  • 示範要短、完整、聚焦，不要把無關清理動作一起錄進去。
  • 開始前先告訴 Codex 目標與會變動的 inputs。
  • 使用真實但不含秘密的測試資料。
  • 錄完後補充隱性偏好：命名規則、預設欄位、分支判斷、驗收標準。
  • 若要跨團隊穩定分發，官方建議可考慮包成 plugin，而不只是單一 recorded skill。

限制與注意

  • 本功能目前以 macOS 與 Computer Use 為前提，且有地區可用性限制。
  • 錄製過程可能捕捉視窗內容，應避免密碼、金鑰、客戶個資與敏感內部資料。
  • 複雜流程仍需驗證容錯；錄完後要檢查 skill 是否誤解條件或漏掉驗收步驟。
  • Min Tseng Chang 在 Facebook 留言提醒：若流程跑過幾次後可被明確程式化或批次化，長期可能不必每次都包在 skill 裡消耗 token；可把穩定步驟改成腳本/API/RPA，保留 agent skill 處理例外、判斷、摘要與人類語氣報告。
