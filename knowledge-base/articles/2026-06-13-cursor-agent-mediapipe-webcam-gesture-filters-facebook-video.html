Cursor Agent + MediaPipe｜用自然語言打造 webcam 手勢濾鏡互動 demo

來源為 ui.ananya 的 Facebook 影片。影片 caption 直接公開三段 prompt，展示如何用 Cursor Agent 從零生成一個 webcam 互動濾鏡 demo：用 MediaPipe Hands 追蹤食指白點，從 21 個 hand landmarks 判斷 fist、peace sign、index finger、open hand，再把不同手勢映射成 dither、VHS chromatic aberration、spotlight、water ripple 等視覺效果。

一句話摘要

這是一個很好的「Vibe Coding 進階範例」：不是只叫 AI 做靜態網頁，而是把瀏覽器 webcam、MediaPipe 即時手部追蹤、手勢分類、Canvas/影像濾鏡與互動映射拆成連續 prompt，讓 Cursor Agent 逐步寫出可跑的互動作品。

技術架構拆解

| Layer | 角色 | 實作要點 |

|---|---|---|

| Camera input | 取得即時 webcam stream | `getUserMedia` 權限、全螢幕顯示、鏡像處理 |

| Hand tracking | 偵測手掌與手指點位 | MediaPipe Hand Landmarker / Gesture Recognizer |

| Landmark logic | 從 21 landmarks 推論手勢 | 可用距離、角度、指尖/關節相對位置判斷，也可用官方 Gesture Recognizer |

| Gesture mapping | 把手勢轉成狀態 | fist、peace、pointing、open hand 對應不同 filter |

| Visual effects | 即時影像濾鏡 | Canvas shader、pixel processing、CSS filter 或 WebGL |

| Agent workflow | 自然語言迭代 | 每次只加一個效果，讓 Cursor Agent 小步修正 |

| 7. 優化安全與隱私 | webcam 不上傳、local-only、加權限提示與停止按鈕 |

實務限制

  • **零程式知識不等於零審查**：Cursor 可能能生成 demo，但仍需要有人檢查攝影機權限、套件來源、瀏覽器相容性與錯誤處理。
  • **手勢辨識受光線與鏡頭影響**：低光、手部遮擋、背景雜訊都會讓 landmarks 不穩。
  • **濾鏡效能可能吃 CPU/GPU**：Canvas pixel-level processing、VHS、water ripple 若未優化，手機或舊電腦會掉幀。
  • **自訂手勢規則容易 brittle**：直接用 21 landmarks 寫規則可快速 demo，但要產品化通常需更完整測試。
  • **Agent 生成 code 可能引入不必要依賴**：需避免加入過重套件或不明 CDN。

安全檢查清單

  • **webcam 權限**：明確顯示是否正在使用攝影機，提供停止鏡頭按鈕。
  • **影像不上傳**：若是 local demo，確認 webcam frame 不被送到第三方 API。
  • **CDN / npm 套件審查**：只使用官方或可信來源，例如 Google MediaPipe 官方 package。
  • **Cursor Agent 權限**：若讓 Agent 執行 terminal command，需檢查 install script、postinstall、網路請求與檔案變更。
  • **避免敏感場景**：不要在含個資、公司機密、未成年人影像的場景下未經同意錄製或分析。
  • **部署前檢查**：公開 demo 前移除 API key、debug log、截圖與測試資料。

欒帥可延伸應用

  • **AI 工作坊 demo**：讓學員用三段 prompt 做出可玩的 webcam 互動濾鏡，理解 AI coding agent 的迭代方式。
  • **課程首頁互動元件**：用手勢觸發不同課程模組視覺效果，做成展場/招生 demo。
  • **簡報控制器**：將 pointing / open hand / fist 映射成下一頁、上一頁、雷射筆、暫停。
  • **知識庫視覺實驗**：把 gesture filter 做成「AI 學習雷達」互動案例文章，展示 AI 生成前端互動作品。
  • **無障礙探索**：測試用簡單手勢代替鍵盤/滑鼠操作，但需加入誤觸防護。

判讀限制

  • Facebook 影片證明「這組 prompt 能被作者用 Cursor Agent 做出 demo」，但不等於任何人都能在所有環境一次成功。
  • Cursor Agent 的輸出品質會受模型、專案初始狀態、依賴版本與使用者是否能讀懂錯誤訊息影響。
  • MediaPipe 21 landmarks 與 Gesture Recognizer 官方可行，但自訂 fist / peace / pointing / open hand 的規則仍需測試與調參。
  • 社群 caption 的「Zero code knowledge needed」適合描述入門門檻降低；若要部署成可靠產品，仍需要工程審查。
