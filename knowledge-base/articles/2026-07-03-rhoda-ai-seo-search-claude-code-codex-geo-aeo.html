羅達 Rhoda｜用 Claude Code/Codex 打造 AV8D AI SEO Search：SEO、GEO、AEO 三維健檢工具

一句話摘要

這則貼文是一個 AI coding 真實案例：用 Claude Code / Codex 把 SEO 經驗產品化成免費健檢工具，核心不是炫技，而是規則式評分、反爬處理、async job、部署與持續迭代。

Facebook 可擷取資訊

  • 作者：羅達 Rhoda（AV8D.levelup）。
  • 產品：AV8D AI SEO Search。
  • 產品定位：免費 GEO 軟體與 SEO 測評工具，輸入網址約 90 秒得到 SEO / GEO / AEO 三維度分數，也可拉競品比較。
  • 背景：作者過去帶過 SEO 團隊，看到 AI 生成網站越來越多，但許多網站不知道是否被 Google / AI 搜尋看見。
  • 核心觀點：Google 排第一不代表 AI 會引用你；GA 也不一定看得出 AI 可見度。

架構拆解

貼文描述的架構非常值得保存：

1. 抓取層 fetch

  • 使用 `curl_cffi` 偽裝 Chrome 131 TLS 指紋與瀏覽器 headers。
  • 同網址 5 分鐘 cache。
  • 偵測 Cloudflare 封鎖、robots / sitemap 驗簽。
  • 避免 SPA 把首頁當 200 假冒 robots.txt。

2. 評分引擎 scoring

  • SEO / GEO / AEO 三維度。
  • 每維 6 個指標。
  • 用 BeautifulSoup 與規則式 heuristic，而非 LLM。
  • 加權平均成 0–100 分。

3. 外部訊號 integrations

  • Google PageSpeed / Lighthouse。
  • JSON-LD、microdata、RDFa、Open Graph。
  • 10 大 AI 爬蟲 robots 開放度。

4. API / 前端 / 部署

  • Flask `/api/analyze`。
  • 因行動網路長請求容易斷線，改為 async job：送出 + 輪詢。
  • 前端單檔 HTML + 原生 JS。
  • nginx + Cloudflare + VM + rsync。
  • 提交 GSC / Bing，並監測 Claude / GPTBot 等 AI 爬蟲。

迭代與踩坑

貼文列出的踩坑是這篇最有價值的地方：

  • Cloudflare WAF 擋公開網域爬取：用 `curl_cffi` 偽裝 TLS 指紋與 headers，並偵測封鎖頁。
  • gunicorn 自己 fetch 自家網域造成 deadlock：worker 數必須 ≥2。
  • PageSpeed 四維同跑 timeout：只跑 Performance + SEO。
  • SPA 假冒 robots.txt：加內容簽名驗證。
  • 行動網路 Failed to fetch：改 async job pattern。
