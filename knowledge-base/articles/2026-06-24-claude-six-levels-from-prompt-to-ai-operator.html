一句話摘要

這則 Reel 值得收錄的重點是：高手使用 Claude 的差別不在於「提示詞更長」，而是逐步把 Claude 從回答問題的聊天框，升級成會先訪談、會自我檢查、能讀取專案資料、可操作工具、能打包成 skill / plan mode，最後變成需要人類批准與指揮的 AI 工作團隊。

六個段位的工作流解讀

1. 初學者：只把 Claude 當問答機

影片開頭批評的低階用法是「問一個問題，拿到答案後就關掉」。這種方式沒有背景、沒有目標、沒有輸出標準，Claude 很容易給出一般化答案。

可改善方式：不要只問「幫我寫一封銷售信」，而要補上受眾、產品、價格、語氣、禁忌、成功標準與後續用途。

2. 讓 Claude 先訪談你：Make Claude interview you

畫面中可見提示詞概念：「Help me write a sales email to close a big deal. Before you answer, ask me any questions that you need.」

這對應到一個重要技巧：當任務條件不足時，先要求 Claude 反問，而不是直接輸出。這可以把 Claude 從「猜答案」變成「先補需求」。

可重用 prompt：

請先不要直接回答。請像顧問一樣訪談我，列出完成這個任務前你必須知道的 5–10 個問題。等我回答後，再產出第一版成果。

3. 讓 Claude 自我檢查：Make Claude Self-Check

影片中提到「第二個提示詞是在生出結果後，自動審查更正」。這可視為讓 Claude 增加一層品質檢查：邏輯、漏洞、語氣、受眾、格式、是否符合原始目標。

可重用 prompt：

請先依任務產出結果。完成後，請用審稿人角度檢查：1. 是否符合目標；2. 是否有邏輯漏洞；3. 哪些地方太空泛；4. 是否有更好的結構；5. 請直接給我修正版。

4. 進階玩家：建立 Project 與個人使用說明書

影片中可見「Create a project」「個人使用說明書」「全部丟到他的資料庫」。這與 Claude Projects 官方功能高度相關：Projects 可讓使用者把文件、文字、程式碼或其他檔案放入專案知識庫，讓 Claude 在同一專案中持續使用脈絡。

對欒帥來說，這可以做成「個人 AI 作業系統」：

  • 個人背景與偏好。
  • 常用語氣與設計標準。
  • 課程定位與受眾。
  • 品牌資料與簡報規範。
  • 常用輸出格式。
  • 禁止事項與審稿標準。

5. Operator / 數位工頭：讓 Claude 串工作流與行事曆

影片中把第 4 段稱為「The Operator / 數位工頭」，並提到串聯行事曆、每天早上整理一份、變成只需要批准的工頭。這代表 Claude 不只是產生文字，而是進入工作流編排：讀資料、整理任務、提出日程、準備信件或報告，再交由人類批准。

這層的重點不是全自動，而是「人類審核與批准」：Claude 可以先整理、排程、草擬、檢查，但高風險動作仍應由人確認。

6. Builder / 指揮官：把流程打包成 Skill / Plan Mode

影片後段出現「5. The Builder」「把它打包變成一個 skills」「Plan Mode」「6. 終極指揮官」。這與 Claude Code Skills、Plan mode 或可重用工作流的概念相通：把一次成功的流程沉澱成可重複使用的技能，而不是每次重寫 prompt。

可落地成：

  • `SKILL.md`：記錄觸發條件、步驟、輸出格式、坑點。
  • `templates/`：保存範本，如報告、簡報、社群貼文。
  • `references/`：保存背景資料與品牌規範。
  • `scripts/`：自動化固定操作，例如轉檔、檢查、部署。
  • Plan mode：先規劃步驟，等確認後才執行。

對欒帥的實作建議

1. 把「AI 學習雷達」變成 Claude Project

可建立一個 Claude Project，放入：

  • AI 學習雷達首頁與 index。
  • 課程定位與常用受眾。
  • 欒帥常用語氣、設計標準與輸出格式。
  • 文章摘要模板。
  • 簡報與課程企劃模板。

這樣 Claude 處理新文章、簡報或課程草案時，不需要每次重新交代背景。

2. 對重要任務固定使用「訪談 → 初稿 → 自檢 → 修正版」

比起直接要求 Claude 一次完成，建議固定四步：

  • Claude 先問問題。
  • 產出第一版。
  • 自我審查與列出弱點。
  • 產出修正版與下一步。

3. 把高頻工作打包成技能

例如：

  • 課程 landing page 規劃。
  • AI 工具文章整理。
  • Facebook / Threads 連結歸檔。
  • 簡報大綱與樣張生成。
  • 會議紀錄轉行動項目。

每個流程都應有觸發條件、輸入資料、步驟、輸出格式與人工審核點。
