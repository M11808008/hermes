Leo Wu｜Browser Use video-use：讓 Codex、Claude Code、Hermes 等 Coding Agent 具備影片剪輯能力

一句話摘要

video-use 是「AI agent 影片剪輯」很值得追蹤的開源案例：它不是讓模型逐幀看完整支影片，而是把影片轉成文字、時間軸、波形與必要視覺切片，讓 coding agent 透過腳本與 ffmpeg 做可控剪輯。

Threads 貼文重點

Leo Wu（@wutsungtse）在 Threads 介紹 Browser Use 團隊推出的 `video-use`：

  • 它是開源 Skill。
  • 主要面向 Codex、Claude Code、Cursor、Hermes Agent 等 AI Coding Agents。
  • 目標是讓這些 agent 具備影片剪輯能力。
  • 它不是要取代 Premiere 或 CapCut 這類傳統剪輯工具。
  • 它採用「prompt-engineering + 工具腳本」工作流。
  • LLM 先讀影片轉寫文本，需要時再呼叫視覺化工具輔助判斷，最後透過 ffmpeg 等工具完成剪輯任務。
  • 核心價值：AI 不必真的「看懂」整支影片，而是透過文字、時間軸與工具協作，完成更精準、可控的影片剪輯。

官方 README 描述的能力

README 列出的能力包括：

  • 剪掉 filler words、false starts 與 dead space。
  • 自動做 color grade。
  • 每個剪點加 30ms audio fades，避免爆音。
  • 依風格燒入字幕，預設為 2-word uppercase chunks，可自訂。
  • 透過 HyperFrames、Remotion、Manim 或 PIL 生成動畫 overlay，並可由 sub-agents 平行處理。
  • 在每個 cut boundary 自我檢查 rendered output，再交付給使用者。
  • 用 `project.md` 保存 session memory，讓下一週的剪輯 session 能延續脈絡。

核心工作流

README 的關鍵設計是：LLM 不直接觀看整支影片，而是「讀」影片。

Layer 1：Audio transcript

每個 source 先透過 ElevenLabs Scribe 取得 word-level timestamps、speaker diarization 與 audio events，例如笑聲、掌聲、嘆氣。所有素材被壓成 `takes_packed.md`，成為 LLM 的主要閱讀視圖。

Layer 2：Visual composite

在需要視覺判斷時，`timeline_view` 會輸出某段時間範圍的 filmstrip、waveform、word labels PNG。這不是把所有 frames 丟給模型，而是在剪點、重拍比較、停頓判斷等關鍵時刻才呼叫。

Pipeline

README 描述的流程：

Transcribe → Pack → LLM Reasons → EDL → Render → Self-Eval

若 self-eval 發現問題，會修正並重新 render，最多重跑數次。

為什麼這對 AI Agent 很重要

這個專案的價值在於它把影片剪輯拆成 agent 可操作的結構：

  • 將不可控的視覺長影片轉為可索引的 transcript / timestamps。
  • 用工具補足模型看不到或不該全看的部分。
  • 以 EDL、ffmpeg、render、自我評估形成可重跑 pipeline。
  • 讓 agent 不只是「建議剪哪裡」，而是能真正產出檔案。

這與 Browser Use 將網頁轉成結構化 DOM 的思路相似：不把所有畫面 token 化，而是提供模型能推理與操作的中介表示。

對 Luarn-Pin 的價值

這則內容可放入三個脈絡：

  • AI 影音製作：從 prompt 產圖／產影片，進一步走到素材整理、粗剪、字幕、剪點評估與成片輸出。
  • Agent skill 設計：video-use 是一個完整的 skill 化任務包，結合安裝、工具、規則、helpers 與使用流程。
  • 課程與工作坊案例：非常適合用來講「AI 不必取代專業工具，而是把專業流程封裝成 agent 可執行的工作流」。

使用限制與提醒

  • video-use 仍依賴 ffmpeg、轉寫服務、影片素材品質與 prompt / skill 規則，不能保證一次產出商業級剪輯。
  • 對於高度風格化、節奏感或品牌審美要求很高的影片，仍需要人類剪輯師校對。
  • 若使用 ElevenLabs Scribe 或其他雲端轉寫，需注意影片內容是否包含未公開、客戶或個資資料。
  • 對 Hermes Agent、Codex、Claude Code 等工具而言，關鍵不是「會不會剪片」，而是工作資料夾、輸出目錄、權限、成本、錯誤回復與人工驗收是否設計清楚。
