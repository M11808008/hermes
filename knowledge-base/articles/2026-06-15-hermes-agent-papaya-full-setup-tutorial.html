對欒帥最有價值的導入路線

| 階段 | 影片主題 | 欒帥可用法 | 注意事項 |

|---|---|---|---|

| 1. 入門安裝 | 本機 + VPS | 可作為課程第一段：先讓學員理解本機私人助理與雲端常駐助理的差異 | 本機適合檔案與桌面任務；VPS 適合 24/7 Gateway / Cron，但要處理防火牆與備份 |

| 2. 訊息入口 | Telegram 串接 | 可對照欒帥目前 LINE / Telegram Hermes 工作流，說明同一個 agent 可跨平台服務 | 平台 token、webhook/public URL、home channel 需分開驗證 |

| 3. 長期上下文 | 記憶系統 | 適合展示「私人助理」不是單次聊天，而是保留偏好、環境與流程知識 | 記憶只放穩定事實；任務進度與短期狀態不應塞入長期記憶 |

| 4. 模型成本控管 | OpenRouter | 可讓學員理解多模型 provider、備援與成本/品質取捨 | API key 應放環境變數或 Hermes 設定，不貼進公開文件 |

| 5. 自動化擴展 | Subagent + Cron | 可做成「AI 學習雷達」示範：子代理並行研究、排程每日摘要 | Cron prompt 必須自包含，不能依賴當前聊天上下文 |

| 6. 辦公整合 | Google Workspace / Notion | 適合課程展示 Gmail、Calendar、Docs、Notion 知識庫串接 | OAuth 權限與資料隱私要清楚切分；含個資資料不要公開部署 |

| 7. 多模態輸出 | 生圖與影片 | 可補強教材、社群圖卡、課程宣傳素材生成流程 | 生成內容仍需人工審稿與來源標示 |

| 8. 維運安全 | 備份與防火牆 | 適合把 Hermes 從玩具專案提升為可長期運作的個人 AI OS | VPS 必須限制開放埠、保存 config / skills / sessions / .env 備份 |

可轉成課程或工作坊的設計

  • 第一單元：Hermes Agent 是什麼？和 ChatGPT、Claude Desktop、OpenClaw、n8n 的差異。
  • 第二單元：本機安裝與模型設定，先跑通基本聊天與工具調用。
  • 第三單元：串接 Telegram / LINE，讓 Agent 進入日常通訊入口。
  • 第四單元：建立記憶與 Skills，把個人偏好與 SOP 轉成可複用能力。
  • 第五單元：用 Subagent 做平行研究，用 Cron 做每日自動摘要。
  • 第六單元：串接 Google Workspace / Notion，完成一個實際辦公流程。
  • 第七單元：部署到 VPS，加入備份、防火牆與安全檢查。
