NotebookLM 升級為代理式研究夥伴｜Gemini 3.5、Antigravity、雲端電腦與多格式輸出

NotebookLM 升級為代理式研究夥伴｜Gemini 3.5、Antigravity、雲端電腦與多格式輸出

來源：欒帥分享 Prompt Case 的 Threads 貼文。原貼重點是 NotebookLM 不再只是「丟 PDF 問問題」，而是開始轉向「帶安全雲端電腦的研究型 AI 夥伴」。本文以 Google 官方公告為主，並用 CNET、Ars Technica 報導交叉核對可用範圍與限制。

一句話摘要

NotebookLM 正在從來源問答工具升級成代理式研究工作台：底層導入 Gemini 3.5 與 Antigravity，每本 Notebook 配備可寫碼、跑分析的安全雲端電腦，並能把來源脈絡輸出成 PDF、DOCX、Markdown、圖表、圖片、CSV、JSON、XLSX、PPTX 等可編輯成果；但首波主要開放給 Google AI Ultra 與 Workspace 企業進階用戶。

這次更新真正重要的地方

| 面向 | 更新內容 | 對工作流的意義 |

|---|---|---|

| 推理模型 | NotebookLM 升級到 Gemini 3.5，並結合 Antigravity | 更適合長文件分析、跨來源推理、研究規劃與複雜任務 |

| 可見思考與可信度 | Google 宣稱能提供更準確、可靠的資訊與更清楚的 reasoning / output visibility | 對研究、教學、顧問提案來說，比單純摘要更容易審稿與追溯 |

| 安全雲端電腦 | 每本 Notebook 配備 secure cloud computer，可寫與執行程式碼 | 可做資料清理、轉檔、統計、圖表、報告生成，不必一直跳回本機工具 |

| 100+ software skills | 官方稱內建超過 100 個 curated software skills | NotebookLM 從「讀資料」往「操作資料、產生成果」前進 |

| 多格式輸出 | PDF、DOCX、Markdown、TXT、PNG、SVG、JPG、GIF、CSV、JSON、XLSX、PPTX | 可直接接到簡報、表格、報告、知識庫與自動化流程 |

| 來源發現 | 可在 chat 中用 Google Search 找高品質來源，並由使用者選擇加入 Notebook | 從「你先準備好資料」變成「一起長出研究資料庫」 |

官方公告核對

Google 官方在 2026-06-08 的 The Keyword 公告中寫明：

  • NotebookLM 正在加入新的 agentic capabilities 與 advanced reasoning，用於複雜研究專案。
  • Chat 體驗升級為 Gemini 3.5 + Antigravity，目標是更可靠並讓思考過程更可見。
  • 每本 Notebook 現在有 secure cloud computer，可以協助寫程式與執行程式，用於更深的研究與分析。
  • 系統內含 100+ curated software skills。
  • 新增輸出格式包含資料視覺化、文件、Nano Banana 圖片、結構化資料、Excel、PowerPoint。
  • 可以從鬆散想法開始，讓 NotebookLM 協助用 Google Search 找來源；使用者仍可控制哪些來源加入 Notebook，並維持來源 attribution。
  • 首波全球網頁版 rollout 對象是 Google AI Ultra 用戶，以及 Workspace business customers with AI Ultra Access / AI Expanded Access；其他帳號會逐步擴大。

對欒帥的實用解讀

1. NotebookLM 會變成「研究型 Agent 的前處理與交付層」

過去 NotebookLM 最強的是 grounded RAG：把指定來源丟進去，再問摘要、比較、產生 audio overview。這次如果 cloud computer 與多格式輸出穩定，NotebookLM 就更像一個研究工作台：

  • 蒐集來源：用自己上傳的 PDF / 網頁 / Docs，再讓 NotebookLM 補找相關來源。
  • 結構化理解：比較多份文件、抓差異、建立研究架構。
  • 工具執行：在 sandbox 裡清理資料、跑分析、產圖表。
  • 成果交付：輸出 PDF 報告、PPTX、Excel、Markdown、JSON。
  • 外部接軌：再交給 Codex、Hermes、GitHub Pages、Google Workspace 或內部 SOP 系統。

2. 對課程、顧問案、知識庫尤其有價值

| 場景 | 可用法 |

|---|---|

| AI 課程備課 | 丟官方文件、論文、範例程式，請 NotebookLM 生成講義、測驗、工作坊步驟與 PPTX 草稿 |

| 研究報告 | 匯入多篇文章、政策文件與資料表，讓它做資料清理、圖表與 PDF 報告 |

| 企業顧問 | 把客戶規格、訪談紀錄、流程文件轉成簡報、roadmap、FAQ、SOP |

| 知識庫維護 | 輸出 Markdown / JSON 後再交給 Hermes 進一步整理、發布與索引 |

| 教育行政 | 把課程資料、報名資料、回饋表轉成摘要、表格與可下載文件；注意個資不可公開 |

3. 這不是「一般帳號已全量可用」

需要注意 rollout 範圍：Google 官方與 Ars Technica 都指出首波是 Google AI Ultra 與 Workspace 企業進階用戶。一般免費帳號、Pro 帳號或台灣帳號不一定立即看到完整功能。若欒帥要測，建議先檢查 NotebookLM web 介面是否出現：

  • Chat 是否標示 Gemini 3.5 或新 reasoning engine。
  • Studio Panel 是否可下載 DOCX / PPTX / XLSX / CSV / JSON。
  • 是否有「找來源 / source discovery / Google Search」流程。
  • 是否能請它寫 code、執行分析、產出圖表。
  • 是否有 Workspace / AI Ultra 權限提示。

建議測試流程

  • 建一個新的 Notebook，主題可用「AI 課程設計」或「知識庫自動化」。
  • 加入 3 類來源：官方文件、社群案例、自己的課程大綱或 Markdown note。
  • 試問：請建立研究架構、列出缺口、建議還需要哪些來源。
  • 若介面支援 source discovery，讓它找 5 個高可信來源，手動挑選加入。
  • 丟一份 CSV 或表格資料，要求清理、分析、畫圖表。
  • 要求輸出同一份成果為 Markdown、PDF、PPTX、XLSX 或 JSON。
  • 把 Markdown / JSON 交給 Hermes，測試是否能直接進 AI 學習雷達或課程網站。

判讀限制

  • Threads 貼文是社群轉述；正式功能、名稱、開放資格以 Google 官方公告與帳號實測為準。
  • 「Gemini 3.5 + Antigravity」與「secure cloud computer」代表能力方向，但實際可執行的程式、套件、網路權限、檔案大小與安全限制仍需實測。
  • Google 宣稱新版在核心評測平均 win rate 超過 65%，其中 large document analysis 69.9%、advanced web research and source discovery 78.2%；這是 Google 自家評測，不能直接等同外部 benchmark。
  • 首波功能對 AI Ultra / Workspace 企業進階用戶較友善；台灣與一般帳號可能需要等待。
