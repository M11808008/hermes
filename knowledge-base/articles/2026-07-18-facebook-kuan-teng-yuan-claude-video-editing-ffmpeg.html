用 Claude 剪影片不用背剪輯指令？可行的核心是「讓 AI 產生 FFmpeg 工作流」

來源：關登元的 Facebook Reel，縮圖文字為「用Claude剪影片／根本不用學」，貼文說明「你們最想要看的 Claude 影片教學來了」。本文將其歸檔為「用 Claude / Claude Code 協助影片剪輯」的工作流案例，而不是把 Claude 解讀成完整取代 Premiere、Final Cut 或 CapCut 的視覺剪輯軟體。

一句話摘要

Claude 很適合把「我要裁切、轉檔、壓縮、抽音訊、加字幕、合併片段」這類需求轉成 FFmpeg 或腳本流程；若搭配 Claude Code 的本機檔案與 Bash 執行能力，確實能降低學習剪輯指令的門檻，但它仍需要明確素材、可驗證輸出與人工審看，不能保證自動做出高品質視覺剪輯。

貼文主張與查證結果

|---|---|---|

較穩定的實作流程

  • 準備素材資料夾，檔名避免空白與特殊字元，先備份原始影片。
  • 告訴 Claude 目標：輸出比例、長度、平台、字幕語言、是否保留原音、是否壓縮。
  • 讓 Claude 先產生 FFmpeg plan，不要一開始就覆蓋原檔。
  • 用 Claude Code / terminal 執行指令，輸出到 `exports/` 或 `drafts/`。
  • 抽幀或播放檢查；若不滿意，回饋「第幾秒到第幾秒」再迭代。
  • 最後再做壓縮、封面、字幕與檔名規範。

欒帥可用場景

  • 教學：把它包裝成「AI 不是替你變成剪輯師，而是替你把自然語言轉成可執行剪輯流程」。
  • 課程素材：快速把長影片切成短影音、抽字幕、轉直式比例、壓縮成 LINE / 社群可傳格式。
  • 工作流示範：用同一份影片讓學員比較「手查 FFmpeg」與「Claude 產生腳本」的效率差。

注意事項

  • 影片、聲音與人像素材常涉及著作權與肖像權；教學示範應用自有或授權素材。
  • 若素材含學生、客戶或未公開內容，不要上傳到不確定資料政策的外部平台。
  • Claude 產生的 FFmpeg 指令可能會錯，第一次一定輸出到新檔並保留原檔。
  • 長影片處理成本高，建議先用低解析度或短片段試跑，再套用到完整素材。

Why it matters

這則 Reel 反映創作者工具的關鍵變化：AI 生產力不只在「生成內容」，也在「把複雜 CLI / 多媒體處理流程翻譯成可執行步驟」。對教學與內容團隊來說，Claude + FFmpeg 類工作流能把剪輯、轉檔、字幕與壓縮變成可複製 SOP。
