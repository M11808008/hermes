與神｜Windows-AI-Agent：用 Ollama + PowerShell 做本地 Windows 電腦控制 Agent

Threads 作者「與神」分享一個自製的 Windows 本地 AI Agent：`Windows-AI-Agent`。貼文將它定位為 OpenClaw 之外的輕量替代：不連外網、不消耗雲端 token，透過本機 Ollama 模型推理，讓使用者輸入中文指令後，由工具調用 PowerShell 操作 Windows。

一句話摘要

Windows-AI-Agent 是一個「本地 Ollama + PowerShell」的 Windows 電腦控制小工具，重點不是聊天，而是把中文指令轉成可執行的系統操作；但也因此必須把安全審查放在功能體驗之前。

貼文重點

  • 作者認為 OpenClaw 好用，但成本消耗快，且可執行的系統操作有限。
  • 這個工具以 Python 寫成，已打包成 Windows `.exe`。
  • 貼文稱壓縮後大小約 11MB。
  • 不需連外網、不消耗 API token。
  • 使用門檻：Windows 10 以上，以及本機 Ollama。
  • 使用方式：不能像聊天機器人長談，但可輸入一句中文指令，讓它推理並調用 PowerShell 執行。
  • 作者提供 GitHub repo，並表示採 GPL v3 授權，鼓勵自行下載拆解檢查。

README 宣稱功能

README 將它稱為「Windows AI 代理」，主張：

  • 可用日常對話叫它更改電腦設定、開啟指定網頁，或用瀏覽器搜尋資料。
  • 完全在本地端運行，不統計、不分析、不洩漏使用者資料。
  • 軟體大小約 11MB。
  • 背景記憶體占用約 25–30MB。
  • 下載包內建中文版與國際英文版。
  • 使用前需啟動 Ollama，並確認 `http://127.0.0.1:11434` 顯示 `Ollama is running`。
  • 執行時 README 建議以系統管理員身分執行。
  • 作者推薦搭配 Gemma 4 模型。

對欒帥的可用意義

這類工具值得收進 AI 學習雷達，因為它代表「低成本、本地化、中文指令、Windows 電腦控制」這條路線正在被個人開發者快速嘗試。它與 Claude Computer Use、OpenAI Operator、OpenClaw 或 Hermes Computer Use 的差異在於：

  • 不依賴雲端模型或商業 API。
  • 以 Ollama 本地模型做推理。
  • 直接落在 Windows / PowerShell 生態。
  • 技術門檻看似低，但安全門檻其實更高。

安全與實測建議

這個工具涉及 `.exe`、系統管理員權限與 PowerShell，因此不建議直接在主力電腦上裸跑。若要測試，建議：

  • 先閱讀 `Windows ai.py` 原始碼，不只看 README。
  • 在 Windows 虛擬機或可重置環境測試。
  • 不要先用系統管理員權限，除非已確認它需要的最小權限。
  • 觀察它如何把模型輸出轉成 PowerShell 指令，是否有 allowlist、確認機制與危險命令阻擋。
  • 禁止接觸含個資、金流、登入憑證與工作資料的資料夾。
  • 下載 zip / exe 後先做雜湊、病毒掃描與來源比對。
  • 若要教學展示，應把它定位為「本地 computer-use prototype」，而非成熟安全產品。

可延伸問題

  • 本地小模型是否能穩定產生安全的 PowerShell 指令？
  • 是否有 command approval / dry run / rollback？
  • 能否限制工具只執行白名單操作？
  • 是否能輸出可審計日誌，記錄使用者指令、模型推理與實際執行命令？
  • 若模型被 prompt injection 誘導，是否可能執行危險命令？

判讀限制

本文依據 Threads 貼文、GitHub API metadata 與 README 快照整理；尚未在 Windows VM 中實際執行，也未完成安全稽核。由於它會操作系統與 PowerShell，任何實測都應先以隔離環境與最小權限為前提。
