GordenSuperPPTSkills｜GPT 圖片簡報轉可編輯 PPTX 的 AI Skill 工作流

來源為 Govin.AI 在 Threads 分享 Gorden Sun 的開源簡報技能包 GordenSuperPPTSkills。貼文主張這套 skill 可以先用 GPT 生成圖片格式的豪華 PPT，再轉成完全可編輯的 PPTX，解決「AI 圖片簡報很好看但不能改」的實際痛點。本文以 Threads 貼文與 GitHub repo / README 為依據，整理其架構、使用方式、限制與欒帥可採用的實務路線。

一句話摘要

GordenSuperPPTSkills 的核心不是傳統模板套版，而是「先讓 GPT 產生高美感圖片投影片，再用視覺解析把背景、骨架、圖標/裝飾、文字按座標重建成可編輯 PPTX」；它很適合追求設計感與後續可修改性的簡報流程，但目前 README 明確標示僅限 Codex 使用，且圖片轉可編輯 PPTX 會消耗較多 Plus / 推理額度。

三個 Skill 的分工

| Skill | 功能 | 輸入 → 輸出 | 適合情境 |

|---|---|---|---|

| GordenImagePPTGen | 生成圖片格式 PPT | 主題/內容 → 每頁 PNG + 圖片型 PPTX | 先追求視覺效果與豪華排版，不要求每個物件可編輯 |

| GordenImage2PPTX | 圖片轉可編輯 PPTX | 圖片 → 背景、骨架、圖標、文字四層的可編輯 PPTX | 已有圖片簡報、截圖或 AI 生成投影片，想要後續修改文字/元素 |

| GordenSuperPPTSkill | 串接前兩者 | 主題/內容 → 圖片型 PPT + 可編輯 PPTX | 想一鍵完成「先出圖、再重建可編輯檔」 |

工作流原理

README 描述的核心方法是使用 GPT 的生圖能力與視覺解析能力：

  • 先產生設計完成度較高的圖片版投影片。
  • 對圖片做視覺拆解，分離背景圖、框架圖、圖標/裝飾與文字。
  • 依座標在 PPTX 中重建元素。
  • 用 python-pptx / Pillow / numpy 等工具生成可編輯的 PowerPoint 文件。
  • 透過技能規則與中間過程圖修正細節。

這個方向的價值在於把 AI 圖像模型擅長的「整體視覺設計」和 PPTX 需要的「物件可編輯性」拆成兩段處理，而不是期待模型直接一次產出完美的 XML / PPTX 結構。

使用方式與限制

| 項目 | README 資訊 | 欒帥判讀 |

|---|---|---|

| 適用 runner | README 明確寫「僅限 Codex 使用」 | 欒帥已有 Codex CLI，理論上比 Claude-only 環境更適合測試 |

| 推薦模型 | GPT 5.5、推理強度中 | 應視目前 Codex 可用模型與額度調整 |

| 只生成圖片 PPT | 使用 GordenImagePPTGen | 適合 demo、封面、行銷簡報初稿 |

| 只轉可編輯 PPTX | 使用 GordenImage2PPTX | 適合把已完成圖像投影片轉成可修改版本 |

| 一鍵全流程 | 使用 GordenSuperPPTSkill | 方便但成本最高，也最需要人工檢查 |

| 額度成本 | README 提到轉 1 張圖片約耗費 Plus 訂閱 5 小時額度的 10% | 不適合大量頁數無腦轉；應先用 1-2 頁測品質 |

| 依賴安裝 | pip3 install python-pptx pillow numpy | 屬於低摩擦 Python 依賴，但輸出品質仍取決於模型與提示詞 |

欒帥可採用的測試路線

  • 先不要做完整 20 頁簡報；用 2-3 頁小型題目測 GordenSuperPPTSkill。
  • 題目要明確指定：繁體中文、16:9、專業高質感、資訊密度、避免簡體字。
  • 先看圖片版 PPT 是否真的符合設計標準，再決定是否花額度轉可編輯 PPTX。
  • 轉成 PPTX 後檢查三件事：文字是否可選取、圖形是否分層、是否能在 PowerPoint / Keynote 打開。
  • 若只是要高質感投影片而不需要改每個元素，可停在圖片版 PPT，避免額外轉檔成本。
  • 若要正式商用，依 README 要求保留 GitHub 出處或標記作者 @Gorden Sun，並確認 repo 後續是否補上正式 license。

適合 / 不適合場景

| 適合 | 不適合 |

|---|---|

| 高質感簡報初稿、課程投影片、產品提案視覺稿 | 一次大量產出且完全不願人工校稿的正式簡報 |

| 需要先用 AI 追求視覺，再回 PowerPoint 微調 | 公司有嚴格品牌模板與版式規範，不能偏離版型 |

| 把圖片簡報、截圖、AI 投影片轉成可編輯檔 | 機密簡報；因可能需要把圖片交給外部模型視覺解析 |

| AI 簡報工具研究、agent skill 測試 | 需要穩定可預測排版的法務/財務/投資文件 |

與傳統 AI PPT 工具的差異

多數 AI PPT 工具偏向「大綱 → 模板頁面 → PPTX」，版面常有模板感；這個 repo 則走「圖片生成 → 視覺還原 → PPTX」路線，因此更可能做出複雜視覺效果。不過代價是可控性、成本與錯誤修正難度較高：圖片轉物件時可能出現座標偏移、字體不一致、圖標不可完全還原、中文文字辨識錯誤等問題。

判讀限制

Threads 貼文稱效果「幾乎完美」，但實際品質會受模型、頁面複雜度、中文/英文、字體、圖標、圖片解析度與額度限制影響。GitHub API 擷取時 license 欄位為 null，雖 README 寫可商用並要求標明出處，但正式商業使用前仍建議確認 repo 最新授權狀態。另 README 指出目前僅限 Codex，其他 agent runner 可能需要自行適配 image generation 與 vision 能力。
