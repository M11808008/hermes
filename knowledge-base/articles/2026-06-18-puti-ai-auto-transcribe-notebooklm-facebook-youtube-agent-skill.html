Puti-AI Auto Transcribe NotebookLM｜把 FB / YouTube 影片自動轉成 NotebookLM 知識庫的 Agent 技能包

來源為黃朝榮老師在 GEG Taiwan 社團分享的 Facebook 貼文。貼文介紹一套「Puti-AI 自動轉寫與同步工具」：讓使用者把 Facebook Reels、YouTube、Instagram 等影音連結交給 AI Agent，由工具下載影片、用 Gemini 2.5 做視聽雙模態轉寫與書面化整理，再同步到 NotebookLM。本文以 Facebook 貼文作為發現來源，並以 GitHub repo / README 作為 canonical source。

一句話摘要

Puti-AI Auto Transcribe NotebookLM 是一個 Python + yt-dlp + Gemini 2.5 Flash + NotebookLM MCP CLI 的開源工作流，目標是把「下載影片、轉寫、去口語、整理成 Markdown、匯入 NotebookLM」這串複製貼上地獄，包成 AI Agent 可自動執行的技能包。

它解決的三個痛點

| 痛點 | 原本問題 | 這個工具的做法 |

|---|---|---|

| AI 只聽不看 | NotebookLM / 一般轉寫流程常依賴音訊或 YouTube caption；遇到背景音樂、古文、生僻詞、畫面字幕時容易漏 | 先下載 MP4，讓 Gemini 2.5 同時看畫面字幕與聽人聲，做 OCR + ASR 對齊 |

| 平台登入阻擋 | Facebook Reels 等平台常阻擋無登入下載或抓取 | 以 yt-dlp 下載影音；並透過 agent / 本機環境處理登入與下載流程 |

| 手動複製貼上 | 下載、轉寫、整理、貼到 NotebookLM、再生成簡報很耗時 | `transcribe.py` 自動輸出 Markdown，若 `nlm` 可用則建立 NotebookLM 筆記本並匯入來源 |

工作流拆解

| 步驟 | 工具 / 檔案 | 說明 |

|---|---|---|

| 1. 接收影片 URL | Agent 指令或 CLI | 使用者輸入 `自動逐字稿：[影片網址]` 或執行 `python transcribe.py <url>` |

| 2. 下載 MP4 | `yt-dlp` | 抓取 Facebook / YouTube / Instagram 等影音串流並存成暫存影片 |

| 3. 多模態轉寫 | Gemini 2.5 Flash | 上傳影片到 Gemini Files API，要求模型結合畫面字幕 OCR 與人聲 ASR 產出繁中 Markdown 報告 |

| 4. 書面化整理 | Gemini / GPT-4o | 自動去除口語贅字，輸出章節化、重點加粗的書面知識報告 |

| 5. NotebookLM 同步 | `notebooklm-mcp-cli` / `nlm` | 檢查登入、建立新 notebook、匯入 Markdown 來源檔 |

| 6. 清理暫存 | `transcribe.py` | 完成後刪除本地 MP4 暫存檔，避免硬碟殘留 |

Agent 友善設計

這個 repo 特別有一份 `agent_instructions.md`，明確告訴 Cursor、Windsurf、Antigravity、Claude Code 等 AI 編程助理：

  • 自動執行 `pip install -r requirements.txt`。
  • 執行 `nlm login` 檢查 NotebookLM 登入。
  • 註冊 MCP server：server name 為 `notebooklm`，command 為 `nlm mcp`。
  • 執行 `nlm doctor` 驗證狀態。
  • 當使用者輸入 `自動逐字稿：[VIDEO_URL]` 時，直接背景執行 `python transcribe.py [VIDEO_URL]`。

這點很值得注意：它不是只有腳本，而是把「給 AI Agent 的操作規則」也放進 repo，讓工具更接近可被代理接手的技能包。

對欒帥的實用價值

  • **可作為 AI 教學自動化案例**：把影片學習、逐字稿、Markdown 筆記、NotebookLM 知識庫串成完整 pipeline，適合示範 agentic workflow。
  • **適合處理社群短影片知識化**：許多 Facebook Reels / YouTube Shorts 只有短影音，這套方法能把影片變成可搜尋、可問答、可生成簡報的資料來源。
  • **可和 AI 學習雷達互補**：Hermes 負責歸檔公開知識庫；Puti-AI 這類工具可負責把影片內容先變成結構化 Markdown，再進一步供 NotebookLM 或知識庫使用。
  • **可作為 NotebookLM 教學補強**：Google NotebookLM 官方已支援 YouTube / audio sources，但官方文件也說明 YouTube 來源依賴公開影片與 captions；這個工具嘗試用「下載完整 MP4 + Gemini 看畫面」補足 caption 品質不足與平台限制。
  • **可延伸為課程工作坊**：學生只給影片連結，Agent 自動轉寫與建庫，最後用 NotebookLM 生成摘要、FAQ、簡報或學習指南。

導入前注意事項

  • **Facebook / Instagram 下載穩定性不保證**：yt-dlp 對平台登入、防爬、影片格式的支援會變動；有些內容仍可能需要 cookie、登入或人工處理。
  • **NotebookLM 自動登入可能不一定無痛**：README 假設本機 Chrome / Google session 可協助登入，但不同系統、瀏覽器權限、Google 安全檢查都可能讓自動登入失敗。
  • **Gemini 上傳影片涉及資料外傳**：影片會上傳到 Gemini Files API 做處理；若內容包含個資、學生資料或非公開課程，需先確認授權與隱私。
  • **「辨識率近乎 100%」屬作者主張**：實際效果仍取決於影片音質、字幕清晰度、語言、背景音樂、模型限制與影片長度。
  • **授權尚未明確**：GitHub API 未顯示 license，正式商用或改作前需確認作者授權。
  • **NotebookLM 官方來源限制仍需遵守**：NotebookLM 對可匯入來源類型、YouTube captions、網頁擷取與來源數量有產品限制；此工具是把結果轉成 Markdown 後同步，不代表能繞過 NotebookLM 本身規則。

| 輸出檢查 | 檢查 Markdown 是否保留重點、是否過度改寫、是否漏掉畫面文字 |

| NotebookLM 匯入 | 確認新 notebook 名稱、來源檔、引用與後續問答是否正常 |
