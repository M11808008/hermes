NotebookLM × Claude Cowork｜把 YouTube 專業頻道轉成 AI 知識庫與內容產出系統

來源是 Daojie 的 Facebook Reel。貼文主張：「你不需要是行業專家，也能輸出專家級內容，這是 AI 給普通人最大的紅利。」操作被濃縮成三步：打開 NotebookLM、裝進 Claude、開始輸出。

這篇值得收進 AI 學習雷達，但需要把「站在專家肩膀上」轉成合規與高品質的工作流：不是搬運、洗稿或抄襲，而是用 NotebookLM 做研究與來源組織，再用 Claude / Claude Cowork 做重組、提問、草稿與格式化，最後由人補上自己的觀點、案例、查證與引用。

正確理解：不是「複製專家」，而是「建立研究型內容引擎」

Reel 的說法容易被誤解成「把別人的 YouTube 頻道丟進 AI，直接改寫成自己的內容」。更穩健的理解應該是：

```text

專業來源 → NotebookLM 建立來源庫 → Claude / Cowork 做提問與產出草稿 → 人類編輯加入觀點、案例、引用與責任

```

這裡的價值不是把別人的內容洗成自己的，而是：

  • 快速掌握一個領域的主題地圖。
  • 找出專家反覆提到的核心概念與問題框架。
  • 將多個來源的觀點整理成自己的知識結構。
  • 產出面向特定受眾的新解釋、新案例、新格式。
  • 保留來源引用與查證，避免錯誤轉述。

工具分工

| 工具 | 角色 | 適合做什麼 | 不適合做什麼 |

|---|---|---|---|

| NotebookLM | 來源型研究助理 | 放入文件、筆記、影片摘要、文章，基於來源問答、摘要、生成學習卡/簡報/資訊圖表等 | 取代事實查證、直接決定商業/法律/醫療建議 |

| Claude / Claude Cowork | 內容與任務執行代理 | 重新組織素材、定義受眾、寫大綱、草稿、腳本、社群貼文、簡報、表格；Cowork 可在桌面環境執行非程式任務 | 未經授權大量複製原作者內容，或把來源觀點偽裝成原創研究 |

| 人類創作者 | 判斷與責任層 | 選題、定位、觀點、案例、語氣、引用、合法使用、品牌一致性 | 完全放手讓 AI 決定專業結論 |

可複製工作流

Step 1：選一個足夠窄的內容賽道

不要從「法律」、「會計」、「導演」這種大類開始。更好的切法：

| 太大 | 更好的切法 |

|---|---|

| 法律 | 中小企業合約常見坑、創作者授權合約、AI 生成內容版權風險 |

| 會計 | 自由工作者報稅、SaaS 公司月結、餐飲店成本控管 |

| 電影導演 | 手機短片分鏡、訪談影片剪輯節奏、YouTube 開場 30 秒設計 |

| AI 創業 | 一人公司內容自動化、AI 顧問服務包裝、NotebookLM 研究流程 |

Step 2：建立 NotebookLM 來源庫

可放入：

  • YouTube 影片逐字稿或摘要。
  • 專家文章與白皮書。
  • 官方文件、FAQ、案例頁。
  • 自己的訪談筆記、讀書筆記、課程筆記。
  • 競品頁面與常見問題整理。

建議欄位：

| 欄位 | 說明 |

|---|---|

| source-title | 來源標題 |

| source-url | 原始連結 |

| author/channel | 作者或頻道 |

| topic | 主題 |

| key-claims | 來源主張 |

| examples | 案例 |

| useful-quotes | 可引用片段 |

| uncertainty | 需要查證的地方 |

Step 3：先問 NotebookLM，而不是直接叫 Claude 寫文章

先用 NotebookLM 做研究問答：

```text

請根據來源整理這個領域最常被提到的 10 個問題，並標註每個問題來自哪些來源。

```

```text

請把這些影片的共同知識結構整理成「初學者 → 進階者 → 專家」三層地圖。

```

```text

請列出來源中彼此矛盾或需要查證的說法。

```

```text

請找出適合做成短影音的 20 個選題，每個選題附上來源依據。

```

Step 4：交給 Claude / Cowork 轉成內容產品

把 NotebookLM 的來源型整理交給 Claude，再要求它根據受眾與平台重寫：

```text

你是一位內容策略顧問。以下是 NotebookLM 根據多個來源整理的研究筆記。

請不要逐字改寫來源內容，而是幫我產出一個面向「台灣中小企業主」的 10 集短影音企劃。

每集包含：標題、hook、3 個重點、案例、CTA、需引用的來源。

```

```text

請把這份研究筆記改寫成一篇 1200 字繁體中文文章。

要求：

  • 保留來源引用
  • 加入我的觀點：AI 工具只能降低研究門檻，不能替代專業責任
  • 不使用誇大語氣
  • 結尾提供讀者可執行清單

```

若使用 Claude Cowork，可讓它負責桌面任務：例如整理 Google Drive 文件、建立簡報草稿、生成 CSV 選題表、打包素材資料夾等。但需要給它明確資料權限與輸出格式。

品質控制：避免變成 AI 洗稿

這套流程最重要的不是工具，而是約束。

| 風險 | 解法 |

|---|---|

| 直接改寫原作者內容 | 要求 AI 只抽取結構、問題、框架，不逐句重寫 |

| 沒有專業背景卻做專業建議 | 加上「非法律/醫療/投資建議」與專家審稿流程 |

| 來源幻覺 | 每個核心主張都要回指來源，缺來源就標記為待查 |

| 內容同質化 | 加入自己的案例、受眾、立場、在地語境 |

| 版權風險 | 不下載重製影片、不搬運逐字稿長段落、不假冒原作者 |

| 品牌風格混亂 | 建立自己的語氣指南、禁用詞、觀點邊界 |

對欒帥的應用建議

這個 workflow 很適合轉成「AI 課程 / 顧問服務」的一個模組：

1. AI 學習雷達內容研究

可用來快速研究一個 AI 主題：

```text

官方文件 + YouTube 教學 + GitHub README + 社群討論 → NotebookLM → Claude → 知識庫文章

```

這其實和目前 AI 學習雷達的歸檔流程相似，只是把 NotebookLM 放進前段來源理解，把 Claude / Hermes 放在後段結構化與部署。

2. 企業內訓內容庫

企業可把：

  • 內部 SOP。
  • 教學影片逐字稿。
  • 客服 FAQ。
  • 產品文件。
  • 銷售案例。

整理進 NotebookLM，讓 Claude 轉成：新人訓練教材、銷售話術、FAQ、簡報、測驗題。

3. 個人品牌內容引擎

對一人公司 / 顧問 / 講師：

```text

專家來源研究 → 我的觀點 → 台灣案例 → 短影音 / 長文 / 課綱 / 簡報

```

這樣輸出的核心不是「我讀過某專家」，而是「我能把複雜知識轉成我的受眾能用的決策工具」。

建議 SOP

  • 選題：定義受眾、痛點、內容承諾。
  • 收集：找 5–10 個高品質來源，不要只靠單一 YouTube 頻道。
  • NotebookLM：建立來源庫，先問結構、矛盾、核心概念。
  • Claude：把研究筆記轉成不同格式內容。
  • 人工校稿：補觀點、案例、引用、風險聲明。
  • 發布：依平台改寫，不同平台不要同稿硬貼。
  • 回饋：收集留言問題，再放回 NotebookLM 形成下一輪選題。

判讀限制

  • Facebook Reel 本身是短影音宣傳，沒有完整螢幕操作細節；本文依可擷取 caption 與官方工具文件整理成可執行工作流。
  • 「裝進 Claude」不是官方 NotebookLM 與 Claude 的單一標準功能名稱；本文解讀為把 NotebookLM 產出的研究筆記 / 來源摘要 / 結構化輸出交給 Claude 或 Claude Cowork 繼續加工。
  • 對法律、醫療、會計、投資等專業領域，AI 只能協助研究與草稿，不能取代專業資格與責任。
  • 若使用 YouTube 內容，應遵守平台條款、版權與合理引用原則，不應大量複製或冒充原作者。
