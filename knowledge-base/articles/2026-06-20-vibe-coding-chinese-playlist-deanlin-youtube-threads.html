中文 Vibe Coding 系統課｜SDD、自動化測試、Agent Skills、Git Worktree 與 OpenSpec

來源是 欒帥分享的 Threads 回覆。林鼎淵在回覆「Vibe coding 有什麼推薦教學」時，整理了一套中文 YouTube 播放清單《Vibe Coding 教學》。這套資源值得歸檔，因為它不是只教「對 AI 說一句話生成網站」，而是把 Vibe Coding 往工程化方向推：規格、測試、skills、worktree、多代理協作與可持續迭代。

一句話摘要

這是一條從「零基礎用 AI 做出東西」走向「可維護 AI 協作開發」的學習路徑：先學 AI coding 工具，再用 SDD 寫清楚需求，用自動化測試補上品質底線，最後用 Agent Skills、Git Worktree、OpenSpec 與 Codex / Claude Code / Antigravity 類工具建立可重複的開發流程。

為什麼這比一般 Vibe Coding 教學重要

很多 Vibe Coding 教學停在「自然語言 → 生成網頁」。這很適合啟蒙，但也很容易留下四個問題：

  • **需求不穩**：一開始沒有 spec，後面每次改都像重做。
  • **品質不可驗證**：AI 產出看似能跑，但沒有測試、沒有 lint、沒有 edge cases。
  • **脈絡不可延續**：每次對話都重講規則，沒有 CLAUDE.md、skills、memory 或 project docs。
  • **協作不可控**：多個 agent 同時改同一份 repo，容易覆蓋、衝突或產生不可合併分支。

這套清單剛好補上這些缺口：SDD 提供需求骨架，自動化測試提供驗證，Agent Skills 提供可重用專家脈絡，Git Worktree 提供並行隔離，OpenSpec 提供持續變更依據。

給 欒帥的課程轉用建議

若要設計「非工程背景也能用 AI 做工具」的課，可以把這套資源拆成三層：

第一層：讓學員做出第一個東西

  • AI coding 工具差異與選擇。
  • 用中文描述需求。
  • 從個人頁、銷售頁、表單或小工具開始。
  • 快速部署到可分享網址。

第二層：讓東西不會一改就壞

  • 寫簡單 spec：目標、使用者、輸入、輸出、done criteria。
  • 要 AI 先產測試或檢查清單。
  • 每次修改前要求 agent 先讀 spec，再改 code。
  • 用 git commit 保留版本。

第三層：把成果變成可重複流程

  • 建立專案技能：設計規範、品牌語氣、部署方式、常見錯誤。
  • 用 Git Worktree 或分支讓多個方向平行探索。
  • 把成功 prompt / 工具鏈 / 驗證步驟寫成 skill。
  • 讓下一個專案直接複用，而不是從零開始。
