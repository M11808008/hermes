企業級 AI 客服系統：LINE + Supabase + React 開源方案

一句話摘要

scorpioliu0953 在 GitHub 開源了一個企業級 AI 客服系統，支援 LINE 接入、GPT-5 / Gemini 3 模型、PDF 知識庫上傳與真人客服轉接，已獲 8 顆星、40 次 Fork，可透過 Netlify 一鍵部署。

重點整理

  • 主題：企業級 AI 客服系統開源專案。
  • 技術棧：LINE + Supabase + React + Tailwind CSS + Netlify Functions。
  • AI 模型支援：

- OpenAI GPT-5（Responses API）

- Google Gemini 3（Thinking Level）

- 可在後台自由切換模型。

  • 知識庫功能：

- 上傳 PDF 或純文字，讓 AI 根據產品手冊回答客服問題。

- 支援 Storage 上傳與向量檢索概念（RLS 權限控制）。

  • LINE 真人轉接系統：

- 自動偵測關鍵字（如「真人」、「客服」、「人工」）。

- 發送 LINE 推送通知給真人客服專員。

- 支援逾時自動轉接（可設定分鐘數）。

  • 穩定性設計：

- 資料庫級事件 ID 去重機制，解決 LINE Webhook 重複發送的問題。

  • 後台介面：

- React + Tailwind CSS，深色模式與手機響應式。

- 可設定 AI 模型、System Prompt、溫度等參數。

  • 部署方式：

- 方案 A：Fork + 手動部署

- 方案 B：Netlify 一鍵自動部署

  • 42 次 Commit，代表有一定維護歷史。

系統架構

  • 前端：React + Tailwind CSS（Netlify 托管）
  • 後端：Netlify Functions（Node.js）
  • 資料庫：Supabase（PostgreSQL + RLS + Storage）
  • 訊息通道：LINE Messaging API
  • AI：OpenAI GPT-5 / Google Gemini 3

對 Luarn-Pin 的意義

這是一個完整的「聊天機器人 + AI + 真人轉接」實務範本，對以下情境很有價值：

  • **LINE AI 客服實作**：有完整的 LINE Webhook 接收、訊息發送、去重機制與真人轉接邏輯。
  • **Supabase 實戰**：包含 RLS 權限設定、Storage bucket 設定、資料庫 Schema 設計。
  • **AI + RAG 整合**：PDF 知識庫上傳與檢索，是將 AI 接入特定領域知識的經典模式。
  • **Netlify Functions 無伺服器後端**：學習如何用 Functions 處理 LINE Webhook、呼叫 AI API、管理資料庫。
  • **Hermes / AI 工作流參考**：可借鏡其 RAG 設計、Human-in-the-loop 轉接機制作為課程案例。
