對 AI 學習雷達的價值

1. Finance Agent Skill 化的實例

這份清單把金融工作拆成明確任務模組，而不是只用單一「金融分析 agent」包山包海。這很符合 Agent Skills 的設計精神：每個 skill 對應一個高頻、可重複、可驗證的任務。

2. 股票研究最適合做成 KOL 工作流

若目標是金融內容、股票研究或投資型自媒體，最值得優先試的是：

  • `earnings-analysis`：財報後快速更新觀點。
  • `earnings-preview`：財報前建立預期與風險情境。
  • `morning-note`：每天產生市場觀察與交易想法。
  • `sector-overview`：把產業趨勢整理成主題報告。
  • `thesis-tracker`：維護每檔股票的核心投資論點與反證條件。
  • `catalyst-calendar`：追蹤財報、產品發表、法說、政策、併購等事件。
  • `screen` / `idea-generation`：作為選題與投資想法來源。

3. Financial Analysis 可作為內容可信度底座

金融 KOL 若只做新聞摘要，很容易流於轉述。若搭配 Financial Analysis skills，內容可以更像「研究報告」：

  • `comps-analysis`：同業比較與估值區間。
  • `dcf-model`：折現現金流估值。
  • `3-statement-model`：三表連動財務模型。
  • `competitive-analysis`：競爭格局與護城河分析。
  • `clean-data-xls` / `audit-xls`：確保資料表乾淨與可追溯。
  • `pptx-author` / `xlsx-author`：輸出簡報與模型檔。

4. 可以設計成「含股 KOL Agent Stack」

根據截圖內容，可整理出一個最小可用工作流：

  • 用 `screen` 或 `idea-generation` 找潛在標的。
  • 用 `sector-overview` 建立產業脈絡。
  • 用 `comps-analysis` 與 `dcf-model` 做估值基礎。
  • 用 `thesis-tracker` 寫出投資論點、風險與反證條件。
  • 用 `catalyst-calendar` 追蹤未來催化事件。
  • 每天用 `morning-note` 產出市場觀察。
  • 財報前用 `earnings-preview`，財報後用 `earnings-analysis` 更新內容。

注意事項

  • 這些 skills 若要用於投資內容，應避免直接構成個別投資建議。
  • 若處理個股與財報資料，需要標示資料來源、時間戳與假設條件。
  • 估值模型輸出必須保留敏感度分析，避免單一目標價造成過度確定感。
  • 若輸出給公開社群，需加入風險揭露與非投資建議聲明。
  • KYC、基金行政或客戶資料處理需避免上傳個資與未脫敏文件。

可延伸方向

這份清單值得後續追蹤原始 GitHub repo 或 skill marketplace 來源。如果能找到完整 `SKILL.md`，可進一步評估：

  • 每個 skill 是否有明確輸入／輸出格式。
  • 是否包含資料來源與引用要求。
  • 是否有模型、估值、Excel、PPT 相關工具鏈。
  • 是否支援 Claude Code、Hermes Agent、Codex 或其他 agent skill 標準。
  • 是否能改造成中文台股／美股 KOL 研究流程。

結論

這兩張圖的價值不只是列出 55 個 finance skills，而是展示「金融知識工作如何被拆成可重複執行的 AI 技能」。對含股 KOL、投研助理、金融教育內容或企業內部研究流程來說，最值得先試的是 Equity Research 與 Financial Analysis 兩組，再逐步延伸到投行、私募、財富管理與基金行政場景。
