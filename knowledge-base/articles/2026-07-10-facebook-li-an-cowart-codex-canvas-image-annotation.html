李安 Facebook Reel｜Cowart：把 Codex 變成可標註圖片的無限畫布工作流

一句話摘要

Cowart 的價值是把「請把圖片左上角那塊改掉」這種難以用文字精準描述的需求，轉成 Codex 看得懂的畫布標註：直接在圖片上畫箭頭、圈選與註記，再讓 Codex 生成乾淨修正版。

官方 / 原始碼驗證：Cowart 是什麼

Cowart GitHub README 說明：Cowart 是一個面向 Codex 的 native infinite-canvas widget plugin，基於 tldraw 提供可視化畫布，用於：

  • 視覺構思與草圖整理。
  • 在 Codex 中打開 tldraw 無限畫布 widget。
  • 把畫布頁面與圖片資源保存到目前使用者專案的 `canvas/` 目錄。
  • 在畫布中建立 AI 圖片框，輸入 prompt、選擇參考圖，讓 Codex 依選中框的比例生成圖片。
  • 對圖片做箭頭、圈選、文字標註後，將標註截圖送給 Codex，生成去掉標註痕跡的乾淨修正版，並放在原圖旁邊。
  • 透過 Cowart MCP tools 讀取選取狀態、保存畫布、插入圖片與管理頁面本地資源。

換句話說，Cowart 不是單純的圖片生成器，而是把「畫布狀態、圖片版本、視覺標註、Codex 生成」整合在同一個專案工作區。

安裝與使用方式判讀

Cowart README 提供兩種安裝方式：

  • 讓 Codex 自動從 `https://github.com/zhongerxin/cowart.git` 安裝。
  • 手動 clone 到 `~/plugins/cowart`，執行 `npm install` / `npm run build`，再加入 Codex personal marketplace。

README 中的典型命令包含：

  • `codex plugin marketplace add ~`
  • `codex plugin add cowart@personal`

安裝後通常需要開啟新的 Codex 對話，讓新的 skills 與 MCP tools 完整載入。

可複用工作流

圖片標註改版流程

  • 在 Codex 中打開 Cowart canvas。
  • 將原圖放進畫布。
  • 在圖上用箭頭、圈選、框線、文字說明修改需求。
  • 選中被標註的圖片，觸發「按標註修改」或等效流程。
  • Cowart 匯出包含原圖與標註的截圖，交給 Codex。
  • Codex 依標註生成乾淨修正版。
  • 新圖放在原圖旁邊，保留 before / after 與標註脈絡。
  • 若結果不準，再在新版本上繼續標註迭代。

AI 圖片生成流程

  • 在畫布建立 AI 圖片框。
  • 輸入 prompt 或指定參考圖。
  • Codex 讀取圖片框尺寸與比例。
  • 生成圖片並替換該圖片框。
  • 繼續以畫布標註方式迭代。

為什麼這件事重要

圖片修改最大的問題常常不是模型不會改，而是使用者很難用文字準確描述空間關係。例如：

  • 「把右上角那個亮點拿掉」
  • 「這個人物的手不要擋住產品」
  • 「保留背景，只換中間標題」
  • 「箭頭指的地方加一點陰影」

純文字 prompt 需要反覆補充位置與語義；Cowart 這類畫布工作流則把空間資訊直接留在圖上。這讓 Codex 不只讀文字，也能讀「你畫在哪裡」。

適合場景

  • 社群貼圖、封面、廣告圖的局部修改。
  • 課程投影片、教學圖解、流程圖的視覺改版。
  • 網頁 mockup / UI 截圖的標註式修改。
  • AI 生成圖的多版本對比與迭代。
  • 設計師與非設計師協作時，用畫面標註取代冗長文字描述。
  • 將圖片修改需求保存為專案內可追蹤的 before / after 歷史。

限制與風險

  • Cowart 是第三方 Codex plugin，不是 OpenAI 官方內建功能；導入前要檢查 repo、權限與維護狀態。
  • 需要 Codex plugin / MCP widget 相關環境可用；不同地區、帳號或版本可能不完全支援。
  • 圖片生成結果仍可能不穩定，尤其是文字、人物手部、品牌 logo、精準排版。
  • 若圖片含個資、客戶資料、未授權肖像或商業機密，不應直接丟給外部模型處理。
  • 外掛會在專案目錄保存 `canvas/` 資料與圖片資源，公開 repo 前要檢查是否含不該公開的素材。

對 AI 學習雷達的判讀

這支 Reel 延續近期「Codex 不只寫程式，而是變成可插外掛的工作台」的趨勢。Cowart 代表一種很實用的 Agent UI 方向：

  • 文字 prompt 負責意圖與規則。
  • 畫布負責空間與視覺脈絡。
  • Codex / MCP tools 負責把畫布狀態轉成可執行操作。
  • 專案資料夾保存資產與版本，而不是散落在聊天紀錄裡。
