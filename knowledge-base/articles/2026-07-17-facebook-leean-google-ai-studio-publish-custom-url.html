Facebook Reel｜Google AI Studio 可直接發布網站並設定 ai.studio 自訂網址

一句話摘要

李安的 Facebook Reel 指出 Google AI Studio 現在可將 vibe coding 做出的網頁直接發布到網路上，並設定自訂網域 / 網址；Google 官方文件確認 AI Studio Build mode 可快速建置與部署 app，部署會對應到 Cloud Run，且可設定 `your-app-name.ai.studio` 這類自訂子網域。

官方文件交叉驗證

Brave Search 找到的 Google 官方來源確認：

  • Google AI Studio Build mode 文件說明，可快速 build / vibe code apps，並測試 Gemini 最新能力。
  • AI Studio 支援建置 web apps、full-stack runtimes、Firebase Firestore / Authentication、Google Workspace integrations 等功能。
  • Build mode 中的 app 可分享、展示，準備好後可部署到 Cloud Run。
  • Google AI Studio Deploying 文件說明，每次 AI Studio deployment 會建立對應的 Cloud Run service。
  • 發布時可設定 `https://your-app-name.ai.studio` 這類 memorable subdomain；subdomain 全域唯一、先到先得；若取消發布或刪除 app，該 URL 會釋放。

---

需要精準表述的地方

Reel 的「自訂網域」可理解為「可自訂發布網址」，但官方文件目前明確描述的是 AI Studio 發布時可設定 `ai.studio` 底下的自訂子網域，例如：

`https://your-app-name.ai.studio`

若要綁定自己購買的裸網域或公司網域，通常會牽涉 Cloud Run / DNS / Google Cloud domain mapping 等額外設定，與 AI Studio 內建的 `ai.studio` custom URL 不是同一層能力。

---

對 AI 學習雷達的保存價值

這則內容值得保存，因為它把 Google AI Studio 從「prompt / demo 工具」推向「快速做出可展示網頁 app」：

  • 教學 demo：老師可以用自然語言快速做出互動網頁，再用公開網址讓學員打開。
  • MVP 原型：產品或行銷團隊可快速把概念變成可點擊、可分享的頁面。
  • AI app builder：AI Studio 同時支援 Gemini、Full-stack runtime、Firebase、Workspace integrations，逐步接近低摩擦的 AI app 工作台。
  • 部署門檻降低：不必先理解 Vercel、Netlify、Docker 或 Cloud Run 細節，也能把初版 app 發布到網路上。

---

導入注意事項

  • Cloud Run 可能產生成本；Starter Tier 與免費部署有數量、帳號資格與區域限制。
  • 分享 / 發布 app 時，API calls 可能計入使用者配額；若使用付費模型，成本可能增加。
  • `ai.studio` 子網域是全域唯一、先到先得；刪除或取消發布後會釋放，不能當成永久品牌資產。
  • 若 app 會連接 Gmail、Drive、Sheets、Calendar 或其他 Workspace API，要檢查 OAuth 權限、資料存取範圍與隱私告知。
  • Vibe coding 產物仍需做安全檢查：前端是否暴露 API key、後端權限是否過大、資料庫規則是否安全。
