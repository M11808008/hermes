可複用的 7 個 Claude 工作流模組

1. Projects：把「重複 prompt」變成任務容器

影片中的 SEO writing project 示範一個重要觀念：如果任務會重複，就不要每次重新貼 prompt。更好的結構是：

  • instructions：固定任務規則，例如文章格式、語氣、CTA、SEO 寫法。
  • files：放品牌資料、範例文章、產品頁、舊稿、關鍵字清單。
  • memory / preferences：持續修正偏好，讓輸出越來越像自己的工作方式。
  • bookmark：把常用 project 直接加入瀏覽器書籤，降低啟動成本。

對欒帥而言，這可轉成「課程頁生成器」「活動文案生成器」「LINE 分享歸檔器」「AI 工具評測模板」等固定 Project。

2. Artifacts：把輸出從文字升級成可檢查的介面

影片展示用 Claude 產生 landing page、Pomodoro timer、SEO content writer。這類輸出要用 Artifact 思維看待：

| 用途 | 不是只要 Claude 寫什麼 | 應該檢查什麼 |

|---|---|---|

| Landing page | 寫 HTML | CTA 是否一致、版面是否可信、行動版是否可讀 |

| 小工具 | 寫 JavaScript | 狀態是否保存、錯誤處理是否清楚、是否可部署 |

| 內容生成器 | 寫 prompt UI | 輸入欄位是否足夠、輸出是否可複製、是否需要 API key |

| Tracker | 做表格 | 欄位是否對應 input / output、是否有公式與週期 |

重點是讓 Claude 產出「可被看見、可被點、可被測」的 artifact，而不是只產出漂亮文字。

3. Skills：把成功方法封裝成可攜帶能力

影片中最有價值的段落之一，是把 YouTube title workflow 轉成 Claude Skill。這與欒帥近期關注的 Codex / Claude Skills 很接近：

  • 先在一般聊天中跑出一個有效流程。
  • 請 Claude 將該流程整理成 skill instruction。
  • 檢查 `SKILL.md` 是否包含適用情境、輸入、步驟、反例與輸出格式。
  • 匯入 Claude 或放到 Claude Code / Hermes / Codex 類工具可讀的位置。
  • 後續用一句「use the YouTube title skill」呼叫。

治理重點：Skill 不應只是 prompt 收藏，而要有版本、範例、限制與測試方式；否則很容易變成一堆不可維護的咒語。

4. Claude Code：把「不想手動做的技術步驟」交給 agent

影片用 Claude Code 開 GitHub repo、處理 Docker、本機專案與 Antigravity 產生的 app。這類用法適合：

  • clone / install / run GitHub 專案。
  • 讀 README 並執行 setup。
  • 修 bug、開 local server、跑測試。
  • 改善現有 HTML / CSS / JS。
  • 把一次性 prototype 變成較可維護的 repo。

但影片也提醒要注意權限：Claude Code 會在本機執行命令，不能無腦 yes；應先看它要做什麼、是否會修改檔案、是否需要 token、是否會跑危險命令。

5. Connectors / MCP：讓 Claude 接觸真資料與真工具

影片提到 Google Drive、Calendar、Gmail、GitHub、Zapier、PayPal 與 MCP。這部分的實作價值在於：

  • 將檔案、會議、郵件、repo、表單、CRM 等資料帶進 Claude。
  • 讓 Claude 不是「猜」，而是根據連接的資料工作。
  • 讓工具鏈從單次回覆變成可編排流程。

對課程或企業導入而言，這裡要加上權限邊界：誰能讀取什麼資料、是否可寫入、是否需人工審核、log 是否保存。

6. Computer Use / Browser：跨網站操作必須加 review gate

影片示範 browser/computer-use 類能力可協助 email、購物、WordPress 發文。這種能力很強，但更需要人類審核：

  • 發文前一定要人工確認內容與帳號。
  • 涉及付款、寄信、刪除、公開發布時要二次確認。
  • 不要把登入憑證或付款流程完全交給自動化。
  • 適合做「草稿、填表、整理、預覽」，不適合無監督執行高風險動作。

7. Tracker / Spreadsheet：把目標拆成 input / output

影片最後用 Claude 建 30 天 hiring tracker。這個模式很值得拿來教：

  • 先問「想要的 output 是什麼？」例如應徵者、銷售電話、報名人數、文章流量。
  • 再拆「每日 input 是什麼？」例如發文、外展、內容產出、聯絡候選人。
  • 用 Claude 生成表格、日期、公式、gap 欄位。
  • 每週檢查哪些 input 真的推動 output。

這比單純要求 Claude「給我一個計畫」更可執行，因為它把計畫轉成可追蹤的儀表板。

建議 Prompt 模板

將任務升級成 Project

```text

我想把以下重複任務做成 Claude Project。

任務：{任務名稱}

使用者：{誰會用}

輸入資料：{每次會提供什麼}

輸出格式：{想要什麼結果}

固定規則：{品牌語氣、限制、CTA、審核標準}

請幫我整理：

  • Project instructions
  • 建議上傳的 files / examples
  • 每次使用時的最短啟動 prompt
  • 品質檢查清單

```

將成功聊天轉成 Skill

```text

請把我們剛剛完成的流程整理成一個可重複使用的 Claude / agent skill。

請輸出 SKILL.md 結構，包含：

  • name / description
  • when to use
  • inputs
  • step-by-step workflow
  • output format
  • examples
  • pitfalls / do not do
  • verification checklist

```

將計畫轉成 input / output tracker

```text

目標：{30 天內要達成的結果}

請不要只給建議，請幫我做一個 tracker 設計：

  • 每日 input 指標
  • 每週 output 指標
  • 欄位名稱
  • 日期列
  • total / gap / conversion rate 公式
  • 每週檢討問題

```
