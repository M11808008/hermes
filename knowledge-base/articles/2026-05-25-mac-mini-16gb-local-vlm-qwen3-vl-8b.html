Mac mini 16GB 本地 VLM 實測：Ollama Qwen3-VL-8B-Instruct 可行且效果佳

一句話摘要

KC Wong 在 OpenClaw 中文社群分享 Mac mini 16GB 跑本地 VLM 的實測結果：在多個 9B 以下 vision model 中，Ollama Qwen3-VL-8B-Instruct 約使用 6GB RAM，中文辨識與圖像描述表現最佳，成為入門本地 Vision 功能的可行選擇。

重點整理

  • 主題：Mac mini 16GB 跑本地 VLM / Vision model 的可行性。
  • 作者建議新入門者可先用 DeepSeek 協助安裝 Ollama，再跑 `Ollama Qwen3-VL-8B-Instruct`，取得本地讀圖能力。
  • 作者原本使用 Minimax Token Plan，但到期後失去 Vision 功能，因此開始比較 OpenRouter 與本地模型。
  • OpenRouter 上表現較好的 vision model 是 `qwen/qwen3-vl-235b-a22b-instruct`。
  • OpenRouter 成本約每張圖 HKD 0.02，耗時約 20–70 秒，可能受到連線影響。
  • 作者測試多個 9B 或以下模型後，結論是 `Ollama Qwen3-VL-8B-Instruct` 整體最佳。
  • Ollama Qwen3-VL-8B-Instruct 約使用 6GB RAM，速度與 MLX 跑 Qwen 3.5 9B 接近，每張圖約 40 多秒。
  • 許多其他模型問題包含：不懂中文、中文字錯誤、描述品質差、幻覺、16GB RAM 不足等。

實測觀察

MLX Qwen 3.5 9B

  • 蝦餃圖：能看到「極品酒家」與拼音，但年份辨識錯誤。
  • 超市圖：部分飲品辨識錯誤。
  • 道路描述：表現尚可。

Ollama Qwen3-VL-8B-Instruct

  • 可能與 235B 模型類似，能從拼音名字推論「廣州酒家」。
  • 年份能辨識為 1935 年。
  • 超市圖描述完整、準確。
  • 街景描述也表現良好。
  • 在 16GB Mac mini 上約使用 6GB RAM，對本地 Vision 入門相對友善。

OpenRouter qwen3-vl-235b-a22b-instruct

  • 可從拼音名字推論「廣州酒家」。
  • 但因過度推論，超市圖會把商品看成王老吉。
  • 街景看到 YOHO MALL 後，甚至推論出看不到的巴士路線，顯示強模型也可能因過度補全而產生幻覺。

對 Luarn-Pin 的意義

這篇對 AI 學習雷達很有價值，因為它提供了「本地 Vision Agent」的實務門檻參考。Mac mini 16GB 不是高階 AI 工作站，但若 Qwen3-VL-8B-Instruct 能在約 6GB RAM 下穩定完成中文讀圖、街景理解、招牌辨識與商品描述，就代表許多日常圖像任務可以不依賴雲端 API。

這也很適合延伸到 Hermes Agent / OpenClaw / 本地 LLM 工作流：例如讓本地模型先做低成本圖片初判，再把需要高準確度的案例交給雲端模型；或用本地 VLM 處理私人截圖、文件截圖、LINE 圖片歸檔，降低隱私與 API 成本壓力。

可行動建議

  • 若要建立本地 Vision 工作流，可優先測試：

- Ollama

- Qwen3-VL-8B-Instruct

- Mac mini 16GB / Apple Silicon 環境

  • 測試時不要只看單張圖，要至少包含：

- 中文招牌/菜單

- 超市商品

- 街景與遠距文字

- 截圖與文件圖片

  • 評估指標應包含：

- 中文 OCR 準確度

- 是否過度推論

- 幻覺率

- 每張圖耗時

- RAM 使用量

- 是否能穩定串接自動化流程

  • 對私密圖片或大量圖片歸檔，可採用「本地 VLM 初篩 + 雲端 VLM 複核」的混合策略。
