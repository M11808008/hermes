重點摘要

最新版 Claude Code 重新啟用 ultrawork Workflows 功能，透過設定環境變數即可啟動深度研究自動化流程。功能啟動後，Claude Code 會啟動多個 Subagent 並行搜尋、驗證、合併，最終產出含來源的深度研究報告。

---

🚀 啟動方式

### Step 1：設定環境變數

在終端機輸入以下指令後再進入 Claude Code：

```bash

export CLAUDE_CODE_WORKFLOWS=1

export DISABLE_GROWTHBOOK=1

```

### Step 2：啟動 ultrawork

進入 Claude Code 後，輸入：

```

ultrawork 幫我研究最新美國 UFO 的報告

```

### Step 3：查看進度

在 Claude Code 中隨時輸入：

```

/workflows

```

即可查看每個大項的即時進度。

---

⚙️ 深度研究工作流原理

ultrawork 啟動後會執行五個自動化階段：

| 階段 | 說明 |

|------|------|

| **1. Scope** | 把主題拆成多個搜尋角度 |

| **2. Search** | 5 個平行 WebSearch 代理同時搜尋 |

| **3. Verify** | 每個主張用多輪對抗式查核（需 2/3 反駁才剔除） |

| **4. Synthesize** | 合併重複、按可信度排序、附來源 |

| **5. Output** | 產出引用報告（含繁中整理 + 保留英文原始引文） |

實測截圖重點

  • **呈現格式**：在 Claude Code v2.1.152 中 ultrawork 變成彩色，代表已啟動
  • **回饋速度**：作者回覆「呂小霹」的留言提到「幾個 workflow 一堆 agents 真的很燒」，暗示大量 subagent 同時運作，計算資源消耗可觀

---

使用者回饋

> 呂小霹：「身為 Pro 訂閱用戶昨晚看到很開心安裝使用，馬上觸發 limit 到凌晨 1 點，早上起床繼續執行又再度觸發 limit。我知道問題不在 Claude，問題是出在我是小資 Pro 訂閱戶身上。」

→ ultrawork 的多 Subagent 架構會消耗大量 Agent Credits，小資 Pro 用戶需注意配額限制。

---

結論

ultrawork 讓 Claude Code 從簡單對話進化成真正的深度研究助理。適合需要：

  • 多角度文獻調研
  • 事實查核與引用報告
  • 自動化研究流程

的專業使用者。唯需注意 Agent Credits 消耗速度，小資 Pro 訂閱戶建議控制使用頻率。
