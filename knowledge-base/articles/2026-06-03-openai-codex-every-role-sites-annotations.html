OpenAI Codex for every role｜角色插件、Sites 與 Annotations 讓 Codex 走向全職位工作流平台

來源：Threads@resen168；官方公告：OpenAI「Codex for every role, tool, and workflow」

一句話摘要

OpenAI 將 Codex 從工程任務代理擴展成跨職能工作流平台：透過角色專屬 plugins 串接資料、設計、銷售與投資工具；透過 Sites 產生可分享的互動式網站與輕量 App；透過 annotations 讓使用者精準標註局部內容並要求 Codex 修改。

Threads 貼文重點

貼文將這次更新解讀為 Codex 從「寫扣助手」進化為「全職位工作流平台」，特別提到：

  • 行銷、業務、設計、分析師等非工程角色也能使用。
  • 角色專屬插件打包 62 款主流應用與 110 項技能。
  • 使用者不用懂程式，只要用白話文下指令，即可跨軟體自動做事。
  • Sites 可把需求轉成可分享 URL 的互動式網頁應用。
  • Annotations 可針對文件、網站、簡報、圖表的局部區塊精準要求修改。

官方公告重點

OpenAI 官方公告指出，Codex 每週已有超過 500 萬人使用。雖然 Codex 起初是軟體開發工具，但非開發者也正在快速採用：分析師、行銷、營運、設計師、研究員、投資人與銀行業等使用者約占整體 Codex 用戶 20%，且成長速度是開發者的 3 倍以上。

這次 2026-06-02 更新包含三個主軸：

  • 角色專屬 plugins
  • Sites
  • Annotations

1. 角色專屬 plugins

OpenAI 推出六類 role-specific plugins，讓 Codex 能依不同職能預先帶入工具、技能、指令與工作流。官方提到這批插件合計包含 62 個 popular apps 與 110 個 skills。

| 插件類型 | 典型用途 | 範例工具 |

|---|---|---|

| Data analytics | 分析業務與產品數據、解釋指標變動、建立報告與 dashboard | Snowflake、Databricks Genie、Hex、Tableau |

| Creative production | 從 brief 產出 campaign board、廣告變體、產品生活情境圖 | Figma、Canva、Shutterstock、Picsart、Fal |

| Sales | 找高優先客戶、準備會議、追蹤 follow-up、更新客戶紀錄、檢查風險 deal | Salesforce、HubSpot、Slack、Outreach、Clay、Rox、Actively |

| Product design | 將早期想法轉成可評審 prototype、分析 user flow、從 live URL prototype | Figma、Canva |

| Public equity investing | 整理財報、比較公司、追蹤訊號、評估投資 thesis | Moody’s、Daloopa、Datasite、FactSet、LSEG、S&P、PitchBook、Hebbia |

| Investment banking | 將研究與 diligence 轉為 client-ready materials、pitch、comps 分析 | trusted financial data sources |

OpenAI 也預告未來會有 Corporate Finance、Private Equity Investing、Marketing Strategy、Strategy Consulting、Legal 等更多職能插件。

2. Sites：產生可分享的互動式網站與 App

Sites 目前以 preview 形式提供給 Business 與 Enterprise customers。它讓 Codex 不只是產出靜態檔案，而是能建立 hosted websites and apps，並以 URL 分享給 workspace 裡的人。

官方列出的應用包括：

  • dashboard
  • planner
  • review workspace
  • project board
  • gallery
  • lightweight tools
  • customer review site
  • scenario planner
  • product launch hub
  • event operations dashboard

這代表 Codex 可以把分析、計畫、資料與決策流程轉成「可互動的工作空間」，而不是只輸出 PDF、Markdown 或 Excel。

3. Annotations：精準標註、局部修改

Annotations 讓使用者指向要修改的精確區塊，再告訴 Codex 需要改什麼。官方說這個工作方式現在可延伸到 documents、spreadsheets、slides 與 websites。

範例：

  • 選取網站導覽列，要求 Codex 更新字體。
  • 標出投資 thesis 中的一句話，要求 Codex 說明來源。
  • 標註簡報中的圖表，要求加上更清楚的標籤。

這讓 Codex 更適合第一版產出後的審稿與迭代，不必因小錯重生整份內容。

為什麼值得收藏

1. Codex 正在從「寫程式」轉成「做工作」

這次更新的關鍵不只是插件數量，而是定位轉變：Codex 不再只是工程師用來改 code、補測試、送 PR 的工具，而是開始承接跨職能任務，例如分析、簡報、銷售準備、投資研究與活動規劃。

2. 對白領工作流的影響會很大

如果 plugins、Sites 與 annotations 能穩定運作，日常知識工作會被重組成：

  • 先用自然語言描述目標
  • Codex 串接既有工具與資料
  • 產出可互動成果，而不是靜態報告
  • 使用者用 annotation 做精準修改
  • 最後把成果分享給團隊或主管

這會壓縮許多「整理資料、做初版、改格式、做 dashboard、發佈成果」的時間。

3. 和 open-slide / Hermes Agent 工作流可互補

同日收錄的 open-slide 是「讓 Agent 生成簡報」；這篇 Codex 更新則是「讓 Agent 變成跨職能平台」。兩者可組成新的工作流：

  • Codex / Hermes 整理資料與產出分析
  • open-slide 將內容轉成可修改的簡報
  • Sites 將分析模型轉成互動式網頁
  • Annotations 用於精準改稿
  • GitHub Pages / Vercel 負責部署

注意事項

  • Sites 目前是 Business / Enterprise preview，並非所有個人用戶都能立即使用。
  • 角色插件涉及第三方工具與資料權限，企業導入時需由 admin 管控 app permissions。
  • 貼文用語偏行銷化；實際效果仍取決於工具連線、資料品質、權限設計與 workspace 管理。
  • Codex 權限高，若能讀寫文件、CRM、資料庫或專案檔案，必須建立審核、備份與最小權限原則。
