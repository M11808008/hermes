  • 要做產品介紹影片時，可直接跟 Codex 說：

他的評語是：效果雖然粗糙，但比純文字生動多了。

這段很有保存價值，因為它把 ChatCut 的使用門檻濃縮成「安裝 plugin → OAuth → 用 URL + 影片需求 prompt 產出 demo video」。

典型產品介紹影片 prompt

可沉澱成以下模板：

```text

用 chatcut 的 MCP 和 skill，根據以下網址製作一支產品功能演示影片：

【網址】

要求：

  • 先理解產品定位、核心功能與目標使用者。
  • 產出 60-90 秒腳本。
  • 生成圖文並茂的畫面安排。
  • 加入配音解讀與字幕。
  • 補上功能亮點、使用情境、CTA。
  • 在 ChatCut 時間線中完成粗剪並匯出可預覽版本。
  • 最後列出哪些地方仍需人工微調。

```

這類 prompt 很適合用在 SaaS demo、AI 工具介紹、課程頁導覽、產品 launch teaser。

與 Palmier Pro 的差異

同一天也收錄 Palmier Pro。兩者都屬於 agentic video editing，但切入點不同：

  • ChatCut：偏 Codex plugin / MCP / Skill，把 ChatCut 變成 Codex 可操作的影片工作區。
  • Palmier Pro：偏開源 macOS 原生剪輯器，讓 Claude / Codex / Cursor 直接操作 timeline。

共同趨勢是：影片剪輯正在變成「agent 可讀、可改、可驗證」的工作流。

---

實務注意事項

  • OAuth 授權時不要把 token、cookie 或認證檔案貼到公開對話。
  • 第一次使用應讓 Codex 先讀 plugin 文件與 MCP tool list，避免誤用工具。
  • 生成影片仍需人工 QA：字幕、配音、品牌資訊、畫面節奏與事實正確性。
  • 若用 URL 做產品介紹，需檢查網站內容是否可公開使用。
  • 粗剪可交給 AI，正式商用前仍要做版權、素材與音訊授權檢查。
