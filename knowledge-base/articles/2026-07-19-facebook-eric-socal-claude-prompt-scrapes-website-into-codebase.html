This Claude Prompt Scrapes Any Website：把網站「抓進 codebase」前要先拆成資料擷取、重建與合規三件事

來源：Eric Socal 的 Facebook Reel。貼文主張「This Claude Prompt Scrapes Any Website Into Your Codebase」，標籤包含 `#claude`、`#vibecoding`、`#aitools`。縮圖可見創作者對鏡頭講解，白底大字標示「This Claude Prompt Scrapes Any Website」。Facebook 登出頁沒有揭露完整 prompt 或實作細節。

一句話摘要

這則內容可作為「AI 輔助從參考網站萃取設計與結構」的靈感，但不能直接理解成 Claude 能合法、穩定、完整複製任何網站到 codebase。實務上應拆成三層：先用可授權方式擷取公開頁面資料，再讓 Claude Code 產生自己的元件與樣式，最後檢查版權、robots/ToS、資料隱私、反爬與安全風險。

欒帥可用的整理框架

  • Capture：只擷取允許存取的公開資訊，例如 HTML、文字、截圖、顏色、版面結構；保留來源與時間。
  • Interpret：請 Claude 將網站拆成資訊架構、設計 token、component tree 與互動狀態，而不是複製原始碼。
  • Rebuild：在自己的 codebase 中用現有 framework 重建「同類型體驗」，避免照抄品牌資產。
  • Verify：用測試、截圖比對、accessibility audit、responsive checks 確認輸出品質。
  • Comply：檢查 ToS、robots、著作權、商標、個資與資料再利用限制。

實務 prompt 方向

可把社群短影音中的「scrape any website」改寫成更安全的教學 prompt：

```text

請分析這個公開網站的資訊架構與視覺設計，不要複製原始碼、圖片、商標或受保護文案。

輸出：

  • 頁面區塊與 component tree
  • 色彩、字體、間距與互動模式的抽象設計 token
  • 可在我現有專案中實作的 React/Tailwind 元件草稿
  • 與原站不同的文案與品牌化建議
  • 需要人工檢查的授權、隱私與 accessibility 風險

```

實務注意事項

  • 不要把「scrape」等同「clone」。學習版面模式可以，複製專有內容與程式碼風險很高。
  • 若網站禁止爬取、需要登入、含個資或有明確 API/授權限制，應停止或改走官方 API。
  • 對大型或動態網站，Claude Code 通常需要外部工具，例如 Playwright、Firecrawl、截圖 OCR、MCP server 或自寫 scraper。
  • 進 codebase 前先建立隔離分支，避免 AI 把大量不明來源 HTML/CSS 直接混入正式專案。
  • 若是教學示範，建議使用自家網站、開源模板或明確授權範例站，降低法律與倫理風險。

Why it matters

「把任意網站抓進 codebase」代表 vibe coding 的誘惑：速度很快，但邊界很容易失焦。更成熟的做法是把它變成一套可審計的 reverse-design workflow：擷取公開線索、抽象化設計語言、生成自有實作、再由人檢查授權與品質。這樣既保留 Claude Code 對前端原型的速度，也避免把侵權、個資或脆弱 scraper 帶進產品。
