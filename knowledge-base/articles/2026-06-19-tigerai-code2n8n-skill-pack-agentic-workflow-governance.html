TigerAI Code2n8n Skill Pack｜把自然語言與既有程式轉成可治理的 n8n 工作流

來源是 GitHub repository `MorrisLu-Taipei/TigerAI-Code2n8n-Skill-Pack`。這個專案由 n8n Taipei Ambassador Morris Lu 發布，定位為把 AI coding 與企業級 n8n 工作流治理接起來的 Skill Pack：讓使用者用自然語言、便利貼需求，或指向既有 Apps Script / Express / Lambda / Docker stack，由 AI 產出可審查、可交接、可治理的 n8n workflow。

一句話摘要

Code2n8n 不是「把每一行 code 逐字翻成 n8n 節點」的轉譯器，而是一套 porting / review / validation / governance 方法：決定哪些複雜演算法留在 code，哪些觸發、跨系統串接、重試、人工審批、通知與執行歷史應該升到 n8n 可視化流程裡。

核心定位

README 把 Code2n8n 說得很清楚：

  • AI Coding（Claude Code / Codex / Antigravity）擅長寫 code。
  • n8n 擅長讓 code 變成企業可管理、可審查、可交接的 workflow。
  • Code2n8n Skill Pack 是兩者中間的橋。
  • 輸入可以是自然語言需求，也可以是既有系統。
  • 輸出不是單純 JSON，而是包含設計、治理、驗證與交接脈絡的 workflow package。

兩條使用路徑

```text

Path A：from zero

自然語言 / 黃色便利貼需求

→ sticky-note-to-workflow

→ n8n workflow

Path B：port existing system

Apps Script / Express / Lambda / Netlify Functions / Docker stack

→ code-to-workflow

→ code modules + n8n workflow + migration docs

```

專案包含什麼

  • 15 個 Claude Code / Antigravity skills。
  • DSL v1.2 與三層 sticky-note workflow 結構。
  • Cookbook：自然語言到 workflow 的範例。
  • Reference corpus：2,061 個 community workflows，定位是 design lookup corpus，不是 certified templates。
  • 4 個 enterprise patterns。
  • A2A directive：提供多語言 AI-to-AI 指令，規範什麼情況才能聲稱 validated / tested / production-ready。
  • Security Review gate、Validation SOP、scanner、manifest、installer 與 optional live round-trip。
  • Case studies：Google Workspace admin、LINE customer service、on-prem LINE customer service、Taiwan e-invoice unified SDK 等。

安裝與使用脈絡

  • Antigravity：在 n8n 專案中執行 `/install-n8n-pack`，再用自然語言描述需求。
  • Claude Code：在工作目錄執行 `bash install.sh` 或 `install.ps1`，再描述新需求或指向既有 codebase。
  • 其他 AI assistant：可貼 cookbook 範例當 few-shot prompt，模仿三層結構輸出 n8n workflow JSON。

對欒帥的實用判讀

  • 這很適合放進「AI Agent × n8n 自動化」課程，因為它不是只有炫技，而是把企業會問的治理、交接、驗證、責任邊界一起放進方法論。
  • 它可以補上一般 AI coding demo 的缺口：AI 寫得出功能，但企業需要知道流程怎麼被審核、誰能接手、錯誤怎麼回滾、權限怎麼治理。
  • 與 OpenAI Codex Record & Replay、Claude Code Artifacts 可串成一條趨勢線：AI agent 正在從「生成程式」走向「沉澱技能、可視化成果、可治理流程」。
  • 可用於製造業、客服、報表、內部工具、Google Workspace 管理、LINE OA 流程等需要跨系統協作的場景。
  • 導入時要保留 n8n Enterprise / IT 的責任邊界：SSO、IAM、HA、Audit、Source Control、Environments 不是 Skill Pack 自己提供，而是工作流要能乾淨落在企業平台能力之上。

2026-06-22 補充：電子發票 SDK → 14 個 n8n workflows 的實戰案例

Morris Lu 在 n8n Taiwan User Group 補充分享了 v1.0 的實戰里程碑：以 `paid-tw/einvoice` 台灣電子發票 TypeScript SDK 為來源，透過 Claude Code × Codex × Antigravity 與 TigerAI A2A Code2n8n Skill Pack，轉成可治理的 n8n workflow 案例。

Facebook 貼文重點

  • `paid-tw/einvoice` 是台灣電子發票統一 SDK，支援 Amego、ECPay 綠界、ezPay 藍新、ezPay 跨境、ezReceipt，對應財政部 MIG 4.0 情境。
  • TigerAI A2A Code2n8n Skill Pack 將 SDK 轉為 14 個 n8n workflow，涵蓋開立、作廢、折讓、查詢、B2B、載具、外幣、排程、對帳、稽核匯出等流程。
  • 貼文宣稱已對接 Amego 真實 sandbox 測試：Amego 10/10 capability runtime PASS，產生 11 筆真實發票 trace，可在 Amego 後台查驗。
  • 22 個資安 SEC entries 中，20 個已修，1 個 upstream gap 已 mitigated。
  • 4-Tier 外部依賴安全檢查被放進 CI gate。
  • 目標不是只畫流程圖，而是把工程師看得懂的 SDK，轉成公司也管得住、查得到、交接得下去的 n8n workflow。

對欒帥的實用判讀更新

這個補充讓原本的 Code2n8n 概念更具體：它不是只說「AI 可以把 code 變 n8n」，而是用台灣電子發票這種有法規、供應商、稅務、稽核與 sandbox trace 的情境，示範企業級 agent workflow 必須同時處理三件事：

  • **Porting**：把 SDK 包成服務與 n8n workflow，而不是硬把所有邏輯塞進節點。
  • **Validation**：用真實供應商 sandbox、trace 與 capability coverage 證明流程能跑。
  • **Governance**：把資安項目、外部依賴檢查、upstream gap 與 CI gate 放進交付物。

這很適合放進「AI Agent × n8n × 企業自動化」課程：案例本土化、場景夠硬、也能讓學員理解 AI coding demo 與真正可治理工作流之間的距離。

風險與限制

  • Reference workflows 是設計查找素材，不代表可直接上 production。
  • Code2n8n 是 partition methodology，不是萬能 code-to-workflow compiler。
  • Security Review gate 不是完整 SAST / DAST / fuzzer。
  • workflow 上線仍需要 credentials、環境隔離、測試資料、rollback 與權限設計。
  • Facebook 貼文中的 sandbox trace、SEC entries 與 CI gate 數字已與 GitHub README 大方向一致，但本文未登入 Amego 後台或本機執行 repo；正式導入前仍需自行重跑測試與審查 workflow credentials。
  • repo 本身是很好的 agentic engineering case study，但正式採用仍要針對內部 SOP 做審查。
