HY Office AI Newsbot：用 Google 快訊、試算表與 Apps Script 做每日內容自動化

來源：HY Office / Mark 的公開教學頁〈AI 每天自動抓新聞、寫文案、產圖——免費自建教學〉。頁面提供一套以 Google 快訊 RSS、Google 試算表、Apps Script 與 Gemini/Claude API 組成的每日新聞內容自動化流程，並公開完整 Apps Script 程式碼 `code.gs.txt`。

一句話摘要

這是一個很適合中小企業與個人品牌入門的「低成本內容雷達」範本：Google 快訊負責抓產業新聞，Sheets 當資料庫，Apps Script 排程每天整理，AI 從新聞與產品表中挑選角度，產出 FB／IG／LinkedIn 三版文案與生圖提示詞，最後由人審稿後發布。

流程拆解

| 模組 | 角色 | 實作方式 | 注意事項 |

|---|---|---|---|

| Google 快訊 | 新聞雷達 | 針對產業、上下游、客戶產業建立 Alert，接收方式改成 RSS | 關鍵字要加限定詞，避免高雜訊主題把資料庫灌爆。 |

| Google Sheets | 內容資料庫 | 建立快訊設定、新聞資料庫、產品輪播、貼文草稿等分頁 | 適合可視化審稿與人工微調，不需要另架 DB。 |

| Apps Script | 自動化工讀生 | 用排程觸發 `dailyRoutine`：抓 RSS、寫入 Sheets、呼叫 AI、輸出草稿 | Installable time-driven triggers 會以建立者帳號權限執行；權限要保守。 |

| Gemini / Claude API | 選題與文案引擎 | 用 API key 呼叫模型，挑選新聞、分析角度、產出三平台文案與圖片 prompt | API key 要放在 Script Properties，不要硬寫在程式碼。 |

| 人工審稿 | 最後品質閘門 | 每天挑 1 則微調後發文，產品介紹文配實拍，趨勢/觀點文才用 AI 圖 | AI 產圖不是實際產品，避免用於產品細節介紹。 |

公開教學的高價值設計

  • 不是只給概念，而是公開可複製的 Apps Script 程式碼：`https://hy-ai-newsbot.pages.dev/code.gs.txt`。
  • 把內容產線拆成四個 Sheets 分頁：快訊設定、新聞資料庫、產品輪播、貼文草稿，降低非工程使用者的理解門檻。
  • 同一則選題一次產出 FB、IG、LinkedIn 三個平台版本，避免一稿多貼造成語氣不合平台。
  • 內建「近 7 天角度回讀」與角度庫概念，降低 AI 每天寫出同一套說法的機率。
  • 強調乾淨 JSON 輸出與程式端寬鬆解析，這是把 AI 接進自動化流程時很實用的穩定性技巧。
  • 提供 SEO 文章模式：同一條抓新聞→資料庫→AI→草稿管線，只要改提示詞就能從社群貼文切換成 SEO 長文草稿。

官方查證與狀態

  • Brave Search 找到 Google Apps Script 官方文件，確認 installable time-driven triggers 可讓 script 在特定時間或間隔執行，且會在建立者帳號權限下運作。
  • Google Alerts 入口仍存在；RSS 接收方式屬於頁面教學可操作流程，但 Google 官方搜尋結果未明確提供完整 RSS 教學頁，因此應視為「網站實測/教學作者流程」而非官方保證長期介面。
  • Google AI for Developers 官方文件確認可取得 Gemini API key；教學把 Gemini 作為新手推薦路線合理。
  • Anthropic 官方學習頁確認可在 Anthropic Console 建立 API key；若走 Claude 路線，仍需注意模型名稱、定價與可用區域會變動。
  • 本次只做公開頁與程式碼可讀性歸檔，未實際在使用者 Google 帳號中部署 Apps Script，也未驗證作者程式在所有帳號/地區/模型版本下可直接執行。

欒帥可直接採用的教學框架

  • 先讓學生畫出內容產線：資料來源 → 資料庫 → 自動化腳本 → AI → 草稿 → 人審 → 發布。
  • 用 Sheets 當「非工程師也能看懂的 DB」，降低 agent / API 教學阻力。
  • 教 API key 時同步教安全：金鑰是密碼，放 Script Properties，不放公開程式碼、不放共享試算表欄位。
  • 教 AI 文案時同步教輸出格式：JSON schema、禁止 Markdown code fence、程式端容錯解析。
  • 教排程時同步教成本與資料保留：每天跑幾次、每次處理幾筆、保留幾列、失敗時怎麼通知。
  • 教品牌內容時同步教倫理：產品照用實拍，AI 圖只做情境/概念/趨勢配圖，不假裝是真品。

實務注意事項

  • Google Alerts RSS 介面若未來改版，整套流程的入水口需要替換成 Brave Search API、RSSHub、新聞 API 或自家爬蟲。
  • Apps Script 有執行時間、觸發次數、UrlFetch、Sheets 寫入等配額限制；每日 1～2 次通常合理，但大量關鍵字或多人團隊要先估配額。
  • 不要把客戶名單、合約、內部營收、未公開新品資料直接放進 prompt；若要做企業版，需加資料分級與審批。
  • AI 產出的新聞摘要、數字與來源需要人工抽查；尤其引用統計、價格、法規或醫療/金融資訊時不可直接發布。
  • 這類系統最適合先作為「草稿生成器」，不要一開始就全自動發文。
