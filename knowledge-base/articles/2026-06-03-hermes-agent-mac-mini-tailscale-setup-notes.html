Hermes Agent 實機部署筆記｜Mac Mini、Tailscale、Telegram Gateway 與遠端維護

來源：Threads@a1740942002

一句話摘要

這篇 Threads 是一份偏實戰的 Hermes Agent 自架部署筆記，重點不是功能介紹，而是如何讓 Hermes 在一台 Mac Mini 上穩定常駐、可遠端維護、可用既有 Claude / Codex 訂閱，並把 Gateway 與手機救援路徑設計好。

重點整理

1. 為什麼選 Mac Mini，而不是遠端 VM

作者選擇買一台 Mac Mini 安裝 Hermes Agent，而不是先放在遠端 VM，主要原因有兩個：

  • 還不熟 Hermes Agent 的部署與維護細節，先用自己能完全控制的實體機器比較安全。
  • 模型主要使用既有的 Claude 與 Codex 訂閱；在本機環境處理 OAuth access token 會比遠端 VM 更直覺。

這對進階個人使用者很有參考價值：Hermes Agent 並不一定要一開始就雲端化。若重視 OAuth、桌面工具、檔案存取與低摩擦維護，本機 Mac Mini 會是合理起點。

2. Claude 訂閱與第三方 App 的限制

作者提醒：Claude 訂閱預設不支援第三方 App 直接使用。若想讓 Hermes Agent 使用 Claude 訂閱，可參考：

  • https://github.com/kristianvast/hermes-claude-auth

這一點很關鍵，因為很多人會誤以為 Claude Pro / Max 訂閱能直接等同 API 額度。實務上通常需要確認工具支援的授權模式、OAuth 狀態與是否符合供應商條款。

3. 用 Tailscale 解決遠端維護

作者建議在平常使用的電腦與 Agent 的 Mac Mini 上都安裝 Tailscale。這樣即使不在同一個 Wi-Fi，也可以 SSH 回 Mac Mini。

用途包括：

  • Hermes 當機時，遠端執行 hermes gateway restart。
  • 不在家或手邊沒有同網段環境時，仍可維護 Agent 主機。
  • 手機端也可搭配 Termius + Tailscale，臨時 SSH 回主機重啟或調整設定。

這其實是在替個人 Agent 建立「救援通道」。Agent 越常駐，越需要能從外部安全進入主機的維護方案。

4. Telegram Gateway 目前仍是最完整選項

作者選擇 Telegram 作為 Gateway，理由是目前支援度最完整。雖然台灣使用者比較少用 Telegram，但 BotFather、Topic、多對話視窗等能力，對 Hermes Agent 的長期使用很實用。

設定提醒：

  • 用 BotFather 建立 Bot。
  • 記得開啟 Topic。
  • Topic 可讓使用者與 Agent 同時保留多個對話視窗，避免所有任務混在同一條訊息流。

5. Mac Mini 系統設定不可忽略

作者補充幾個 Mac Mini 作為 Agent 主機時的必要設定：

  • General → Sharing → Remote Login：開啟後才能 SSH 回主機。
  • Energy → Prevent automatic sleeping：避免電腦休眠導致 Agent 無法即時回應。
  • Hermes Agent 設定與 Skills 預設放在 ~/.hermes，裝在自己的 Mac Mini 上也方便直接用 IDE 查看與修改。

6. 下一步：備份與遷移

作者下一步想研究如何備份電腦與還原，方便未來 migrate 到新機器。

這也是 Hermes Agent 長期營運會遇到的題目：不只要「跑得起來」，還要能備份設定、技能、記憶、Gateway 設定與本地資料，避免主機更換時重頭配置。

對 Luarn-Pin 的使用價值

這篇可作為個人 Hermes Agent 常駐主機的實務檢查表：

  • 若目標是長期使用 LINE / Telegram Gateway，本機常駐主機比臨時筆電更穩。
  • Tailscale + SSH 是低摩擦且相對安全的遠端維護基本盤。
  • Mac Mini 休眠與 Remote Login 設定應納入部署 checklist。
  • Telegram Topic 很適合多任務、多討論串的 Agent 使用模式。
  • 備份與遷移策略需要提早設計，尤其是 ~/.hermes 內的設定、Skills、記憶與本地知識庫。
