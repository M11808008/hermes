  • 手機 Remote Control 可掃 QR Code 連回家中電腦，遠端控制桌面端 ChatGPT APP。
  • 語音背後模型變得更聰明，且 ChatGPT 本身內建生圖能力。
  • 作者實演「一句話生出 3D 賽車遊戲，並發布到網路上給大家玩」。

更精準的產品解讀

這次改版的核心不是單一模型，而是 OpenAI 正把幾種能力合併成「任務作業系統」：

  • Model tiering：Sol / Terra / Luna 讓使用者用不同成本與速度處理不同任務。
  • Coding agent：Codex 負責 repo、程式碼、測試、PR、網站與工程型任務。
  • Artifact publishing：Sites 讓輸出不只停留在文字，而是能變成可展示、可互動、可分享的網站。
  • Remote execution：Remote Control 讓手機成為控制桌面 agent 的入口，接近「遠端交辦工作」模式。
  • Multimodal creation：語音與生圖能力讓 ChatGPT APP 更像整合式創作與工作環境。

對教學與工作流的價值

對教師、創作者與工作者而言，這則內容有幾個值得追蹤的方向：

  • 教學展示：可從一句話生成遊戲、互動頁、課堂 demo 或作業範例。
  • 原型設計：快速把概念變成可瀏覽網站，降低從 prompt 到 artifact 的摩擦。
  • 遠端工作：手機可在外面交辦家中或辦公室電腦端 agent 繼續執行任務。
  • AI 素養：學生不只學會問答，而是學會規劃任務、選模型、驗收輸出與管理風險。
  • 工具比較：Claude Code、Codex、Hermes 類 agent 的差異會從「誰比較會寫 code」變成「誰比較能完成整條工作流」。

風險與注意事項

  • 網站發布功能若涉及 API Key、學生資料、作業成績或校內資料，必須確認後端代理、環境變數與存取權限，不可把金鑰放到前端。
  • Remote Control 需要同帳號、裝置授權與電腦維持可連線；便利性也代表帳號安全與裝置管理更重要。
  • 影片中的「公開分享」示範要確認是否真的對所有人開放，以及是否會暴露未完成或含錯誤的 demo。
  • AI 生成的遊戲與網站仍需人工測試；影片中也提到方向鍵、難度等問題要再修正。
  • 模型名稱、額度、上下文與方案權限是高變動資訊，建議以 OpenAI 官方文件與帳號內實際顯示為準。

對 AI 學習雷達的保存價值

這則貼文不是單純的新功能介紹，而是記錄了 AI agent 產品的一個重要演化：從聊天視窗、coding assistant，走向「可遠端交辦、可產出網站、可整合多模態輸出」的工作平台。

可用一句框架記住：

  • Chat → Work
  • Prompt → Task
  • Code → Site
  • Desktop → Remote Control
  • Model choice → Cost / speed / reasoning strategy
  • Demo → Deployable artifact
