影片可歸納的核心主張

  • Codex 不再只是處理程式碼，而是能透過電腦 / 瀏覽器控制完成更複雜的任務。
  • Computer Use 類能力讓 agent 可以操作桌面 app、視窗、鍵盤與滑鼠。
  • Browser / Chrome extension 類能力讓 agent 能處理登入後網站、跨分頁資料、DevTools、網站測試與資料蒐集。
  • 若 Codex 能直接使用這些能力，部分原本需要 OpenClaw、Hermes 或其他 automation agent 的場景，可能可以改由 Codex 完成。
  • 真正的落地重點在於：任務設計、授權範圍、帳號隔離、人工 approval 與結果驗證，而不是只看模型名稱。

官方 / Brave Search 查證

Brave Search 與官方文件可確認以下方向：

  • OpenAI Codex Computer Use docs：確認 ChatGPT desktop app 的 Work / Codex 可安裝或啟用 Computer Use plugin，讓 agent 執行電腦操作類任務。
  • OpenAI Codex Browser docs：確認 Codex / ChatGPT 具備 Browser 相關能力，且 Developer mode 可與 Computer Use、built-in browser 搭配，讓 agent 透過 Chrome DevTools Protocol 取得受控存取。
  • OpenAI Codex Chrome extension docs：確認 ChatGPT desktop app 可透過 Chrome extension 讓 Work / Codex 使用瀏覽器情境；搜尋摘要也提到 browser control 不會另外儲存完整 Chrome action record，但會受 Memories 設定影響。
  • OpenAI Codex Plugins docs：確認 plugins 可封裝 skills、apps、MCP、hooks、scheduled task templates 等能力，並在 Work / Codex 內安裝使用。
  • OpenClaw docs 有 Codex harness 相關文件，代表 OpenClaw 不是單純被取代的對象，也可以把 Codex 作為 harness / runtime 的一部分整合。
  • Hermes Agent 則是本機 / 多平台 agent 工作流系統，包含 skills、工具、cron、訊息平台與個人知識庫等長流程能力；是否被 Codex 取代，取決於使用者需要的是桌面 app automation、開源可改造框架，還是長期個人工作流中樞。

因此，影片中「Codex 可操作電腦與瀏覽器」這個方向是有官方來源支持的；但「取代 OpenClaw 和 Hermes」比較像創作者標題與立場，不能當成客觀評測結論。

對 OpenClaw / Hermes 的更精準比較

可把三者想成不同層級：

  • Codex / ChatGPT desktop：偏向使用者可立即使用的商業 agent app，強項是模型能力、官方 plugins、Computer Use、Chrome / Browser 整合與 ChatGPT 帳號體驗。
  • OpenClaw：偏向開源 agent / automation framework，可整合不同 model provider、harness、Computer Use 與自訂 runtime，適合想改造底層與自建工作流的人。
  • Hermes Agent：偏向個人 / 團隊 agent operating layer，強項是持久記憶、skills、訊息平台整合、定時任務、檔案與知識庫自動化、以及可在 LINE / Telegram 等渠道中長期服務使用者。

若任務是「在桌面上臨時操作一個網站或 app」，Codex Computer Use 可能非常方便。若任務是「長期自動接收連結、歸檔、部署、跨平台回報、排程監控」，Hermes / OpenClaw 類框架仍有不同價值。

實務風險與安全提醒

  • 電腦與瀏覽器控制比一般聊天更高風險，因為 agent 可能接觸登入狀態、cookie、密碼管理器、下載檔案、本機資料與付款頁面。
  • 建議使用 sandbox browser profile、demo 帳號、低權限帳號與獨立測試資料，不要直接讓 agent 操作主帳號或生產環境。
  • 需要明確設定人工 approval：送出表單、下單、刪檔、寄信、發文、提交 PR、改 production config 前都應停下確認。
  • 對企業或課堂環境，應建立操作日誌、螢幕錄影或可回放紀錄，方便事後追蹤 agent 做了什麼。
  • 標題中的「全自動完成複雜任務」要保守理解：越複雜的任務越需要中途驗證、錯誤恢復與人類判斷。

可轉成課堂練習的方式

任務：設計一個「Browser / Computer Use 安全實驗」。

  • 準備一個 sandbox Chrome profile 與 demo 帳號。
  • 指派任務：登入測試網站、下載範例 CSV、整理成表格、產生簡報或簡短報告。
  • 要求學生列出 agent 可能接觸到的敏感資訊。
  • 設定三個必須人工確認的 checkpoint：登入後、下載前、提交/發送前。
  • 最後比較 Codex、OpenClaw、Hermes 類工具在「立即可用性、可自訂性、長期自動化、權限治理」上的差異。

保存重點：Codex 的 Computer Use / Browser / Chrome extension 讓 AI Agent 更接近真正操作工作環境；但「能操作」不等於「可以無限制自動化」，未來的核心能力會是權限治理、任務分解、可驗證輸出與安全回退。
