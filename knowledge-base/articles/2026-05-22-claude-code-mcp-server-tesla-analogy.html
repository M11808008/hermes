Claude Code 沒裝 MCP Server，就像買 Tesla 還用腳踩

一句話摘要

這則 Facebook Reel 用一句很鮮明的比喻提醒：「用 Claude Code 沒裝 MCP server，等於買了 Tesla 還在用腳踩」，核心觀點是 Claude Code 若沒有接上 MCP servers，就無法充分使用外部工具、資料源與工作流能力。

重點整理

  • 原文重點：

- 「用 Claude Code 没装 MCP server，等于买了 Tesla 还在用脚踩。」

- 英文版：「Using Claude Code without MCP servers is like buying a Tesla and still stepping on pedal.」

  • 這個比喻的意思是：Claude Code 本身很強，但如果沒有連接 MCP server，就像只用到基本能力，沒有把工具生態打開。
  • MCP server 的價值在於把 AI coding agent 接到外部能力，例如：

- 檔案與專案工具。

- GitHub / issue / PR 工作流。

- 文件、資料庫、瀏覽器、搜尋或內部系統。

- 可重複使用的工具接口與上下文來源。

  • 對 AI coding agent 而言，模型能力只是其中一部分；能不能安全、穩定、可控地使用工具，會決定實際工作效率。
  • 這類短影音很適合作為「AI agent 工具鏈」課程中的開場比喻：不是只買模型，而是要把模型接上工具與流程。

對 Luarn-Pin 的意義

這則內容與 Luarn-Pin 目前關注的 Hermes、Codex、Claude Code、MCP、Computer Use 與個人知識庫自動化高度相關。它提醒一個重要原則：AI agent 的價值不只在聊天或生成程式碼，而是在正確配置工具後，能真正進入工作現場執行任務。這也可用來解釋為什麼 Hermes Agent 的工具、skills、gateway、cron、browser、file 與 MCP 連接都很關鍵。

可行動建議

  • 檢查目前常用 coding agent 是否已接上必要工具：GitHub、檔案系統、瀏覽器、文件庫、資料庫或內部 API。
  • 在團隊導入時，不只比較模型，也要比較：
