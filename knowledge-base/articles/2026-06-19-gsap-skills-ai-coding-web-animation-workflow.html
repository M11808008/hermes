GSAP Skills｜讓 Claude、Cursor、Codex 寫出更高質感網頁動畫的官方 AI 技能包

來源是 AI郵報在 Threads 分享的 GSAP Skills 實測串文，並連到 AI郵報完整文章與 GreenSock 官方 GitHub repository。Threads 的核心觀點是：AI 會寫程式，但做網站常有「紫藍漸層、玻璃卡片、普通 fade in」的模板味；GSAP Skills 補上的是動畫技術正確性，真正的高質感仍需要設計參考與明確的負面約束。

一句話摘要

GSAP Skills 是 GreenSock 官方為 AI coding agents 準備的技能包，讓 Claude Code、Cursor、Codex、Windsurf、Copilot 等工具更正確使用 GSAP 的 timeline、ScrollTrigger、plugins、React hooks、cleanup 與 performance pattern；它解決「動畫寫法正確」，但不會自動解決「設計品味」。

| 關鍵 prompt | 不要紫藍漸層球、不要玻璃卡片、不要普通 fade in、不要制式 AI SaaS landing page |

內建 8 個 Skills

| Skill | 用途 |

|---|---|

| gsap-core | `gsap.to()` / `from()` / `fromTo()`、duration、stagger、easing、defaults |

| gsap-timeline | sequencing、position parameter、labels、nesting、playback |

| gsap-scrolltrigger | scroll-linked animations、pinning、scrub、refresh、cleanup |

| gsap-plugins | ScrollToPlugin、ScrollSmoother、Flip、Draggable、SplitText、MorphSVG、CustomEase 等 |

| gsap-utils | clamp、mapRange、normalize、interpolate、random、snap、toArray、selector 等 helper |

| gsap-react | React useGSAP hook、refs、`gsap.context()`、cleanup、SSR 注意事項 |

| gsap-performance | transforms over layout props、will-change、batching、ScrollTrigger 效能建議 |

| gsap-frameworks | Vue、Svelte 等框架 lifecycle、selector scoping、unmount cleanup |

對欒帥的實用判讀

  • **這是「AI 做前端」從能跑到有質感的關鍵補丁**：AI coding tools 本來就會寫 HTML / CSS / JS，但常缺動畫結構與效能判斷；GSAP Skills 讓它更像懂 GSAP 的前端工程師。
  • **適合做課程示範**：可用同一個 landing page prompt，示範「未安裝 skill」與「安裝 GSAP Skills + 設計參考」的差異。
  • **不要把它當設計師替代品**：GSAP Skills 解決動畫 API 與 pattern；高質感仍來自 reference、art direction、節奏判斷、品牌語彙與人工 review。
  • **很適合搭配你的知識庫／課程頁設計標準**：若要做有質感的 course landing、AI dashboard、互動簡報或 demo site，可把 GSAP Skills 當成 coding agent 的動畫上下文。
  • **prompt 要先做設計方案，不要直接 code**：先要求 AI 產出視覺風格、Hero 分鏡、ScrollTrigger 敘事、Dashboard 動畫與排除項目，確認後再實作，能大幅降低模板味。

可複製的工作流

```text

  • 安裝 GSAP Skills。
  • 收集 3–5 個高質感網站參考：creative developer portfolio、digital magazine、premium product site、AI intelligence dashboard。
  • 明確列出不要什麼：不要紫藍漸層球、不要玻璃擬態卡片、不要普通 fade in、不要制式 AI SaaS 模板。
  • 先請 AI 產出設計方案，不要寫 code。
  • 確認 Hero、scroll narrative、dashboard preview、micro-interactions 與 transition 節奏。
  • 再請 AI 實作 HTML / CSS / JS 或 React / Vue / Svelte。
  • 逐段 review animation timing、cleanup、performance 與可讀性。

```

可用 Prompt 片段

```text

請使用 GSAP 做一個高質感互動網站，但不要做成一般 AI SaaS 模板。

避免紫藍漸層球、玻璃擬態卡片、普通 fade in、制式 landing page。

方向請參考 creative developer portfolio + AI intelligence dashboard + digital magazine。

請先不要寫 code，先產出完整設計方案：視覺風格、Hero 進場分鏡、ScrollTrigger 滾動敘事、Dashboard 動畫、Login 到 Dashboard 轉場、要用哪些 GSAP 技術、要避免哪些老套效果。

等我確認設計方向後，再開始實作。

```
