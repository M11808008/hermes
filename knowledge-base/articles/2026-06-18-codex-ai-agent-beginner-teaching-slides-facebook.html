Codex AI Agent 代理人教學簡報｜從問 AI 到指揮 AI 助理

來源是呂聰賢老師的 Facebook 貼文「Codex AI Agent 代理人教學簡報」。貼文搭配多張簡報圖，主軸是把初學者從「問 AI 問題」帶到「指揮 AI 助理執行專案任務」。本文以 Facebook 貼文作為發現來源，並用 OpenAI Codex 官方文件與 GitHub repo 補強可驗證資訊。

一句話摘要

Codex 的教學重點不只是「讓 AI 寫程式」，而是理解 agentic workflow：它能在指定專案資料夾中讀檔、改檔、執行命令、根據 AGENTS.md 延續專案規則，並在使用者授權與驗證下完成整理、開發、測試與部署任務。

建議教學示範流程

| 階段 | 示範任務 | 驗證方式 |

|---|---|---|

| 1. 只讀探索 | 請 Codex 說明這個資料夾有哪些檔案、用途與可能重複項目 | 不改檔，只輸出清單與建議 |

| 2. 小範圍整理 | 只針對 5 個測試檔案改名，先不要動全部資料 | 人工確認命名規則正確 |

| 3. 建立規則 | 寫一份 AGENTS.md，規定檔名用連字號、不得用底線、變更前先列計畫 | 再次執行時檢查 Codex 是否遵守 |

| 4. 自動化輸出 | 產生 CSV 檔案清單或 Markdown 索引 | 比對檔案數與欄位是否正確 |

| 5. 專案任務 | 在 Git repo 中修一個小 bug 或補 README | git diff、測試、人工 review |

欒帥可複用的入門 Prompt

```text

請先不要修改任何檔案。請讀取這個專案資料夾，完成三件事：

  • 用表格整理主要檔案與資料夾用途。
  • 找出可能需要整理、重新命名或補文件的地方。
  • 提出一個最小安全修改計畫，列出會改哪些檔案、為什麼、如何驗證。

等我確認後，才可以開始修改。

```

導入提醒

  • **先選對工作資料夾**：Codex 能操作的是你授權的 project folder；不要在含敏感資料或未備份資料夾中做初學測試。
  • **先要求計畫再執行**：初學者應先用「請先不要修改」建立安全邊界。
  • **用 Git 或備份保護現場**：程式專案用 Git；一般檔案整理則先複製一份 sample folder。
  • **AGENTS.md 是團隊規則，不是聊天記憶**：穩定規範應放在 repo 裡，避免靠臨時 prompt 反覆提醒。
  • **驗證比生成重要**：每個任務都要有檔案清單、測試、diff、截圖或輸出報表作為完成證據。
