15 種組合的實務判讀

| 組合 | Reel 主張 | 實務上應如何理解 |

|---|---|---|

| GitHub | 更快 code review 與 PR 管理 | Claude Code 可讀 repo、改檔、產生 commit/PR 摘要；GitHub CLI/API 或 MCP 可補上 issue、PR、review context。 |

| Docker | 在隔離環境 build/run app | 很適合用來建立可重現測試環境；agent 應被限制在專案容器或 devcontainer，降低污染本機風險。 |

| Supabase | 幾分鐘建立 DB/backend | 可協助產生 schema、SQL、RLS policy 與 client code，但 production 權限與資料遷移仍需人工 review。 |

| Figma | 把設計轉成可實作 UI | 可從設計 tokens、截圖或 MCP/外掛取得線索，再轉 HTML/CSS/React；仍需設計 QA 與 responsive 檢查。 |

| Playwright | 撰寫與執行自動化測試 | 是 Claude Code 很關鍵的驗證工具：讓 agent 不只改 code，也能用 browser test 看到錯誤。 |

| Vercel | 一鍵部署 app | 可自動準備 build settings、環境變數檢查與 deploy 指令；secret 與 production deploy gate 應保留人工確認。 |

| Linear | 任務轉 code | 可把 issue acceptance criteria 轉成修改計畫與分支工作；避免讓 agent 在未釐清需求時直接實作。 |

| Notion | 自動建立與更新文件 | 適合把 PR 摘要、release note、runbook、meeting action items 回寫知識庫。 |

| Sentry | 找錯、分析、修復 | Sentry issue context 可餵給 agent 做 root cause debugging；修復後仍要測試與回歸驗證。 |

| Postgres | 設計 DB 與查詢 | 可協助 SQL、index、migration；資料破壞性操作必須有 dry-run/backup/approval。 |

| Cloudflare | Edge 部署與效能 | 可產生 Workers/Pages 設定、快取規則與 deploy workflow；環境變數、KV/D1/R2 權限需控管。 |

| GitHub Actions | 自動 CI/CD | Anthropic 已有 Claude Code GitHub Actions 文件；適合做 PR automation、review、測試修復，但要設定權限邊界。 |

| MCP Servers | 串接外部工具與服務 | MCP 是把 Claude Code 接到資料源/工具的標準路徑之一；要注意 server trust boundary 與 token scope。 |

| Terminal | 直接執行系統命令 | Claude Code 的核心能力之一；最好搭配 sandbox、git diff、測試、lint 與明確允許清單。 |

| Browser Automation | 自動測網頁與操作頁面 | 可用 Playwright/browser tools 驗證 UX、登入流程、表單與 visual regression；不可把它當成人類 QA 的完全替代。 |

對 AI 學習雷達的意義

這則貼文可以放進「coding agent stack」主題：

  • 第一層是 agent shell：Claude Code/Codex 讀專案、改檔、跑測試。
  • 第二層是 verification tools：Playwright、Docker、CI、lint、typecheck。
  • 第三層是 delivery tools：GitHub、Vercel、Cloudflare、GitHub Actions。
  • 第四層是 knowledge/ops tools：Linear、Notion、Sentry、Postgres、Supabase。
  • 第五層是 connector layer：MCP servers、API、CLI、browser automation。

它也很適合做課堂示範：讓學生比較「只問 ChatGPT 怎麼寫功能」與「讓 agent 在 repo 中修改、測試、部署、回寫文件」的差異。

Caveats

  • Reel 的 15 個組合是高層次地圖，不等於每個工具都有同等成熟、官方、零設定的 Claude Code integration。
  • 讓 agent 連接 GitHub、Sentry、Notion、Vercel、Cloudflare 等服務時，token scope、審批流程、費用上限與資料外流風險要先設計。
  • Browser automation 與 terminal 權限很強，應搭配 sandbox、專案目錄限制、git diff review、測試與人工 gate。
  • 「一人像一隊」是生產力隱喻，不代表可以省略產品判斷、資安、架構設計、code review 與上線責任。

Why it matters

Claude Code 類工具的價值不只是「幫我寫一段程式」，而是把開發流程變成可觀測、可驗證、可回滾的 agent loop。這類內容很適合轉成欒帥課程中的工作流案例：AI agent 的關鍵能力，是能拿工具、看結果、修正錯誤，而不是只產生漂亮文字。
