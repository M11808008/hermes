Claude Code × LibTV｜一句話自動生成 15 秒手機廣告的 AI Agent 工作流

來源：Threads 貼文示範把 Claude Code、Hao0321 開源 skill 與自己的 LibTV 串接後，只輸入一句「幫我做一支 15 秒手機廣告」，AI 就自動完成分鏡、生成與出片。此筆以 Threads 為發現來源，並以 `libtv-labs/libtv-skills` README / GitHub API 補充可重複實作的技術脈絡。

一句話摘要

這則案例的價值不只是「AI 生成影片」，而是把 Claude Code 變成創意總控，透過 Agent Skill 呼叫 LibTV 的生圖/生影片能力，自動拆解廣告任務、產生分鏡 prompt、送出生成、查詢進度並取得成片結果。

技術脈絡：LibTV Skill 是什麼

`libtv-labs/libtv-skills` README 說明，這是一組面向 AI Agent 的 Skills，透過 LibLib.tv / LibTV OpenAPI 提供 AI 生圖、生影片等 AIGC 能力。Agent 可以建立會話、發送創作指令、查詢生成進度，並下載最終圖片或影片結果。

為什麼值得收藏

1. 這是「影片工具」轉成「Agent 可調用能力」的典型案例

傳統影片工具通常需要人進 GUI、調參數、排時間線；這個案例展示的是另一種結構：Claude Code 負責理解任務與拆工作，LibTV skill 負責把自然語言創作任務丟給生成平台，再用腳本查詢與下載結果。也就是把影片生成變成可編排的 API workflow。

2. Prompt 本身就是可教學的分鏡模板

作者附上的 prompt 很適合拆成課程素材：Style / Duration / Scene / Shots / Visuals / Action / Details / Atmosphere。這套結構能幫學員理解「好看的 AI 影片不是只有一句 prompt」，而是需要鏡頭、節奏、材質、光線、情緒與品牌語言。

3. 創作角色正在從手工執行轉向導演與系統設計

這個例子不代表人類創作者不重要，而是角色轉移：人需要定義品牌、審美、敘事、限制與檢核標準；AI Agent 負責把可描述的流程大量執行。對廣告、課程、社群短影音、產品 demo 來說，這會大幅降低原型成本。

欒帥可延伸應用

  • 課程 demo：用同一個分鏡模板，讓學員改成「課程招生 15 秒廣告」、「AI 工作坊短宣傳」、「NCCU 課程形象片」。
  • 知識庫分類：可歸到「AI Agent / 自動化」、「多媒體與內容」、「Claude Code」三類，作為 agentic creative workflow 範例。
  • 實作研究：若要本機複製，先驗證 `libtv-skills` 安裝、`LIBTV_ACCESS_KEY`、會話建立、進度查詢與下載結果，不要先追求完整商業片品質。
  • 品質控管：建議建立評分表，例如品牌一致性、鏡頭連貫、產品外觀穩定、文字/logo 是否錯誤、是否侵權或過度仿 Apple 風格。

風險與注意事項

  • 商業使用前要注意品牌風格相似度；「Apple-style」適合做美學參考，但正式客戶稿應避免誤導或侵犯品牌識別。
  • 生成影片可能有產品細節漂移、鏡頭連續性、logo/文字錯誤、比例不穩等問題，仍需人類 review。
  • 若影片素材含真實產品、商標、人物肖像或客戶資料，需確認授權與平台資料處理方式。
  • Threads 貼文未提供完整執行環境、模型、成本、生成耗時與成片檔案，因此此筆歸檔把它視為 workflow 案例，而不是可直接保證相同結果的教學。
