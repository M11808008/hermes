原文重點

Prompt Case 轉述的流程是：

  • 讓 Codex 掃描手機相簿，找出照片中曾出現過的衣物。
  • 對每件衣物建立 bounding box，裁切出可用參考圖。
  • 交給 GPT-Image 重新生成背景乾淨、風格統一的商品圖。
  • 將所有衣物整理成數位衣櫥。
  • AI 可根據現有衣物推薦新穿搭，甚至把推薦造型合成到本人身上，預覽實際穿著效果。
  • 作者公開分享 `Extract Clothing Skill`，使用者提供照片資料夾或允許 Codex 探索相簿即可啟動。

這個 Skill 值得注意的設計

`Extract Clothing Skill` 不是簡單的「去背工具」，而是把複雜影像工作流拆成可驗證步驟：

  • 發現並正規化來源照片，處理 EXIF orientation 與格式轉換。
  • 建立 source contact sheets，要求視覺檢查，不只靠檔名推斷。
  • 盤點每一件刻意穿著的衣物，建立 manifest 與 normalized bounding boxes。
  • 對每件衣物建立 crop reference，再用 image generation 重建完整商品圖。
  • 去重時要求來源照片支持「同一件衣物」，不能只靠生成圖相似。
  • 最終輸出只保留透明 RGBA PNG，臨時資料不污染成果資料夾。

這讓它很像一個小型個人商品目錄生成器，而不是一次性的圖片編修 prompt。

對 AI 學習雷達的價值

這個案例示範了 AI Agent 與影像生成真正變成「個人資料整理工具」的方向。重點不在單張漂亮圖片，而在把相簿中未結構化的生活資料轉成：

  • 可搜尋的衣物清單；
  • 可重用的透明商品圖；
  • 可搭配的 wardrobe inventory；
  • 可做虛擬試穿與個人風格建議的資料基礎。

這也很適合延伸到其他領域：家具盤點、藏書封面整理、食材庫存、展覽作品建檔、商品 catalog 生成等。

---

風險與實務提醒

  • 相簿高度私密：若讓 Agent 掃描手機相簿，需確認本機權限、上傳範圍與雲端模型資料政策。
  • 生成圖不是原始照片切割：Skill 也明確說明，這是依可見證據重建，不是 literal segmentation。
  • 不要猜不可見細節：被身體或外層遮住的衣物結構，應標為 unknown 或略過。
  • 人像與身體要排除：輸出應只保留衣物，不應包含皮膚、頭髮、身體、道具或背景。
  • 虛擬試穿需標示合成：若對外展示，應避免讓人誤以為是真實拍攝成果。
