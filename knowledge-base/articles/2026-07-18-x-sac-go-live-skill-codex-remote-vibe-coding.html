Go Live.skill：讓手機遠端 Codex Vibe Coding 不再卡在 localhost 預覽

來源：Sac (@Saccc_c) 的 X Article〈这个SKILL真正能让你的 Codex 实现远程 Vibe Coding〉。原文指出 Codex 遠端操作已推出，手機可控制電腦裡的 Codex 工作；但手機端做 Vibe Coding 時有一個核心痛點：改完本地網站後，手機無法直接打開 localhost 看效果。因此作者做了 Go Live.skill，讓 Codex 在完成改動後自動部署公開 URL，手機即可打開檢查。

一句話摘要

Go Live.skill 的價值不是「讓 Codex 會部署」這件事本身，而是把遠端手機工作流的驗收標準改成：不要只回 localhost；改完前端後要跑檢查、沿用既有部署管線、打開公開 URL 驗證，最後回傳可在手機打開的連結。

原文主張

  • Codex remote / 手機端操作讓使用者能從手機控制電腦上的 Codex 工作。
  • 手機 Vibe Coding 最大問題是：本地網站 localhost 不能直接在手機打開。
  • Go Live.skill 的做法是讓 Codex 改完後部署到線上，回傳公開預覽連結。
  • 安裝方式：CLI 使用 `npx skills add Sac-Y/go-live`；或在 Codex / ChatGPT mobile 對 Codex 說「請幫我安裝這個 skill：https://github.com/Sac-Y/go-live」。

GitHub / SKILL.md 查證

GitHub repo `Sac-Y/go-live` 確認存在，README 與 SKILL.md 說明如下：

  • 名稱：Go Live.skill。
  • 定位：手機優先的 Codex workflow；當使用者不方便查看 localhost 時，要求 Codex 部署並驗證 app / website / frontend change，再回傳公開 URL。
  • 部署策略：不綁定單一平台，優先沿用專案既有部署方式，例如 Vercel、Netlify、Cloudflare、Firebase、GitHub Pages 或 CI/CD。
  • 驗證要求：部署後要打開公開 URL，確認頁面可訪問且能看到最新修改。
  • 失敗處理：若缺少登入、project linking、環境變數、commit/push 或 build 失敗，Codex 應說明阻塞點與下一步，不可直接宣稱成功。

官方背景查證

OpenAI / ChatGPT Learn 的 Remote connections 文件確認：

  • ChatGPT mobile app 可透過 Remote 存取連接的 Mac / Windows 主機上的 ChatGPT 或 Codex chats。
  • Remote 可送 follow-up instructions、回答問題、批准命令、查看 outputs / diffs / test results / terminal output / screenshots，並在任務完成或需要注意時通知使用者。
  • Remote 使用 connected host 的 projects、files、credentials、permissions、plugins、Computer Use、browser setup 與 local tools。

這支持作者描述的「手機端接手電腦工作」方向；Go Live.skill 則補上「手機無法看本機 localhost 預覽」這個實務缺口。

欒帥可用場景

  • 課程 demo：讓學員理解 agent 工作流的最後一哩不是「改完程式」，而是「可被使用者在真實裝置驗收」。
  • 手機遠端工作：外出時用 ChatGPT mobile 指揮桌面 Codex 修改前端，要求它每輪給公開 preview URL。
  • Hermes 對照：這和 AI 學習雷達部署流程相同——寫完內容不算完成，公開頁面 200 OK 且有 marker 才算完成。
  • 前端 / prototype：適合小型網站、demo、Dashboard、教學頁、landing page；不適合未設好權限與 secrets 的正式生產系統直接全自動發布。

注意事項

  • 公開 URL 代表資料可能被外部看到；不可把內部文件、API keys、學生個資或客戶資料部署到公開預覽。
  • 如果專案已有正式 production deployment，不應為了預覽而覆蓋正式站；應使用 preview branch、staging、Vercel preview 或 GitHub Pages sandbox。
  • 若部署需要 commit/push、登入授權或環境變數，必須明確由使用者批准或提供安全替代方案。
  • Skill 本身是第三方社群 repo，不是 OpenAI 官方內建 skill；安裝前應檢查 SKILL.md 與執行權限。
  • 手機遠端批准命令時更容易快速點過，需特別注意 destructive action、公開發布與費用。

Why it matters

這則 X Article 是一個很好的 harness engineering 小案例：真正可靠的 agent workflow 不是只讓 AI 能遠端寫 code，而是把「部署、公開連結、可視化驗收、失敗回報」寫成可重複的 skill。對 Codex / Hermes / Claude Code 類 agent 來說，這種 skill 會把 Vibe Coding 從本機玩具推向可驗證的遠端生產力流程。
