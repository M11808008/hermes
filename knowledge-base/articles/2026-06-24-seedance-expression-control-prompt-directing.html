原始 15 秒表情控制 Prompt 結構

0–2 秒：含蓄起始

低頭垂眸，嘴角噙含蓄淡笑，不露齒，神態溫柔安靜。

2–4 秒：笑意收斂，注意力轉向

笑意緩緩收斂，嘴角歸平；雙眼、頭部慢慢抬起，視線轉向畫面右側，神情轉為認真留意。

4–6 秒：疑惑與擔憂浮現

持續望向右方，眼神添上淺淡疑惑與擔憂，雙眼微睜（幅度自然），嘴唇輕啟呈欲言又止狀，眉眼微收，不皺眉。

6–8 秒：情緒下沉

視線緩緩下沉，頭部微低，笑意盡數褪去，神情安靜失落，內斂壓抑情緒，不流淚、不哭泣。

8–10 秒：壓抑後的自嘲

低頭輕閔雙眼，平復心緒，嘴角浮現一抹藏起難過的淺淡自嘲式淺笑，呼吸自然，動作極輕。

10–12 秒：重新迎光

緩緩抬頭，眼眸重新迎向柔光，眼神漸亮卻依舊克制，臉部轉向前方 / 偏右，唇角帶一絲溫柔又委屈的淡意。

12–15 秒：凝視收尾

目視前方 / 畫面右側，眼神柔和微潤，唇瓣輕動似欲語還休，最終保持安靜凝視狀態，定格在溫柔清冷、略帶憂鬱的情緒中，結尾無黑屏、無轉場。

可重用的寫法拆解

1. 先切時間軸，不要只寫情緒形容詞

這份 prompt 的有效之處在於它不是只說「憂鬱、溫柔、失落」，而是把每 2 秒的狀態拆開。AI 影片模型通常更容易跟隨具體時間節點，而不是抽象的一段感覺。

可重用格式：

  • 0–2 秒：初始姿態與情緒基調。
  • 2–4 秒：第一個注意力轉折。
  • 4–6 秒：情緒變化開始被看見。
  • 6–8 秒：核心情緒下沉或爆發。
  • 8–10 秒：角色自我壓抑或反應。
  • 10–12 秒：情緒回收或重新轉向。
  • 12–15 秒：定格收尾，避免黑屏與突兀轉場。

2. 把臉部表演拆成可觀察部件

貼文中反覆使用「眼神、雙眼、頭部、視線、嘴角、嘴唇、眉眼、呼吸」等具體部件。這比單純寫「很難過」更像導演給演員的動作指令。

可用欄位：

  • 視線：看前方、下沉、右移、重新迎光。
  • 頭部：低頭、慢慢抬起、微低、偏右。
  • 嘴部：含笑、不露齒、嘴角歸平、唇瓣輕動。
  • 眼部：垂眸、微睜、微潤、眼神漸亮。
  • 眉眼：微收、不皺眉、克制。
  • 呼吸 / 動作：自然、極輕、不要過度表演。

3. 同時寫正向要求與禁止事項

「不露齒、不皺眉、不流淚、不哭泣、無黑屏、無轉場」是重要限制。這類負面條件可以降低模型產生過度戲劇化畫面的機率，但仍需用實際出圖檢查，因為留言中也有人指出畫面可能仍違反部分細節。

4. 把 prompt 當成劇本與分鏡，而不是神奇咒語

這則貼文的啟示是：AI 影片生成越接近角色表演，就越需要文學、劇本、分鏡、演員調度與導演語言。對教學來說，可把它轉成「情緒弧線設計練習」，讓學生先寫時間軸，再用模型測試哪些描述最穩定。

與 Seedance 2.0 官方定位的關係

ByteDance Seed 官方頁面稱 Seedance 2.0 支援文字、圖片、音訊與影片等多模態參考，並強調 performance、lighting、shadow、camera movement 等導演級控制。BytePlus ModelArk 的 Dreamina / Seedance prompt guide 也把 prompt 寫作視為生成品質的重要環節。

因此，這則 Threads prompt 可以視為一個「表演控制」案例：它把官方所說的 director-level control 落在臉部表演與情緒節奏上，而不是只控制鏡頭或燈光。

給欒帥的使用價值

AI 課程示範

很適合拿來教「AI 影片不是按鈕生成，而是導演語言」。課堂可讓學員把同一段情緒改寫成三種版本：溫柔、壓抑、崩潰，觀察模型是否跟得上細節。

品牌角色短片

若未來要做 KOL 角色、課程引導人物、客服 avatar 或產品情境劇，可以用這種秒數節點控制表情與情緒，不必只依賴模型自行理解抽象角色設定。

Prompt 模板化

可以做成一個表情控制模板：

  • 角色身份與情境。
  • 起始姿態。
  • 逐秒情緒節點。
  • 臉部 / 眼神 / 嘴角 / 頭部 / 呼吸欄位。
  • 禁止事項。
  • 收尾畫面。
  • 生成後人工檢查清單。

建議檢查清單

  • 每個時間節點是否只有一個主要情緒變化？
  • 是否把抽象情緒轉成可見的臉部與身體動作？
  • 是否指定視線方向與鏡頭收尾？
  • 是否有過度矛盾的指令，例如同時要求低頭又直視鏡頭？
  • 是否列出禁止事項，但不把限制寫得過多導致模型混亂？
  • 生成後是否逐秒檢查，不只看整體氛圍？
