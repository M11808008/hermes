工作流拆解

1. 錄製：先把內容輸入標準化

影片中提到的起點不是複雜攝影棚，而是 QuickTime + 提詞機。重點是先降低錄製摩擦：

  • 用提詞機讓內容結構更穩定。
  • 用固定版面或直式影片格式降低後製變因。
  • 不追求一次完美，後面交給剪輯流程處理停頓與字幕。

對一人公司 / 知識創作者來說，這是一個很好的原則：錄製端只要穩定產出素材，不要把所有品質壓力放在拍攝當下。

2. Whisper：把口語內容轉成 SRT

第一個技術節點是語音轉文字。影片提到用 Whisper 產生字幕稿 / SRT。

官方驗證：OpenAI Whisper 是開源語音辨識模型，官方 GitHub repo 為 `openai/whisper`，OpenAI 也有 Introducing Whisper 發布文。

在短影音流程中，SRT 不只是字幕，而是整個自動化的時間軸資料：

  • 可用來找停頓。
  • 可用來切分段落。
  • 可用來產生大字標題。
  • 可作為日後重新輸出影片的最小保存格式。

3. 字幕燒錄：把 SRT 變成可控版面

影片描述提到「把 SRT 做成一張張 PNG 燒上去」。這代表字幕不是單純交給剪輯軟體自動疊字，而是把字幕視為可程式化的視覺元素：

  • 每段字幕對應時間戳。
  • 每段文字可轉成圖片或 HTML canvas。
  • 可控制字體、大小、顏色、描邊、位置。
  • 可做成品牌一致的模板。

這讓 Claude Code 能處理字幕版面，而不需要懂傳統剪輯 GUI 操作。

4. Auto-Editor：自動剪掉停頓

影片提到用 Auto-Editor 自動剪掉停頓並加入推鏡頭效果。

官方驗證：常見的 auto-editor 專案為 WyattBlue/auto-editor，定位是 effort-free video editing，可依音量、靜音、動作等條件自動剪輯。

這一步的價值在於：

  • 去掉口語錄製中的空白與停頓。
  • 讓剪輯節奏更接近短影音平台期待。
  • 把粗剪從人工時間軸操作，轉成程式規則。

但實務上仍需人工抽查，因為自動剪停頓可能切掉語氣、反應時間或重要轉折。

5. Pexels API：讓 AI 自動抓 B-roll

B-roll 通常是短影音最花時間的部分。影片提到用 Pexels 免費素材庫，並申請 API Key 讓 AI 自動抓 B-roll。

官方驗證：Pexels API 可程式化存取 Pexels photo / video library，官方文件說內容可免費使用，但仍需遵守 API terms、credit、logo、request limit 等規則。

這一步的核心不是「素材免費」，而是把 B-roll 搜尋變成機器可執行任務：

  • 從字幕或腳本抽關鍵詞。
  • 查 Pexels 影片素材。
  • 依主題、畫面方向、長度挑候選。
  • 下載並插入對應段落。

風險：AI 可能挑到語意相近但情境不準的素材，因此 B-roll 仍需要人工快速檢查。

6. Claude Code：把整套流程接成可執行 pipeline

Claude Code 在這個工作流中的角色，不是單純寫一個腳本，而是把素材處理、字幕、剪輯、資料夾管理、模板輸出串成 pipeline。

適合交給 Claude Code 的任務包括：

  • 讀取原始影片與音訊。
  • 呼叫 Whisper 產生 SRT。
  • 呼叫 Auto-Editor 產出粗剪。
  • 呼叫 Pexels API 搜尋與下載素材。
  • 依模板疊字幕、大字與 CTA。
  • 建立輸出資料夾。
  • 產出可人工檢查的中間檔與成品。

這很符合近期 AI coding 的趨勢：把創作流程工程化，讓 AI 代理人處理重複、可規則化、可驗收的部分。

檔案管理：六個資料夾的思路

影片章節提到「剪完的檔案怎麼管理：六個資料夾結構」。雖未能取得逐字稿細節，但可推定這類流程應至少區分：

  • input：原始錄影 / 音訊。
  • transcript：SRT / TXT / 分段稿。
  • b-roll：下載或生成的素材。
  • intermediate：粗剪、字幕圖層、暫存檔。
  • output：最終輸出影片。
  • templates / config：版型、字體、配色、CTA、參數。

AI 自動化最怕檔案混亂。若沒有資料夾約定，Claude Code 很容易把輸出、暫存與素材混在一起，後續很難重跑或修正。

儲存策略：只留 SRT，把社群當影片儲藏庫？

影片章節提到「每天剪很多支怎麼存？只留 SRT，影片上傳社群當儲藏庫」。這是一個很值得討論的知識工作流觀點。

好處：

  • 節省本機硬碟空間。
  • SRT 保留了內容、時間軸與可再加工文本。
  • 影片成品由社群平台承擔儲存與分發。

限制與風險：

  • 社群平台不是可靠長期備份，可能壓縮、下架、限流或帳號風險。
  • 只留 SRT 無法完整保留畫面、B-roll、剪輯節奏與版型。
  • 若未保存 config / prompt / template，未來很難重現同一版本。

建議：至少保存 SRT、final prompt、config、模板、成品低解析 proxy，以及重要原始素材索引。

模板化：三區式版面與自拍風模板

影片提到兩種短影音版面：

三區式版面

  • 上方：標題 / hook。
  • 中間：影片主畫面。
  • 下方：CTA 或重點文字。

這類模板適合資訊型短影音，因為觀眾一眼能知道主題，畫面與文字也能分工。

隨手自拍風 + 封面遮罩壓字

這類模板更接近社群原生感：

  • 看起來不像過度製作。
  • 容易保留真人感。
  • 可用封面遮罩與大字強化點擊率。

重點是：作者不是要 AI 從零發明美術，而是給 AI 截圖 / 樣式參考，讓它仿版。這跟設計工作流中的 style reference / visual target 很接近。

Remotion / Hyperframes：把 motion graphics 變成程式

影片後段提到用 Remotion / Hyperframes 讓 AI 做動畫特效。

官方驗證：

  • Remotion：官方定位是用 React programmatically make videos，可把 React component 渲染成真實 MP4。
  • Hyperframes：官方站與 HeyGen HyperFrames 皆將它描述為用 HTML / CSS / JS 讓 AI agents compose videos 的方向，適合 browser-based motion graphics。

這代表短影音製作正在從剪輯軟體 GUI，轉向「可程式化影片」：

  • 字幕是資料。
  • 版面是 component。
  • 動畫是 code。
  • 輸出是 render pipeline。

對 Claude Code 這類 coding agent 來說，這比操作傳統 timeline 更友善。

CapCut Lab 與 Mega Prompt

影片章節提到 CapCut Lab 課程與 Mega Prompt 自動生成 Motion Graphics。Brave Search 可驗證 CapCut 官方有 AI design / motion graphics 相關資源，也具備 text-to-video、auto captions、模板與 4K export 等 AI-first video editing 能力。

但「CapCut Lab 課程」與「Mega Prompt」本身較像教學產品 / 課程方法，應視為創作者的工作流設計，而不是 CapCut 官方單一標準功能。保存時應避免把它寫成已被官方完整驗證的產品特性。

最值得保存的洞見：買剪輯課是在買「語言」

影片最後的章節標題很重要：買剪輯課真正的價值，是學會用技術名詞跟 AI 溝通。

這點非常符合 AI 時代的學習重點：

  • 不一定每個人都要成為專業剪輯師。
  • 但要知道剪輯語彙：B-roll、jump cut、caption burn-in、mask、CTA、motion graphics、push-in、SRT、render、template。
  • 有了專業詞彙，才能把需求精準交給 Claude Code 或其他 agent。
  • AI 不是替代所有專業，而是放大懂流程、懂語彙、懂驗收的人。

對 AI 學習雷達的實踐價值

這支影片可放入「AI 內容工作流 / 一人公司自動化」類別，保存價值高，原因如下：

  • 它把短影音製作從剪輯軟體操作，轉成可被 AI coding agent 執行的 pipeline。
  • 它示範了 Claude Code 不只適合寫產品程式，也可用於創作者後製流程。
  • 它把影音素材、字幕、B-roll、模板、motion graphics 都轉成可管理的資料與程式。
  • 它提醒「自動化」不是取消人類，而是讓人類從時間軸苦工移到版型、風格、驗收與策略。
  • 它能延伸到課程短影音、AI 學習雷達精華切片、社群貼文再利用等場景。

建議的可落地 SOP

若要把這套流程落到欒帥 / Hermes 的內容產線，可設計為：

  • 錄製原始影片：QuickTime + 提詞機 + 固定收音。
  • 產生字幕：Whisper → SRT。
  • 清理文本：修正專有名詞、斷句與標點。
  • 粗剪：Auto-Editor 去停頓。
  • 抽重點：Claude Code 依 SRT 找 hook、章節與短影音片段。
  • B-roll：Pexels API 依段落關鍵詞抓候選素材。
  • 套模板：三區式版面 / 自拍風 / 課程宣傳版。
  • 動畫：Remotion 或 Hyperframes 產生 motion graphics。
  • 人工 review：檢查字幕、素材語意、版權、CTA 與輸出規格。
  • 發布與歸檔：保存 SRT、prompt、config、template、成品 URL。

風險與注意事項

  • 影片未提供 transcript，本筆記依 metadata 與章節整理，細節可能不如逐字稿完整。
  • Pexels 素材雖可免費使用，但仍需遵守授權、API terms 與 attribution 指引。
  • Whisper 轉寫需人工校正專有名詞，尤其是中文人名、產品名與英文術語。
  • Auto-Editor 自動去停頓可能破壞語氣與敘事節奏，需抽查。
  • AI 自動抓 B-roll 可能語意不準或風格不一致，需人工驗收。
  • Remotion / Hyperframes 偏工程化，初期建置成本高，但長期可重複使用。
  • 社群平台不應是唯一影片備份；至少保存 SRT、模板、config 與成品索引。
