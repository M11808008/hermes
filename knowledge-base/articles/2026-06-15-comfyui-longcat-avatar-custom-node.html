ComfyUI-LongCat-Avatar｜LongCat-Video-Avatar 1.5 的 ComfyUI 節點與 CUDA/Mac 限制

來源為 rookiestar28 在 Threads 宣布開源第 7 個 ComfyUI 節點：`LongCat-Avatar 1.5`。作者補充第一版是 CUDA-only，暫不支援 macOS 的 MPS 後端。GitHub README 進一步顯示，主線專案名為 `ComfyUI-LongCat-Avatar`，用途是把官方 LongCat-Video-Avatar 1.5 的音訊驅動人像影片生成流程包成 ComfyUI custom nodes。

一句話摘要

這個節點讓 ComfyUI 使用者能在節點式工作流中操作 LongCat-Video-Avatar 1.5，做單音訊/多音訊的人像說話影片生成；但主線仍以 NVIDIA CUDA 為核心，Apple Silicon 目前只適合追蹤 `macOS-MLX` 實驗分支，不應期待 MPS 直接可用。

支援能力整理

| 能力 | 狀態 |

|---|---|

| Avatar 1.5 single audio ai2v | 支援 |

| Avatar 1.5 single audio at2v | sampler mode 支援，但目前 ComfyUI graph 仍需要 image socket |

| Avatar 1.5 multi audio | 支援 ai2v |

| 480p / 720p | 支援 |

| Distill inference | Avatar 1.5 必要；README 預設 8 steps、text CFG 1.0、audio CFG 1.0 |

| 單檔 DiT safetensors | 支援，放在 ComfyUI/models/diffusion-models/ |

| 官方 sharded DiT | 支援 Avatar 1.5 base-model/ checkpoints |

| 官方 INT8 sharded DiT | 支援 base-model-int8/ checkpoints |

| 自動下載已知官方權重 | 支援部分 sharded DiT 與 shared LongCat text encoder assets |

| Attention backends | auto、sdpa、flash-attn-2、flash-attn-3、xformers、sageattn；sageattn-3 保留但會 fail fast |

| GGUF DiT | 尚不支援 |

| Avatar 1.0 | 不在此 ComfyUI contract 支援範圍 |

| CPU / MPS | 主線 CUDA-only path 不支援 |

安裝與模型檔重點

```bash

cd ComfyUI/custom-nodes

git clone https://github.com/rookiestar28/ComfyUI-LongCat-Avatar

cd ComfyUI-LongCat-Avatar

pip install -r requirements.txt

```

README 強調要使用啟動 ComfyUI 的同一個 Python 環境，且預期已有相容 NVIDIA GPU、CUDA PyTorch 與 FFmpeg。`requirements.txt` 刻意不安裝 torch、torchvision、torchaudio、FlashAttention、xFormers、SageAttention 等大型或堆疊敏感套件，以免破壞既有 ComfyUI 環境。

必要模型檔包括：

  • `ComfyUI/models/loras/longcat-avatar-dmd-lora.safetensors`
  • `ComfyUI/models/vae/LongCat-Video-Avatar-vae.safetensors`
  • `ComfyUI/models/audio-encoders/whisper-large-v3.safetensors`
  • DiT 來源三擇一：官方 sharded、官方 INT8 sharded、或單檔 `LongCat-Video-Avatar-1.5-int8.safetensors`
  • Text encoder 可走官方 tokenizer/text-encoder 或既有 `Load CLIP` fallback
  • `Kim-Vocal-2.onnx` 為可選 vocal separation 模型

官方 LongCat-Video-Avatar 1.5 背景

官方 LongCat-Video README 在 2026-05-21 宣布 LongCat-Video-Avatar-1.5：它是音訊驅動人像影片生成框架的升級版，將 Wav2Vec2 換成 Whisper-Large 以提升唇同步，強調物理合理性、時間穩定性、長影片能力、風格化領域泛化、單音訊與多音訊輸入，並透過 step distillation 加速到 8 steps。

因此 ComfyUI-LongCat-Avatar 的價值不是重新訓練模型，而是把官方 Avatar 1.5 權重與推論流程轉成 ComfyUI 可組裝的節點，讓既有 ComfyUI 使用者能把 talking-avatar 生成接進自己的圖像/影片工作流。

對欒帥的實用判斷

  • 若目標是 Windows / Linux + NVIDIA GPU 的 ComfyUI 影片生成工作站，這個節點值得列入測試清單。
  • 若目標是 Apple Silicon Mac，主線不適合直接期待可用；可追蹤 `macOS-MLX` 分支，但應視為實驗而非穩定方案。
  • 若要做課程示範，建議把它放在「開源 ComfyUI 影片生成節點」單元，並明確標註硬體門檻與模型檔配置。
  • 若要做長期生產，應先驗證：模型下載大小、VRAM、720p 時間、音訊品質、唇同步、長片穩定性、授權與輸出素材合規。

判讀限制

  • Threads 貼文只提供開源訊息與 CUDA-only 限制；技術細節以 GitHub README 與官方 LongCat repo 為準。
