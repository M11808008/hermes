章節架構

此 repo 把 agent architecture 拆成 22 個章節，分層理解：

  • Foundations：harness thesis。
  • Core Loop：agent loop、tool runtime、permission & sandbox、hooks。
  • Complex Work：planning / todos、subagents、skills、context management。
  • Knowledge & Resilience：memory、system prompt assembly、error recovery。
  • Long Running & Async：task system、background execution、scheduling、worktree isolation。
  • Multi Agent：coordination、protocols、autonomy。
  • Extension & Integration：MCP / plugins / channels、observability & evaluation。
  • Composition：loop engineering。

這種拆法很適合作為 AI agent 工程課程大綱：每一章都對應一個實際機制，而不是停留在抽象名詞。

Skills 章節的價值

第 7 章 Skills 將 skill 定義為「self-contained bundle of expertise, instructions plus any scripts and files, loaded only when a task needs it」。它把 Skills 解釋成 progressive disclosure：

  • Catalog：只把 name、description、path 放進 prompt，讓模型知道有哪些技能。
  • Instructions：需要時才讀 `SKILL.md` 本文。
  • Resources：技能引用到的 scripts / references / templates 再按需讀取。

這對理解 ChatGPT Skills、Claude Skills、Codex plugins、Hermes skills 都很有幫助：skill 的工程重點不是把所有知識塞進 system prompt，而是讓 agent 能便宜地發現能力、必要時載入完整工作流，並在用完後留下可演化的 procedural memory。

為什麼值得放進 AI 學習雷達？

這份 repo 的價值是把「agent 變強」拆成可以教、可以實作、可以驗證的架構部件。對欒帥的課程、工作坊或內部工具設計來說，它可以作為：

  • AI agent architecture 課程主線。
  • Hermes / Claude Code / Codex 等工具比較框架。
  • 解釋 memory、skills、MCP、background task、cron、worktree isolation 的教材。
  • 建立「harness engineering」語彙：把模型能力與外層工程責任切開。

尤其 repo 直接把 Hermes Agent 列為研究系統之一，對目前的 AI 學習雷達與愛馬仕小小斌工作流有高度關聯。

實務閱讀建議

  • 先讀 `00-harness-thesis`：建立 model vs harness 的分工觀念。
  • 再讀 `01-agent-loop` 到 `04-hooks`：理解最小 agent loop 與副作用控管。
  • 接著讀 `07-skills`、`09-memory`、`14-scheduling`、`19-mcp-plugins-channels`：對照 Hermes Agent 目前的能力。
  • 若要做課程，可讓學員跑各章節 `src/test.py` / `demo.py`，再比較相鄰章節 diff，觀察每層機制如何加入 loop。
