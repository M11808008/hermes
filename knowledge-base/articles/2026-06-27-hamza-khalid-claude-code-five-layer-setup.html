Hamza Khalid｜Claude Code 5 分鐘修復：CLAUDE.md、Skills、Hooks、Subagents 與 Agent Teams 五層設定

Hamza Khalid 的新貼文分享 Anthropic agent loop 工作坊，同時引用他先前的長文《Your Claude Code Is Broken. Here's the 5-Minute Fix.》。這篇長文主張：多數人不是 Claude Code 用不好，而是第一天就沒有給它 onboarding；沒有規則、記憶與護欄，就像雇用聰明助理卻沒告訴他為誰工作。

一句話摘要

Claude Code 要穩定，不是靠一次更好的 prompt，而是靠五層工作區設定：CLAUDE.md、Skills、Hooks、Subagents、Agent Teams。

五層設定

| 層級 | 原文命名 | 作用 |

|---|---|---|

| 1 | The Instruction Manual / CLAUDE.md | 每次 session 開始都讀取的規則、偏好、專案背景與禁區。 |

| 2 | The Specialists / Skills | 可被召喚的預訓練小專家，例如 review、測試、文案、簡化、語氣檢查。 |

| 3 | The Security Guard / Hooks | 不可跳過的安全規則，例如改檔前檢查、改檔後自動跑品質檢查、重要事件通知。 |

| 4 | The Second Opinion / Subagents | 全新上下文的第二個 Claude，只讀不改，用來抓主 session 看不到的錯。 |

| 5 | The Team / Agent Teams | 多個 Claude session 分工、共享任務、互相溝通；適合更大型工作。 |

CLAUDE.md 的重點

原文建議把 CLAUDE.md 視為新同事 onboarding 文件，而不是長篇百科：

  • 你是誰、在做什麼專案。
  • 專案如何啟動、測試、部署。
  • 程式風格與命名規則。
  • 哪些目錄或檔案不能碰。
  • 完成任務前必須跑哪些檢查。
  • 保持短而清楚，原文建議低於 200 行。

Hooks 為什麼重要

一般 instructions 可能被模型忽略或覆蓋，但 Hooks 是工具執行前後的硬性守門員。原文用交通燈比喻 exit code：0 表示可以繼續，2 表示紅燈停止。

適合用 Hooks 管住的高風險情境：

  • 刪除檔案、重置 Git、改動設定檔。
  • 觸碰密鑰、環境變數、付費 API 或 production。
  • 修改後必須自動跑 lint、test、type check。
  • 任務完成時通知人類檢查。

Subagents 的使用原則

Hamza 特別強調：第二意見 subagent 最好「只讀不改」。它的任務是檢查完成品、找盲點、指出風險，而不是自己下手修。若 reviewer 也能改，它就可能變成問題的一部分。

日常工作流

| 時機 | 操作 |

|---|---|

| 開始前 | 按 Shift + Tab 進入 Plan Mode，先看計畫再批准。 |

| session 變長 | 每 10–15 分鐘用 `/compact` 壓縮上下文。 |

| 做完任務 | 儲存工作，下一個任務開新 session。 |

| 出錯或方向不對 | 用 Esc Esc rewind 回退 Claude 的上一個動作。 |

與川野 Codex 文章的對照

這篇 Claude Code 設定文與川野的 Codex 工作系統文其實講同一件事：coding agent 的成熟度不在模型本身，而在配置層。

| Claude Code 語彙 | Codex / 通用 agent 語彙 |

|---|---|

| CLAUDE.md | AGENTS.md / 專案規則檔 |

| Skills | 可復用 workflow / specialist |

| Hooks | 權限、sandbox、pre/post checks |

| Subagents | 第二意見、平行研究、審查者 |

| Agent Teams | 多代理工程團隊 / planner-builder-judge |

對欒帥的可用意義

  • 可作為「AI agent onboarding」教材，讓學生理解不是開工具就能穩定產出。
  • 可延伸成課堂作業：請學生為自己的專案寫一份短版 CLAUDE.md / AGENTS.md。
  • 可搭配 Agent Loop 文章，說明 Loop 之前必須先有規則、技能、驗證與安全邊界。
  • 可用於企業 AI 導入：先建立最低限度的專案護欄，再談多代理與自動化。
