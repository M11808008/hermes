一句話摘要

這支 Reel 指向一個實用方向：與其每次要求 Claude「寫得像人」，不如把個人語氣、禁用 AI 腔、改寫規則與範例文章整理成 Claude Skill，讓模型在需要寫作或潤稿時自動載入同一套風格規範。

官方驗證：Claude Skills 是什麼

Anthropic 的 Agent Skills / Claude Code Skills 文件與 `anthropics/skills` repo 說明，Skill 是一個包含 instructions、scripts、resources 的資料夾，Claude 會在特定任務中動態載入，以提升專門工作流的表現。

官方 repo 的描述重點包括：

  • Skills are folders of instructions, scripts, and resources。
  • Claude 會在需要時載入技能，用於重複性、專門化任務。
  • Skill 可用於文件產出、企業品牌規範、資料分析、個人自動化等。
  • 每個 skill 通常有 `SKILL.md`，描述何時使用、流程、規則與驗證方式。

因此，把「寫得像某個人」變成 Claude Skill，在概念上是合理的：它把每次都要重講的寫作偏好變成可重用的上下文。

公開 Humanizer Skill 參考

Brave Search 與 GitHub 查到 `blader/humanizer` 這類公開 skill，可作為同類工作流的參考。其 `SKILL.md` 說明：

  • 目標是移除 AI-generated writing 的痕跡。
  • 會掃描 promotional language、過度象徵化、被動語態、rule of three、AI vocabulary、filler phrases、em dash overuse 等模式。
  • 要求 rewrite 而不是只刪除，並保留原意。
  • 支援 voice calibration：如果使用者提供自己的寫作樣本，skill 會分析句長、用字、段落開頭、標點習慣與常見語氣，再套用到改寫。

這驗證了 Reel 主題背後的可行模式：不是只叫模型「更自然」，而是列出可檢查的反 AI 味規則與個人風格訊號。

可複用工作流：寫作型 Claude Skill

一個「Writing Like a Human」skill 可以包含：

  • 觸發條件：當使用者要求撰寫、改寫、潤稿、社群貼文、newsletter、文章摘要時使用。
  • 反 AI 味規則：避免空泛形容、過度條列、行銷腔、模板轉折、過度對稱句型。
  • 語氣規範：正式 / 口語 / 專業 / 幽默 / 犀利 / 高質感等。
  • 個人範例：收錄 2-5 篇真實文字，作為 voice DNA。
  • 改寫流程：先診斷 AI 味，再提出修改策略，最後輸出自然版本。
  • 驗證清單：是否像真人、有具體細節、有節奏變化、有立場，而不是安全但平板。

為什麼 Skill 比單次 prompt 更好

單次 prompt 的問題是：

  • 每次都要重貼規則。
  • 模型容易忘記語氣細節。
  • 不同任務會產生不一致風格。
  • 團隊成員各自寫 prompt，品牌聲音不穩定。

Skill 的優勢是：

  • 可版本管理與持續迭代。
  • 可加入參考資料、範例與檢查表。
  • 可在 Claude Code / Claude 環境中重複觸發。
  • 適合把個人、品牌或課程的語氣規範產品化。

對 AI 學習雷達的判讀

這則 Reel 值得保存，因為它代表 AI 寫作工具從「提示詞技巧」走向「可維護的風格資產」。當內容產量增加時，最重要的不是每篇都重新 prompt，而是建立一個可複用、可修正、可審核的寫作 skill。

對欒帥的應用脈絡，它可以延伸成：

  • 欒帥個人風格 Skill：固定繁中、專業優雅、高質感、少 AI 腔。
  • 課程文案 Skill：招生頁、講義、社群貼文維持一致語氣。
  • AI 學習雷達摘要 Skill：新聞歸檔時維持同一種判讀框架。
  • Humanizer / editor Skill：把初稿先寫快，再用 skill 進行人味化與審稿。

實作注意事項

  • 不要只寫「make it human」；要列出具體可檢查的壞味道與替代策略。
  • 需要真實寫作樣本，否則模型只能套用泛用「自然」風格。
  • 風格 skill 應該保存偏好與範例，不應保存一次性任務細節。
  • 如果用於品牌或課程，需定期檢查是否過度口語、過度誇張或失去專業性。
  • 對外發布前仍應人工審稿，尤其是招生、商業承諾與敏感主題。
