OpenAI Codex OAuth｜OpenClaw 與 Hermes 可用的 ChatGPT 訂閱帳號認證教學

來源為電腦王阿達文章〈OpenClaw、Hermes 都能用！教你怎麼使用 OpenAI 帳號透過 OAuth 認證養龍蝦〉，重點是使用 OpenAI / ChatGPT 付費帳號，透過 OAuth 方式讓 OpenClaw 或 Hermes 使用 Codex 類模型，而不是走傳統 OpenAI API Key 計費模式。

一句話摘要

這篇文章值得收錄的原因，是它清楚區分了「OpenAI API Key 用量計費」與「OpenAI Codex OAuth / ChatGPT 訂閱帳號授權」兩條路徑：前者走平台 API 帳單，後者走 ChatGPT / Codex 訂閱權益，對 Hermes 與 OpenClaw 使用者的成本結構差很多。

OAuth 使用條件與限制

原文指出，使用 OpenAI 帳號授權給 OpenClaw / Hermes 仍然有條件：

  • 至少需要 OpenAI Plus 會員。
  • 付費會員也不是無限量，仍受方案額度限制。
  • Plus、Pro 等級不同，額度與使用量上限不同。
  • Codex 分析頁面可觀察目前使用量。
  • 若使用 API Key，則會走 OpenAI Platform API 用量與帳單；這與 OAuth 授權不同。

OpenClaw 設定流程整理

原文以 Linux 為例，macOS 概念相同，Windows 則可透過 WSL2 / PowerShell 操作。

  • 開啟終端機。
  • 執行 `openclaw config` 進入設定。
  • 選擇 `LOCAL`。
  • 進入 `Model` 設定。
  • 選擇 `OpenAI Codex`，不要選走 API Key 計費的 OpenAI API。
  • 選擇 `Browser Login`。
  • 瀏覽器開啟後，用 OpenAI / ChatGPT 帳號登入。
  • 授權完成後回到終端機。
  • 選擇 GPT-5.5 OAuth 類模型。
  • 選擇 Continue 結束設定。
  • 執行 `openclaw gateway restart` 重啟 session / gateway。

Hermes 對應觀念

Hermes Agent 的對應重點不是照抄 OpenClaw 指令，而是理解「provider / model / OAuth」三件事：

| OpenClaw 概念 | Hermes 對應理解 |

|---|---|

| OpenAI Codex | 使用 OpenAI Codex provider / OAuth provider |

| Browser Login | 透過 Hermes OAuth 登入流程授權 |

| API Key OpenAI | OpenAI Platform API Key 計費模式，與 ChatGPT 訂閱不同 |

| gateway restart | 設定後通常需重啟 Hermes gateway 或開新 session |

Hermes 技術參考指令可包含：

```bash

hermes model

hermes login --provider openai-codex

hermes gateway restart

```

實際可用選項仍以當前 Hermes 版本、`hermes model` 互動選單與 `hermes doctor` 檢查為準。

為什麼對欒帥重要

  • 欒帥目前 Codex CLI 是 ChatGPT OAuth auth，不是 OPENAI_API_KEY billing；這篇可作為「OAuth 授權 vs API Key 計費」的外部說明來源。
  • 若之後要教學 Hermes / OpenClaw 成本結構，這篇文章可作為中文入門案例。
  • 使用者常誤以為所有 OpenAI 模型使用都會扣 API Key 帳單；本文正好能澄清 ChatGPT 訂閱權益與 API 平台帳單不同。
  • 若遇到額度問題，應先確認是 ChatGPT / Codex 訂閱額度、OpenAI API 額度，還是 LINE / Gateway 推播額度，不能混為一談。

實務檢查清單

| 檢查項目 | 指令 / 位置 |

|---|---|

| Hermes 狀態 | `hermes status --all` |

| Hermes 設定 | `hermes config` 或 `hermes model` |

| OpenAI Codex OAuth | `hermes login --provider openai-codex` |

| Codex CLI 登入狀態 | `codex login status` |

| Codex 健康檢查 | `codex doctor` |

| Gateway 重啟 | `hermes gateway restart` |

注意事項

  • 不要把 ChatGPT Plus / Pro 額度與 OpenAI Platform API Key 額度混在一起。
  • OAuth 模式通常看的是 ChatGPT / Codex 介面中的用量，而不是 OpenAI Platform Usage。
  • 若在 LINE 上收不到回覆，可能是 LINE Messaging API 月推播額度用完，與 Codex / OpenAI 額度不同。
  • 設定 provider 或 model 後，Gateway 平台通常需要重啟才會吃到新設定。

延伸觀察

AI Agent 的成本正在分成兩條路：一條是 API Key 逐量計費，一條是訂閱帳號透過 OAuth 授權使用。對個人創作者與教學者而言，OAuth 模式的最大價值是降低心理門檻與帳務複雜度；但它仍然有方案額度與服務條款限制，適合一般工作流，不應被視為無限資源。
