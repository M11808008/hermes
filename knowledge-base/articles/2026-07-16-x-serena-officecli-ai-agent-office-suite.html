一句話摘要

OfficeCLI 值得追蹤的重點，是它把 Office 文件變成 AI Agent 可讀、可改、可預覽、可迭代的命令列介面：不只是產生 `.docx/.xlsx/.pptx`，還能把文件渲染成 HTML / PNG，讓多模態 Agent 「看見」輸出效果後再修正。

專案設計觀察

1. Office 文件被轉成可程式化 DOM

OfficeCLI 的核心價值不是「又一個 CLI」，而是把 Office 文件拆成 Agent 可操作的元素：建立檔案、加入 slide/shape/paragraph/cell、設定屬性、查詢結構、移除元素、讀取 JSON。對 AI Agent 來說，這比直接操控 Word/Excel/PPT GUI 更穩定，也更容易做版本化與錯誤回復。

2. Render → Look → Fix 是關鍵閉環

README 強調內建 HTML rendering engine，能把 `.docx/.xlsx/.pptx` 渲染成 HTML 或 PNG。這很重要：Agent 只會改 XML/DOM 還不夠，必須能看到排版結果。渲染成 HTML/PNG 後，多模態模型可以檢查版面、重疊、表格寬度、投影片視覺層級，再回頭修正。

3. `SKILL.md` 把使用方法包成 Agent 操作規格

OfficeCLI repo 內的 `SKILL.md` 本身就是一份 Agent skill：

  • 使用情境：建立、分析、校對、修改 Office 文件。
  • 安裝方式：macOS/Linux/Windows 指令。
  • 策略：L1 read → L2 DOM edit → L3 raw XML，優先用高層操作。
  • Help-first 原則：不確定 command/property 時先跑 help，不要猜。
  • 支援 resident mode，降低反覆開檔 I/O 與檔案鎖問題。

這對 Hermes 技能設計也很有參考價值：好的 Skill 不是只有工具介紹，而是包含「遇到不確定時如何查 schema、何時 save/close、如何避免檔案鎖」等操作規則。

風險與注意事項

  • **不要把行銷語「first and best」當客觀結論**：這是 repo README 的定位語，本文只保留為專案主張。
  • **Office 相容性仍需實測**：不需要安裝 Office 不代表與 Microsoft Office、Google Docs/Sheets/Slides、Keynote/Numbers/Pages 的顯示完全一致。
  • **複雜版面需要人工審稿**：商務簡報、正式合約、財務表、招生素材仍需人工檢查字體、表格、圖表與頁面溢出。
  • **巨集與外部連結要謹慎**：處理外部 Office 檔案時需注意 macro、嵌入物件、外部連結與潛在惡意內容。
  • **個資與內部文件不可公開部署**：若用於名單、報名資料、公司文件，輸出應保留在私密本機/私有空間，不進公開知識庫。

可轉成課堂示範的練習

  • 給 Agent 一份課程大綱，要求建立 5 頁 PPT。
  • 用 OfficeCLI render 成 HTML/PNG。
  • 讓多模態模型檢查投影片是否文字過多、重點不清或版面擁擠。
  • 用 OfficeCLI 指令修改 slide title、shape position、font size、table style。
  • 再 render 一次，比較修正前後。

這個練習能清楚展示 Agentic Office Automation 的核心：不是一次生成完美文件，而是建立可觀察、可修改、可驗證的迭代回路。
