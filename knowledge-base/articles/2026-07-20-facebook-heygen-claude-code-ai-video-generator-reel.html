欒帥可用的工作流判讀

  • Input：研究資料、課程大綱、逐字稿、Notion/Obsidian 筆記、Slack/Gong/會議紀錄。
  • Agent：Claude Code/Codex 先整理重點、產生腳本、分鏡、avatar voice/tone、CTA 與 metadata。
  • Integration：用 HeyGen MCP、Skills 或 Direct API 把腳本與 avatar/video 參數送進 HeyGen。
  • Output：生成短影音、教學片、內訓影片、產品更新或社群素材，並把 video URL / log 存回專案或知識庫。
  • Review：人工檢查事實、語氣、品牌、授權、肖像、字幕、無障礙與平台規範後才發布。

對 AI 學習雷達的意義

這則 Reel 可以和近期的 Hyperframes、Remotion、Claude motion editing 類內容放在同一條趨勢線：

  • Hyperframes / Remotion 偏向程式化 motion graphics、HTML/React/renderer 工作流。
  • HeyGen 偏向 avatar video、AI presenter、translation、lipsync 與商務/教育影片生產。
  • Claude Code / Codex 的角色是 orchestrator：把研究、腳本、素材描述與 API 呼叫串起來，而不是自己「直接變成剪輯軟體」。

對課程設計來說，這是很好的範例：學生可以看到 AI agent 如何從「回答問題」進化成「連接外部工具產出媒體資產」。

Caveats

  • 本篇未取得 Reel 完整逐字稿；只保存公開 metadata、縮圖 OCR 與官方來源查核。
  • HeyGen 生成 avatar video 涉及肖像權、聲音授權、素材來源、商標與廣告揭露；不可只因「no camera」就忽略合規。
  • Claude Code/Codex 使用 HeyGen Skills 或 Direct API 時可能需要 API key，會消耗 HeyGen API 額度；MCP 則可能走 OAuth 授權，兩者安全模型不同。
  • 若讓 agent 自動研究並生成影片，事實查核與引用來源要明確保存，避免把未驗證內容變成看似專業的影片。
  • 真正量產前應建立模板、審稿、版本控制、成本上限、失敗重試與發布前人工 gate。

Why it matters

「Turn your expertise into video」背後其實是知識工作自動化的下一步：不是把每個人都變成剪輯師，而是讓 agent 讀懂你的專業內容，轉成可視化、可發布、可複用的媒體輸出。這對欒帥的 AI 課程、知識庫、企業培訓與內容生產都很有價值；關鍵在於把它設計成可審核的 pipeline，而不是只追求一鍵生成。
