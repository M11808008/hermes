重點摘要：

作者原本對「Hermes Agent + 本地 LLM」沒什麼興趣，原因有兩個：

  • Agent 權限太大，不太敢放心把電腦交給它操作。
  • 覺得本地 LLM 只是玩具，不夠可靠。

後來因為一直看到 Hermes Agent 的內容，決定實測：

設備與模型：

  • 退役 MacBook Pro
  • 外接／搭配魔改 22GB VRAM 的 RTX 2080 Ti
  • 模型：Qwen3.6 27B Q4 量化
  • 推論框架：llama.cpp
  • Context：55,000 tokens
  • 速度：約 20 tokens/s

實測任務：

他在 Discord 上很模糊地要求 Agent 去 Grafana 查 logs。

結果 Hermes Agent 能夠：

  • 像人類一樣使用瀏覽器
  • 登入 Grafana
  • 找 Dashboard
  • 找 Data Sources
  • 推理出 logs 存在 Loki
  • 把 logs 抓出來分析

作者結論：

雖然速度不快，但任務可以慢慢完成，而且整個過程都在內網完成，讓他非常驚豔。最後提到想進一步研究 Harness engineering。

這篇可用的價值很高，尤其適合整理成：

  • Hermes Agent 本地部署案例
  • Local LLM + Agentic Browser Use 實證
  • 企業內網／私有環境 AI Agent 應用案例
  • Grafana / Loki log investigation workflow
  • 「從不信任到驚豔」的轉換型見證文案

如果之後你要做 Hermes Agent 的案例頁，我會建議把它轉成這種標題：

「從懷疑到驚豔：本地 LLM 驅動 Hermes Agent 在內網完成 Grafana Logs 調查」

核心賣點可以抓：

「不出內網、不依賴雲端模型、可操作瀏覽器、可推理資料來源、可完成 DevOps 調查任務。」
