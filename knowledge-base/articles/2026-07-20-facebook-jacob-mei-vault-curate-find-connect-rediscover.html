Vault Curate 1.3.1：find → connect → rediscover 的 Obsidian 本地第二大腦循環

來源是梅驊的 Facebook 更新貼文。貼文表示他做的 Obsidian plugin `Vault Curate` 在 Obsidian Plugin 社群超過 2,500 人次下載，最新定位很明確：`find → connect → rediscover`。本篇延續 2026-07-19 的 Vault Curate 1.2.2 歸檔，補上 1.3.1 之後的語意路徑、Hot/Cold rediscovery 與關係圖擴張脈絡。

一句話摘要

Vault Curate 正從「中文友善本地語意搜尋」擴展成 Obsidian 的第二大腦循環：先用 BM25 + on-device embeddings + fuzzy title 找到筆記，再把未手動連結但語意接近的筆記畫成 Canvas 關係圖，最後把冷掉但相關的舊筆記重新浮出來，讓人不用一直重跑 LLM 也能發現知識庫內的隱形連結。

貼文更新重點

  • 社群採用：Obsidian Plugin 社群超過 2,500 人次下載使用。
  • 新增關聯與語意相近連結：強化文章與文章之間的關聯性。
  • Semantic paths：找出兩篇筆記之間關聯最強的連結鏈；關聯不夠強會誠實告訴你「沒連上」。
  • Canvas 互動：可從關聯圖中的筆記繼續發掘其他筆記，支援右鍵就地擴張。
  • Find：BM25 + 裝置端 vector embedding + 模糊標題比對，以 RRF 融合；內建 `bge-small-zh` 中文模型、WebGPU 加速、可離線跑。
  • Connect：以任一筆記為中心生成關係圖，紫色邊標出語意相近但未連結的筆記。
  • Rediscover：依連結數與最近是否碰過分 Hot / Cold，Discover 面板主動浮出可能被忘記的 Cold 筆記。
  • AI 整理：description / MOC 分群預設關閉，要手動開才使用 LLM；貼文強調不蒐集使用紀錄。

和 1.2.2 歸檔的差異

先前 1.2.2 歸檔重點是：繁中 embedding 正規化、WebGPU、本地索引與可編輯 Canvas 關聯圖。這次更新更像是把功能串成完整 loop：

  • Find：搜尋與相似筆記尋找。
  • Connect：關係圖、語意相近但未連結的紫色邊、semantic path。
  • Rediscover：Hot / Cold 機制，把舊但相關的筆記浮出來。

這個方向很符合欒帥的 AI 學習雷達：不是只問 LLM「幫我整理」，而是先用本地索引與圖結構讓人重新看見自己的材料。

對欒帥的實務價值

  • 私密課程資料、研究筆記、長期素材庫可先留在本機，降低全文送雲端 LLM 的需求。
  • 對繁中內容友善，特別適合台灣課程、中文研究、社群貼文、逐字稿與個人筆記。
  • Canvas 關係圖可用來做課程模組、主題地圖、舊文章再利用與知識庫重構。
  • Semantic paths 可當「兩個概念之間如何連起來」的教學工具，很適合展示 AI 輔助思考不是直接代替思考。
  • Hot / Cold rediscovery 對長期知識庫特別重要：讓冷掉的好材料重新浮現，而不是只追最新資料。

Caveats

  • 下載數來自作者 Facebook 貼文，本文未從 Obsidian 官方 API 獨立驗證下載統計。
  • WebGPU 效能取決於 Obsidian Desktop、OS、GPU/driver；不支援時可能退回 WASM。
  • 外掛雖 local-first，但仍會讀取 vault 建索引；私密 vault 應先備份並只安裝信任來源。
  • AI 整理功能雖預設關閉，但一旦切到 Ollama/OpenAI-compatible remote API，仍要重新評估資料流向。
  • Semantic similarity 代表「可能相關」，不等於因果或結論；紫色邊與 semantic paths 應視為人工判斷候選。
