iLab GPT Conjure｜GPT-image-2 本地 AI 生圖 WebUI 與 Codex Responses 接入

來源為 Govin.AI 在 Threads 宣布開源一個基於 GPT-image-2 的 AI 圖片生成 WebUI。作者表示從四月 Codex 開放 image2 生圖後開始製作，這是第一個比較完整的系統專案；目前正式開源，支援 Codex Responses 與 OpenAI 相容 API 接入，歡迎使用與 PR。

一句話摘要

iLab GPT Conjure 是把 GPT-image-2 生圖、參考圖、圖片編輯、提示詞模板、多任務佇列、公用圖庫與歷史庫包成一個本地 WebUI / CLI 的開源工作台；它對欒帥的價值在於可作為「像 ComfyUI 一樣可視化管理 GPT-image 生產流程」的輕量本機工具，而不只是單次 API 呼叫範例。

安裝與啟動

一般安裝：

```bash

git clone https://github.com/kadevin/ilab-gpt-conjure.git

cd ilab-gpt-conjure

python3 -m venv .venv

.venv/bin/python -m pip install -r requirements-webui.txt

.venv/bin/python -m uvicorn codex-image.webui.app:app --host 127.0.0.1 --port 8787 --no-access-log

```

啟動後瀏覽：

```text

http://127.0.0.1:8787/

```

Portable 方式：下載 v0.3.7 對應平台 zip，解壓到一般使用者目錄，Windows 雙擊 `Start WebUI Portable.bat`，macOS 雙擊 `Start WebUI Portable.command`。macOS 包未簽名，若被 Gatekeeper 攔截，可用右鍵 Open 或移除 quarantine：

```bash

xattr -dr com.apple.quarantine /path/to/ilab-gpt-conjure-macos-portable-arm64

```

認證與安全判斷

| 模式 | 適合情境 | 風險與注意 |

|---|---|---|

| OpenAI-compatible API | 穩定集成、團隊使用、共享工作站、可能公開提供服務 | 需管理 API key、Base URL、模型名與成本 |

| Codex / ChatGPT OAuth | 個人本機工作流、實驗性使用 | README 明確說這不是 OpenAI 官方推薦 API 集成方式；內部接口可能變更、失效，受帳號/產品/用量規則影響 |

README 特別提醒不要提交 OAuth 檔案、API key、本地輸入圖、生成結果、任務 metadata、SQLite 資料庫或 debug logs。

對欒帥的實用價值

  • 可作為 GPT-image 系列的本地生產工作台，適合保存角色、品牌素材、模板與歷史任務。
  • 比單次 ChatGPT 生圖更適合批次、多參考圖、多版本比較與素材庫管理。
  • 可成為 AI 課程中的「從 API 到產品化 WebUI」案例：FastAPI、CLI、portable release、SQLite 歷史庫、更新器與授權都能拆成教學點。
  • 若要在團隊或公開服務使用，要優先走 OpenAI-compatible API 模式，不要依賴個人 ChatGPT OAuth。
  • AGPLv3 代表若修改後透過網路向使用者提供服務，需要遵守開源義務；這點在商用或內部部署前要先確認。
