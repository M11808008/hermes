可重用的三段式簡報 Agent 流程

1. 先產大綱，不直接做整份簡報

第一步應該讓 Agent 只輸出：

  • 目標受眾；
  • 簡報目的；
  • 頁數範圍；
  • 每頁標題；
  • 每頁重點；
  • 建議圖表或視覺素材；
  • 需要使用者補資料的地方。

這一步的價值是把內容結構先鎖住，避免 Agent 一開始就浪費時間做出風格漂亮但邏輯錯誤的 PPT。

2. 先做一張樣張，確認視覺與資訊密度

Reel 提到的「先生成一張給你看」非常重要。簡報最容易失控的地方通常不是內容，而是：

  • 字太多；
  • 主視覺不符合品牌；
  • 表格與圖表不可讀；
  • 每頁資訊密度不一致；
  • AI 生成的版面有模板感；
  • 中文斷行、字重、留白不佳。

先驗收一張樣張，就能在還沒做完整份之前調整版型、字級、色系、圖文比例與敘事節奏。

3. 最後才批次製作完整檔案

大綱與樣張都通過後，再讓 Agent 產出整份簡報。這時應要求它保留：

  • 章節一致性；
  • 標題層級；
  • CTA 與結論頁；
  • 講者備註；
  • 可修改的文字與圖表，而不是整頁截圖；
  • 版本記錄與修改摘要。

和 OpenAI Codex Skills 的關係

OpenAI Developers 的 Codex Skills 文件指出，Skill 是一個含 `SKILL.md` 的資料夾，可搭配 scripts、references、assets、templates 與 agent 設定；Codex 會先看到 skill 的名稱與描述，當任務符合描述時才載入完整 `SKILL.md`。使用者也可以在 CLI / IDE 中透過 `/skills` 或 `$` 明確呼叫 skill。

這代表「簡報 PPT Skill」不應只是單一 prompt，而可以做成可維護的工作流包：

  • `SKILL.md`：定義何時使用簡報流程、產出格式、審核關卡。
  • `templates/`：保存課程簡報、提案簡報、報告簡報版型。
  • `references/`：保存品牌規範、課程定位、受眾設定、常用案例。
  • `scripts/`：可選擇用 Python / Node 產生 PPTX、匯出圖片或檢查頁數。
  • `assets/`：保存 logo、色票、圖示、範例封面。

給欒帥的實作價值

1. AI 課程與工作坊簡報

很適合把「AI 學習雷達」文章、課程大綱、工作坊活動說明轉成簡報初稿。流程可以是：

  • 從知識庫文章整理課程主軸。
  • 產出 10–15 頁簡報大綱。
  • 先做封面或一頁核心概念樣張。
  • 確認視覺語言後再生成整份。
  • 人工審稿後輸出 PPTX / PDF。

2. 避免新手被 Agent 一次帶偏

對新手來說，一次要求 AI「幫我做一份簡報」通常會得到看似完整但難修改的成果。這個 skill 的價值在於把任務拆成「大綱審核 → 樣張審核 → 完整生成」，讓使用者在關鍵節點介入，而不是最後才發現整份都要重做。

3. 可變成 Hermes / Codex 共用的 Skill

同一套流程可以在 Hermes Agent 與 Codex 之間共用概念：Hermes 負責擷取來源、整理知識庫與產出內容架構；Codex 或本機 Python 工具負責生成 PPTX、版面檢查與檔案輸出。

建議的 Prompt 骨架

你是簡報製作 Agent。請不要一開始製作完整簡報。先完成三階段流程：

  • 先詢問或推定簡報目的、受眾、頁數、語氣與輸出格式。
  • 產出完整大綱，等待我確認。
  • 大綱確認後，只先製作第一張代表性樣張，說明版型、色系、資訊密度與可修改元素。
  • 樣張確認後，才製作完整簡報。
  • 完成後提供修改摘要、檔案路徑、可再調整項目與人工審查清單。

檢查清單

  • 大綱是否符合受眾與場合？
  • 每頁是否只有一個主要訊息？
  • 是否先做樣張，而不是直接批次做完整份？
  • 中文字級、留白、對齊與圖文比例是否可讀？
  • 表格、流程圖、數據圖是否能在投影或手機上閱讀？
  • 是否保留可編輯文字，而不是把文字壓成圖片？
  • 是否有講者備註、結論頁與下一步行動？
