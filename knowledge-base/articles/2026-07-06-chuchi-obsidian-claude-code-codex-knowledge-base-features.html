朱騏｜Obsidian 搭配 Claude Code / Codex 建立個人知識庫：10 個容易被低估的功能

一句話摘要

這則貼文值得歸檔，因為它把「Obsidian + Claude Code / Codex」從單純存 Markdown，提升到可被 AI agent 使用的完整知識作業系統：每日入口、資料庫、白板、版本救援、拆分、查詢、圖解與檔案管理都會影響 AI 能否快速找到正確上下文。

貼文重點

朱騏在 Facebook 公開貼文中提到，最近常看到很多人用 Obsidian 搭配 Claude Code / Codex 建立個人知識庫，但多數用法只侷限在 Obsidian 的一兩種功能，十分可惜。作者自 2022 年開始使用 Obsidian，認為它內建與社群外掛有很多實用能力，特別推薦 10 個功能。

10 個功能整理

| 功能 | 貼文定位 | 對 AI 個人知識庫的意義 |

|---|---|---|

| Daily Note | 每天寫日記，記錄發生的事情 | 作為每日入口，保存任務、事件、想法與反思，讓 agent 有時間序列脈絡。 |

| Base | 像 Notion 一樣建立資料庫，但資料範圍可擴及整個資料夾 | 把分散 Markdown 變成資料庫視角，適合整理專案、課程、文章、人物與任務。 |

| Canvas | 白板功能，用鳥瞰視角組織筆記 | 將概念、專案與知識節點視覺化，適合課程架構、系統圖、研究地圖。 |

| File Recovery | 找回被覆蓋、誤刪或寫壞的筆記版本 | AI agent 寫檔時非常重要，可降低誤改知識庫的風險。 |

| Note Composer | 將臃腫筆記拆成新筆記，並保留內部連結 | 幫助長筆記拆分成可檢索、可引用的小單元，避免單檔過長。 |

| Outline | 顯示 H1-H6 標題階層 | 長文件、書籍、課程稿、SOP 需要快速導航。 |

| Calendar | 以日曆介面管理 Daily Note | 讓每日紀錄可以被時間化回顧，連結週報、月報與工作節奏。 |

| Dataview | 用類似 SQL 的查詢語言檢索筆記並列成表格或清單 | 讓 Markdown 變成可查詢資料庫，適合 task、status、tag、frontmatter 管理。 |

| Excalidraw | Obsidian 專屬圖解工具 | 讓知識不只文字化，也能用手繪風圖解呈現架構、流程與關係。 |

| File Explorer Note Count | 在資料夾旁顯示檔案數 | 簡單但實用，幫助管理資料夾規模與辨識知識庫膨脹區域。 |

官方與外部來源交叉確認

Brave Search 可確認多數功能有官方或社群來源：

  • Obsidian 官方 Help 有 Daily notes、Canvas、File recovery、Note composer、Outline、File explorer 等核心外掛文件。
  • Dataview 是社群廣泛使用的 Obsidian 查詢外掛，可透過 GitHub repo 了解語法與用途。
  • Excalidraw plugin 是成熟社群外掛，可在 Obsidian 中編輯與檢視 Excalidraw 圖。
  • File Explorer Note Count 是社群外掛，用於在檔案總管中顯示資料夾下元素數量。

為什麼這對 Claude Code / Codex 很重要

很多人把 Obsidian 當成「AI 可讀的 Markdown 資料夾」，但這只是最低層。真正適合 agent 的知識庫要同時具備：

  • 可定位：README、INDEX、檔名、Outline 讓 AI 快速找到檔案。
  • 可分解：Note Composer 讓長內容拆成可引用單元。
  • 可查詢：Dataview / Bases 讓狀態、標籤、frontmatter 可被列表化。
  • 可回復：File Recovery / Git 讓 AI 誤寫後能救回。
  • 可視覺化：Canvas / Excalidraw 讓複雜系統變成圖。
  • 可時間化：Daily Note / Calendar 讓反思與決策有時間序列。

這與前面歸檔的 Vault for Founders 可以互補：Vault for Founders 提供創辦人 AI 知識庫架構；這則貼文補上 Obsidian 內部功能層，讓使用者不只知道要建 Vault，也知道如何在日常維護它。

可轉成 Luarn-Pin 工作流的地方

1. AI 學習雷達的私密工作區可用 Daily Note 當入口

公開知識庫保存文章與連結，但私密工作流可以每天有一則 Daily Note，記錄：

  • 今日分享的連結批次。
  • 哪些已歸檔、哪些待驗證。
  • 重要決策與卡點。
  • 值得升級成 skill 的流程。

2. Dataview / Bases 可管理課程與專案資料

若每篇課程筆記、學生名單處理紀錄、工具評測都有 frontmatter，就能用 Dataview / Bases 產生：

  • 課程素材清單。
  • AI 工具評測表。
  • 已部署 / 未部署文章。
  • 私密 / 公開資料邊界。
  • 待更新 SOP。

3. File Recovery + Git 是 AI 寫檔的安全網

讓 Claude Code / Codex 操作 Obsidian vault 時，應同時啟用 File Recovery 與 Git。File Recovery 提供 Obsidian 層的筆記版本，Git 提供 repo 層的差異與回復。兩者合用，能降低 agent 誤改、誤刪或格式化錯誤的風險。

4. Canvas / Excalidraw 適合教學與系統設計

對高質感課程設計，Canvas 可做概念地圖，Excalidraw 可做手繪風流程圖。它們很適合用來整理：

  • AI agent 工作流。
  • 知識庫架構。
  • 課程章節地圖。
  • Hermes / Claude / Codex / Obsidian 的資料流。

風險與判讀限制

  • Obsidian 功能很多，但不應一開始全部導入；過多外掛會增加學習與維護成本。
  • Dataview / Bases 需要一致的 frontmatter 與命名規則，否則資料庫視角會失準。
  • File Recovery 不是備份替代品；仍需 Git / GitHub 或其他備份策略。
  • Excalidraw 與 Canvas 很適合思考，但也可能變成整理癖；真正要服務的是可檢索、可行動的知識。
  • AI agent 讀寫 Obsidian 時需有權限邊界，不應把私密資料自動同步到公開知識庫。

後續可追蹤

  • Obsidian Bases 的成熟度與它和 Dataview 的分工。
  • Claude Code / Codex 是否能更好地讀取 Canvas、Excalidraw 或 Dataview 查詢結果。
  • 如何把 Obsidian vault 與 Hermes skills / memory / knowledge-base 分層整合。
  • 是否需要為 Luarn-Pin 建立一份「Obsidian + AI agent 私密知識庫啟動模板」。
