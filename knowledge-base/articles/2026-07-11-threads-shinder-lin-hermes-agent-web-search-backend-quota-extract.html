Shinder Lin Threads｜Hermes Agent Web Search 後端選擇：搜尋、全文擷取與額度管理

一句話摘要

Hermes Agent 的網路工具不要只看「能不能搜尋」；更關鍵的是把 search、extract、額度、隱私與自架成本分開設計。

Threads 貼文重點

Shinder Lin 的貼文提醒：

  • Hermes Agent 的 web search 若走 Nous Portal / Firecrawl 類代管服務，免費層額度可能很快撞牆。
  • web backend 可以切換，包含 DDGS、SearXNG、Tavily、Exa、Brave 等選項。
  • DDGS、SearXNG、Brave 偏「搜尋」；需要讀完整網頁內容時，仍要搭配 Tavily、Exa 或 Firecrawl 類 extract 能力。
  • 作者的建議組合是：搜尋走免費/低成本後端，少數需要讀全文時才吃 Tavily 額度。
  • 可改 `~/.hermes/config.yaml` 的 `web.backend`，或跑 `hermes tools` 精靈設定。
  • 作者回覆補充另一個解法：自架 Firecrawl。

官方文件校準

Hermes Agent 官方文件與 Brave Search 摘要可驗證：

  • Hermes Web Search & Extract 文件顯示可在 `~/.hermes/config.yaml` 設定：

- `web.backend: "searxng"`

- 可選後端包含 `firecrawl | searxng | brave-free | ddgs | tavily | exa | parallel | xai`

  • Hermes Integrations 文件指出：如果 `web.backend` 未設定，會依可用 API key 自動偵測；也支援 self-hosted Firecrawl via `FIRECRAWL_API_URL`。
  • Hermes Web Search Provider Plugins 文件指出，Firecrawl、SearXNG、Tavily、Exa、Parallel、Brave Search、xAI、DDGS 都是內建 web provider plugins。
  • Firecrawl GitHub repo 描述為「The API to search, scrape, and interact with the web at scale」；GitHub API 查詢時約 149,239 stars。

更精準的後端分類

可把 Hermes web 後端分成四類：

1. 免金鑰 / 低摩擦搜尋

  • DDGS：適合低成本搜尋與快速查詢。
  • SearXNG：適合自架、隱私與可控性。

適合：大量搜尋、初步找來源、低成本探索。

限制：不一定能穩定抽取整篇文章內容。

2. Search API 型

  • Brave Search：適合穩定搜尋、查官方頁、產品頁、新聞與文件入口。

適合：需要較好的搜尋品質與可預期結果。

限制：主要是搜尋，不等於全文擷取器。

3. Search + Extract / Content API 型

  • Tavily、Exa：適合搜尋同時抓內容，常用於 agent research。

適合：需要較少工具拼接、直接拿可摘要內容。

限制：免費額度有限，需管理用量。

4. Crawling / Scraping 型

  • Firecrawl：適合抓取網頁、爬站、把網頁轉成 LLM 可用內容。
  • Self-hosted Firecrawl：可控性更高，但需要部署與維護。

適合：需要大量 extract、反覆抓網站、內部流程穩定化。

限制：代管額度、自架成本、反爬限制與站點規範都要考慮。

建議配置策略

貼文中的策略可以整理成一個實務框架：

  • 預設搜尋：用低成本搜尋後端，例如 DDGS、SearXNG 或 Brave。
  • 需要讀全文：再切到 Tavily、Exa 或 Firecrawl。
  • 需要大量爬取：考慮 self-hosted Firecrawl。
  • 需要隱私：優先 SearXNG 自架或內部可控方案。
  • 需要高品質搜尋：Brave Search 是較穩健的通用選項。

對欒帥目前工作流的提醒

欒帥偏好 Brave Search 優先，這個偏好仍合理：Brave 適合找官方來源、產品頁與新聞入口。真正要避免的是把「搜尋」誤當「已讀完整網頁」。

AI 學習雷達歸檔流程可維持：

  • 搜尋與驗證：Brave Search。
  • 社群頁、YouTube、GitHub：用 DOM / oEmbed / GitHub API / transcript。
  • 反爬或長文章：必要時再用 Jina Reader、Tavily、Exa 或 Firecrawl 類 extract。

風險與注意事項

  • 免費額度數字容易變動，任何「每月 X 次」都應以各服務官方頁或帳號 dashboard 為準。
  • 自架 Firecrawl 能提高可控性，但不等於可以無視網站 robots、授權與反爬限制。
  • DDGS / SearXNG / Brave 的搜尋結果不保證等於完整正文，需要 extract 時要另設流程。
  • Hermes 工具設定改完後，通常需要新 session 或 gateway restart 才能在實際 agent 中生效。
