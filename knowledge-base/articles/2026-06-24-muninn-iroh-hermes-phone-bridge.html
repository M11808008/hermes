Muninn + iroh｜Hermes Agent 手機直連橋接方案

Thor Lin 在 Generative AI 技術交流中心分享了 Muninn：一個專為 Hermes Agent 設計的 iOS app，搭配開源的 `hermes-bridge`，讓手機可以透過 iroh P2P 加密通道連回家中或本機的 Hermes。重點是：不必設定 LINE Bot、Telegram Bot、ngrok tunnel 或 webhook，安裝後掃 QR 即可配對。

一句話摘要

Muninn 的價值不是「多一個聊天 app」，而是把 Hermes 從桌面工具變成可隨身操作的個人 agent 介面；它用 iroh 解掉手機連回家中 agent 的網路摩擦，並把 session、cron、kanban、檔案、語音與媒體回傳都納入原生行動介面。

安裝流程

作者建議的低摩擦流程如下：

  • 先安裝並啟動 Hermes Agent。
  • 透過 TestFlight 安裝 Muninn：`https://testflight.apple.com/join/8mcRtXsm`。
  • 對 Hermes 說：

> 把 https://github.com/coolthor/hermes-bridge clone 到 ~/hermes-bridge，跑它的 install.sh，然後顯示配對 QR 讓我連手機。

  • 打開 Muninn，掃描 QR code，輸入手機端顯示的確認碼完成配對。

`hermes-bridge` README 補充：installer 會 clone repo、下載對應平台的 bridge binary（無需先安裝 Rust）、安裝 `connect-phone` skill，並顯示 QR。日後可對 Hermes 說「連接手機」或 `connect my phone` 重新顯示 QR。

Muninn 目前支援能力

| 能力 | 意義 |

|---|---|

| 傳圖 / 傳檔 | 手機相簿與文件可直接送給 agent；模型需具備視覺能力才可理解圖片 |

| 語音輸入 | 透過 Hermes 的語音轉文字能力，把手機語音送成文字任務 |

| 收回媒體 | agent 生成或抓到的圖片、影片、音訊、文件可回傳手機預覽 |

| 繁體中文介面 | 對台灣使用者與教學情境更友善 |

技術架構判讀

`hermes-bridge` 是一支跑在 Hermes 旁邊的 Rust 常駐程式，把本機 Hermes dashboard 經由 iroh 通道開放給已配對手機。README 描述它使用多個 iroh ALPN：

| ALPN | 用途 |

|---|---|

| `hermes-bridge/0` | 透明轉送本機 Hermes dashboard WebSocket |

| `hermes-bridge-upload/0` | 手機上傳圖片、語音或文件給 agent |

| `hermes-bridge-download/0` | agent 回傳圖片、影片、音訊或文件給手機 |

核心設計是「手機 ↔ 家中電腦」之間的 P2P 加密通道。iroh 會先透過公共 relay 協助兩端找到彼此，並嘗試 NAT traversal / hole punching；若網路環境支援（例如 5G IPv6），可升級為 direct 直連。若打洞失敗，仍可經 relay 通訊，但多一跳且可能較慢。

與 LINE / Telegram Bot 的差異

| 面向 | Muninn + iroh | LINE / Telegram Bot |

|---|---|---|

| 設定門檻 | 掃 QR 配對，不需 webhook / tunnel | 需 bot token、public URL、webhook 或 gateway 設定 |

| 介面 | 為 Hermes 設計的原生 agent 介面 | 通訊軟體聊天室介面 |

| 資訊可視化 | 可看到 session、cron、kanban、多 agent | 通常只看到訊息流 |

| 路由 | 兩端 P2P，失敗時 relay | 訊息必經平台伺服器 |

| 檔案與媒體 | 原生設計支援上傳與下載通道 | 依平台 adapter、檔案類型與 webhook 限制而定 |

| 風險 | 配對手機等同取得 agent 操作權 | bot token / webhook / platform 權限風險 |

安全模型與風險

Muninn 的配對 QR 被設計為一次性「介紹信」，不是永久鑰匙。手機掃描後，仍需要使用者輸入確認碼，手機的 NodeId 才會被加入 bridge 白名單。陌生裝置即使拿到 QR，也因 NodeId 不同、未被核准而無法連入。

但這裡的風險不可低估：配對一台手機，就等於讓該手機可以操作 Hermes；而 Hermes 具備工具、檔案與 shell 執行能力。因此只應配對自己的裝置，舊手機或不再使用的裝置要撤銷白名單。

移除方式包括：

  • 在 Muninn app 右上角齒輪中解除配對，並通知電腦端移除白名單。
  • 在電腦端列出與撤銷裝置：`scripts/run-bridge.sh list` / `scripts/run-bridge.sh revoke`。
  • 若整個 bridge 不再使用，停止 bridge，刪除 `~/hermes-bridge` 與 `~/.hermes-bridge`。

對欒帥的實用判讀

1. 這是 Hermes 行動入口的重要補位

欒帥目前已透過 LINE 與 Hermes 互動，但 LINE adapter 的限制包括 webhook、公網 URL、reply token、檔案型別判斷與慢回覆按鈕。Muninn 若穩定，會是更接近「個人 AI 助理 app」的體驗：尤其是 session、cron、kanban 可視化，對高自治 agent 工作流比一般聊天平台更自然。

2. 可作為 AI 課程或工作坊的進階展示

Muninn 很適合放在「AI Agent 從桌面走向隨身助理」單元：

  • 入門層：Hermes Desktop 本機對話。
  • 通訊層：LINE / Telegram bot 讓手機可發訊息。
  • 原生層：Muninn + iroh 讓手機成為 agent 控制面板。

這能幫學員理解 agent 不只是聊天模型，而是具備權限、任務、排程與檔案通道的個人運算介面。

3. 目前仍屬早期測試，宜先做沙盒驗證

作者明確提到目前主要由他與 Claude 開發，功能仍在早期階段。建議正式採用前先以沙盒機測試：

  • bridge 是否能在目前 macOS / Hermes 版本穩定安裝。
  • 配對、重連、撤銷是否可靠。
  • 在 5G、Wi-Fi、公司網路、NAT 環境下 direct / relay 的連線表現。
  • 圖片、文件、語音與大檔案回傳是否符合需求。
  • allow / not allow 的危險操作核准是否能清楚呈現。
  • 若手機遺失，撤銷裝置與刪除金鑰流程是否足夠低摩擦。
