Agnes AI｜免費 Omni-modal API 與 Hermes Agent 串接觀察

電腦王阿達在 2026-06-23 介紹 Agnes AI：一個來自新加坡 / Sapiens AI 生態的 AI API 平台，主打可免費透過 API 呼叫文字、圖片與影片模型，且目前文章實測指出免綁信用卡、暫無明確使用額度限制。這對想測試 OpenClaw、Hermes Agent 或其他 agent runtime 的使用者很有吸引力，因為多數可用於 agent 的 API 不是付費，就是免費層速度與額度非常有限。

一句話摘要

Agnes AI 值得收進雷達的重點不是「免費」本身，而是它把文字 agent、圖片生成、圖片編輯與影片生成都放進同一個 OpenAI-style API 生態；適合作為低成本實驗入口，但不應直接假設能承擔正式產品或敏感資料工作流。

可用模型整理

| 類型 | 模型 | 適用場景 |

|---|---|---|

| 文字 / Agent | `agnes-2.0-flash` / Agnes-2.0-Flash | AI agent、tool calling、coding、一般聊天、多輪任務 |

| 圖片 | Agnes-Image-2.0-Flash、Agnes-Image-2.1-Flash | 圖片生成、圖片編輯、image-to-image |

| 影片 | Agnes-Video-2.0 / Agnes Video V2.0 | 圖生影片或影片生成流程 |

文章提到官方榜單宣稱：Agnes-2.0-Flash 在 ClawEval 通用排行榜分數高於 GPT 5.4；圖片模型在 AA 圖片編輯排行榜接近 Nano Banana；影片模型在 AA 圖生影片排行榜僅略低於 Seedance 1.5 Pro。這些資訊適合做趨勢觀察，但正式採用前仍應自行以固定任務集驗證。

Hermes Agent 串接判讀

文章示範將 Agnes AI 接到 Hermes Agent：

  • 註冊 Agnes AI 帳號。
  • 在 API 金鑰頁面建立新的 API Key。
  • 將 provider 設為 custom / OpenAI-compatible。
  • Base URL 使用 `https://apihub.agnes-ai.com/v1`。
  • Model Name 使用 `agnes-2.0-flash` 或官方文件列出的模型名稱。

對 Hermes 類 runtime 來說，Agnes AI 的意義在於可快速測試 agent 工作流：讓模型使用工具、整理網頁資料、輸出 CSV、生成圖片或觸發影片模型。文章實測 `Agnes-2.0-Flash` 可完成「到電腦王阿達網站整理當日新文章並在桌面儲存 CSV」這類工具型任務。

實測現象

| 測試 | 文章觀察 |

|---|---|

| 文字 agent 任務 | 可完成網站資料整理並輸出 CSV |

| 圖片生成 | 成功，品質可接受，約 1 分鐘內完成 |

| 影片生成 | 成功，約 2 分鐘左右 |

| 影片查詢 / 生成錯誤 | 使用 Agnes-2.0-Flash 要求生成或查詢已生成影片時曾出現「無效的令牌」錯誤；改用 Gemini 3.1 Flash Lite 可解 |

| 尖峰時段 | 非尖峰時刻速度較可接受 |

| 文件頁 | 有時顯示系統繁忙或錯誤，需重新整理 |

對欒帥的實用判讀

1. 可作為課程裡的「零成本 Agent API」案例

如果要教學生或工作坊學員體驗 Hermes / OpenClaw 類 agent runtime，Agnes AI 這類免信用卡 API 是很好的上手案例。它降低了三個門檻：

  • 不必先處理 OpenAI / Anthropic 付費帳單。
  • 不必架本地模型。
  • 同一平台可示範文字、圖片、影片 API 的差異。

但教學時要把它定位成「實驗與入門」，不要包裝成正式產品級 SLA。

2. 適合先做工具鏈 smoke test，不適合直接承接核心任務

建議測試順序：

  • 一般對話與中文回答品質。
  • 工具呼叫與 JSON / structured output 穩定性。
  • 檔案讀寫、網頁整理、CSV 輸出等 agent 任務。
  • 圖片生成與編輯任務。
  • 影片生成與狀態查詢。
  • 高併發或長任務延遲觀察。

若要接進正式工作流，至少要補上 retry、fallback model、timeout、錯誤訊息收斂與資料外送風險控管。

3. 免費 API 的關鍵不是省錢，而是驗證成本降低

免費 API 最有價值的地方，是讓團隊能快速比較 agent runtime、prompt pattern、tool permission、workflow design，而不是一開始就被 token 成本限制。對欒帥的 AI 學習雷達來說，它可放在「低摩擦實驗基礎設施」分類。

導入前檢查清單

| 檢查項 | 為什麼重要 |

|---|---|

| 免費政策是否穩定 | 免費無限制可能是早期推廣，未來可能改為額度制或排隊制 |

| API 文件可用性 | 文件若常出錯，團隊導入與除錯成本會提高 |

| OpenAI compatibility 完整度 | 需測試 streaming、tool calling、image/video endpoint、錯誤格式 |

| 資料保留與隱私條款 | 不應把個資、客戶資料、內部文件直接送入未評估服務 |

| 影片任務錯誤處理 | 文章已觀察到「無效的令牌」錯誤，需設 fallback |

| 速率限制與尖峰延遲 | 免費服務在多人使用時可能不穩 |

| Benchmark 可重現性 | 官方榜單需用自己的任務集驗證，不宜直接採信 |

建議後續測試

| 測試 | 做法 | 觀察點 |

|---|---|---|

| Hermes Agent 基本串接 | custom provider + `https://apihub.agnes-ai.com/v1` + `agnes-2.0-flash` | 是否能穩定出現在模型選單並完成一般對話 |

| Tool calling | 讓模型整理本機資料或網頁資料 | 函式呼叫格式、錯誤修復能力、是否亂編檔案 |

| Structured output | 要求固定 JSON schema | 格式遵循率與重試後成功率 |

| 圖片生成 | 同 prompt 與其他模型比較 | 品質、延遲、失敗率 |

| 影片生成 | 測試生成與查詢狀態 | token error、任務 ID、輪詢流程是否穩定 |

| 免費額度壓力 | 多次小任務連續呼叫 | 是否限速、排隊或回傳非標準錯誤 |
