Qwen3.5-35B + Framework Desktop 128G｜本機模型協作完成簡單網站的實測觀察

來源為林熙哲的 Facebook 圖文貼文。貼文分享在 Framework Desktop 128G 上跑 Qwen 3.5-35B 模型，實測「效果竟然這麼好」，雖然速度稍慢、偶爾會犯笨，但已經可以協作完成一個簡單網站。圖片中可見 demo 網站主題為 skipper.tw，像是台灣帆船活動與課程資訊公告平台。

一句話摘要

這則貼文的價值不在於「十萬元主機一定划算」這句感想，而在於它捕捉到一個 local-first AI coding 的轉折：128GB unified memory 等級的小型桌面機，已經足以讓 35B 級 MoE 模型在本機參與簡單網站開發；但速度、量化、上下文、工具鏈與工程審查仍然決定它能不能變成穩定工作流。

為什麼 128GB unified memory 重要

一般消費級 GPU 的瓶頸常常不是算力，而是 VRAM。35B、70B、120B 以上模型即使量化後，也需要足夠記憶體容納權重、KV cache、上下文與 runtime overhead。Framework Desktop / Ryzen AI Max 這類 128GB unified memory 設計，把 CPU/GPU 可用記憶體集中在同一池，讓「本機跑大模型」比傳統只有 8–24GB VRAM 的桌機更可行。

但 unified memory 不等於無限制：

  • 頻寬通常仍低於高階資料中心 GPU。
  • 長 context 會吃大量 KV cache。
  • MoE 模型雖然 activated parameters 較少，但總權重仍需放進記憶體。
  • coding agent 還要額外跑編輯器、瀏覽器、dev server、terminal、索引器。
  • 速度能否接受取決於量化格式、runtime、batch、context length、GPU 分配與散熱。

Local-first AI coding 的實務判讀

| 面向 | 本機 128G 主機 | 雲端 API / 訂閱 |

|---|---|---|

| 隱私 | 程式碼與 prompt 可留在本機 | 需信任供應商與資料政策 |

| 成本 | 一次性硬體支出高 | 初期低，但長期高頻使用會累積費用 |

| 速度 | 35B 可用但可能慢；大模型更慢 | 通常較快，尤其 frontier model |

| 可控性 | 可選模型、量化、runtime、離線使用 | 受 API、額度、模型版本與供應商限制 |

| 維護 | 需要自己處理模型下載、runtime、driver | 維護成本低 |

| 適合任務 | 私有程式碼、prototype、教學、長期固定工作流 | 高品質推理、快速迭代、需要最強模型時 |

對欒帥的應用價值

  • **AI 課程示範案例**：可用來討論「local AI workstation 是否值得投資」，比單純雲端工具更有判斷深度。
  • **知識庫/網站原型**：簡單網站、資料看板、課程頁、活動平台 mockup，已是 35B 本機模型可協作的合理範圍。
  • **私有資料開發**：若要處理未公開課程資料、內部名單、學員內容，本機模型能降低外洩風險。
  • **硬體採購判讀**：128G unified memory 的價值在於本機大模型，而不只是一般桌機效能。
  • **混合工作流**：可以本機模型做草稿、重構與隱私敏感任務；遇到困難推理再交給 Claude / GPT / Gemini 等雲端模型。

建議實作路線

| 階段 | 做法 |

|---|---|

| 1. 先定任務 | 不要以「跑最大模型」為目標，先定義要做網站、摘要、程式重構或資料整理 |

| 2. 選本機 runtime | LM Studio / Ollama / llama.cpp / vLLM / KTransformers 依硬體與模型格式選擇 |

| 3. 從量化版開始 | 先跑 GGUF 量化模型確認速度，再決定是否追求更高精度 |

| 4. 加 coding agent | 把本機 OpenAI-compatible endpoint 接到 Cursor、Claude Code 類工具或 Hermes custom provider |

| 5. 設計驗收 | 用瀏覽器預覽、測試、lint、git diff 驗證，不要只看模型口頭說完成 |

| 6. 混合雲端 | 本機模型卡住時，用雲端高階模型做 code review 或疑難排解 |

安全與品質檢查清單

  • **不要把本機等同安全**：本機模型仍可能讀到敏感檔案；coding agent 的檔案權限要限制在專案資料夾。
  • **Git diff 必看**：AI 改完網站後，必須檢查新增依賴、外部 CDN、API key、追蹤碼與隱私設定。
  • **不要暴露本機服務**：LM Studio / Ollama / vLLM endpoint 若開在 `0.0.0.0`，需確認防火牆與存取控制。
  • **模型幻覺仍存在**：作者也提到「偶爾會笨笨的」；本機模型不會因為私有部署就更可靠。
  • **成本要算總持有成本**：硬體、電費、散熱、維護、折舊與時間成本，都要和雲端訂閱比較。
  • **授權審查**：Qwen3.5-35B-A3B model card 標示 Apache-2.0，但實際產品化仍應檢查模型、量化版、依賴套件與素材授權。

判讀限制

  • 這是社群個人實測，不是標準 benchmark；沒有提供 tokens/sec、量化格式、runtime、context length、prompt、錯誤率等完整數據。
  • Facebook 圖片只能確認 demo 方向與貼文觀察，不能證明整個網站已達 production-ready。
  • 「值得花十萬元」是作者主觀價值判斷；對不同人而言，雲端 API、3090/4090、Mac Studio、DGX Spark 或租用 GPU 都可能更划算。
  • Qwen3.5-35B-A3B 是 35B total / 3B activated MoE；不能只用 3B activated 低估記憶體需求，也不能只用 35B total 高估推論成本。
