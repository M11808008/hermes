羅達 Rhoda｜Claude Code 指令大禮包：非碼農也能建立品質控管流程

一句話摘要

非工程師使用 Claude Code 的關鍵不是多會下 prompt，而是建立「改完 → 看 diff → code review → security review → verify」的品質控管節奏。

Facebook 可擷取資訊

  • 作者：羅達 Rhoda（AV8D.levelup）。
  • 貼文主題：免費分享給正在用 Claude Code 的非碼農朋友。
  • 核心定位：初階是讓 Claude Code 動起來，中階是讓它省時間、把關品質。
  • 指令分類：程式碼品質、確認真的做對、看清楚變更、設定與紀錄、多資料夾工作。

指令與流程重點

貼文整理的日常流程包括：

  • `/code-review`：綜合審查 bug、可讀性、安全性。
  • `/security-review`：針對 SQL injection、XSS、憑證外洩等安全問題。
  • `/simplify`：不找 bug，只做清理與簡化。
  • `/verify`：實際把程式跑起來，確認改動有效。
  • `/run`：啟動並操作 app，眼見為憑。
  • `/diff`：看清楚本次改了什麼。
  • `/effort`：調整推理強度 low / medium / high。
  • `/config`、`/cost`、`/export`、`/resume`：管理偏好、成本、匯出與歷史 session。
  • `/add-dir`、`/cd`：處理多資料夾與 monorepo。

官方交叉確認

Claude Code docs 中有 code review 相關文件，確認可在本機 Claude Code session 中執行 code review 檢查 diff。部分社群整理指令可能因 Claude Code 版本演進而變動，正式教學應以 `/help` 與官方 docs 為準。

為什麼值得 AI 學習雷達收錄

這則貼文的價值在於它把非工程師使用 AI coding tool 的心法說清楚：

  • 不要只把 Claude 當打字工。
  • 要把它當會自我檢查的工程師。
  • 每次改完都要看 diff、review、verify。
  • 成本與 session 也要管理。
  • 多資料夾專案需要明確切換工作目錄。

對欒帥的應用

  • 可直接納入 AI coding 課程的「中階操作清單」。
  • 可改寫成 Claude Code / Codex / Hermes 的共通品質控管流程。
  • 可作為非工程師導入 vibe coding 的安全邊界：能做很快，但要驗證與 review。

注意事項

Slash command 名稱與可用性會隨 Claude Code 版本更新。教學時要提醒學員先跑 `/help`，並把「檢查結果」視為輔助，不可取代人工驗收。
