Frappe ERPNext｜開源 ERP 平台與未來專案備忘

這份筆記是針對欒帥分享的 GitHub 專案 https://github.com/frappe/erpnext 建立的長期備忘。未來若我們需要做企業內部系統、AI Agent 接企業流程、課程案例、營運管理工具、CRM / 庫存 / 財務整合，可以優先回來看 ERPNext。

一句話摘要

ERPNext 是一套成熟的 100% 開源 ERP 系統，底層跑在 Frappe Framework 上，適合作為中小企業數位化、流程整合、內部管理系統與 AI Agent 企業應用的開源底座。

專案定位

ERPNext 的核心定位是「Powerful, Intuitive and Open-Source ERP」。它不是單一工具，而是一整套企業營運系統，涵蓋從交易、庫存、製造到專案與客戶管理的流程。

官方列出的主要功能包括：

  • Accounting：會計、現金流、交易紀錄與財務報表。
  • Order Management：庫存、銷售訂單、客戶、供應商、出貨與履約。
  • Manufacturing：生產週期、物料消耗、產能規劃、委外加工。
  • Asset Management：資產採購、折舊、維護、報廢與集中管理。
  • Projects：專案、任務、工時、議題與獲利追蹤。

技術底層

ERPNext 不是獨立從零寫的系統，它建立在 Frappe 生態系上：

  • Frappe Framework：Python 與 JavaScript 的全端 Web 應用框架，提供資料模型、權限、使用者認證、REST API、後台與 DocType 機制。
  • Frappe UI：Vue-based UI library，可用於打造現代化單頁應用。
  • ERPNext App：作為 Frappe App 安裝，透過 `erpnext/hooks.py` 註冊前端資源、事件、安裝流程、session boot、doctype 擴充與系統設定。

主要業務模組

目前 repo 內含大量 Frappe DocType。依目錄觀察，核心模組包括：

  • accounts：會計與財務，DocType 數量最大。
  • stock：庫存、倉儲、物料與報表。
  • selling：銷售流程、報價、訂單、客戶相關。
  • buying：採購、供應商、採購流程。
  • manufacturing：BOM、生產、工單、產能與製造流程。
  • crm：商機、客戶關係、銷售活動。
  • projects：專案、任務、工時與交付管理。
  • assets：資產管理。
  • support：支援、客服與問題追蹤。
  • quality-management：品質管理。
  • maintenance：維護與設備保養。
  • regional：各地區法規、稅務與在地化。
  • subcontracting：委外加工。
  • setup：公司、權限、基礎設定與初始化流程。

  • telephony / edi / communication：通訊、EDI 與延伸流程。

對欒帥未來可能有用的方向

1. 企業 AI Agent 的流程底座

ERPNext 很適合作為「企業資料與流程系統」，AI Agent 可以接在上面做：

  • 自動查詢客戶、訂單、庫存、應收帳款。
  • 依照自然語言產生報表摘要。
  • 協助建立採購單、銷售單、專案任務或客服紀錄。
  • 把 LINE / Email / 表單訊息轉成 ERPNext 內部流程。
  • 建立企業內部的 agentic workflow：例如每日營運簡報、異常庫存提醒、逾期應收款追蹤。

2. 課程案例

如果未來要教「AI 如何落地到企業工作流」，ERPNext 可以當作很好的真實案例：

  • 不是玩具資料庫，而是真正企業會用的 ERP。
  • 有模組、有權限、有流程、有報表、有 API。
  • 可示範 AI Agent 如何查詢、整理、跨模組操作。
  • 可示範「AI 不只是聊天，而是接到企業作業系統裡」。

3. 客製化內部系統

對需要 CRM、報價、庫存、採購、專案管理的組織，ERPNext 可能比從零開發更快：

  • 開源可自架。
  • 功能完整。
  • 可用 Frappe App 擴充。
  • 可與外部服務整合。
  • 適合做 PoC，再逐步客製。

需要注意的地方

  • GPL-3.0 授權：若做商業客製或再散布，需要特別注意授權義務。
  • 架構不輕：ERPNext 是完整 ERP，不適合當小型 demo app 隨便塞進現有服務。
  • 生態依賴 Frappe：真正要改功能，需學 Frappe Framework、DocType、hooks、bench、migration、permission 與 app 架構。
  • 官方 README 建議快速評估可用 Docker demo；正式部署則應看 Frappe Docker / Frappe Cloud / bench 相關文件。
  • develop branch 目前指向 v17 開發線，若要正式導入應選穩定版與對應文件。

未來使用時的第一步

若欒帥之後說「我們來用 ERPNext / Frappe 做一個企業系統」，建議流程是：

  • 先判斷需求屬於哪個模組：會計、庫存、銷售、CRM、專案、製造、客服。
  • 查 ERPNext 是否已有內建 DocType 與流程。
  • 用 Docker demo 或 Frappe Cloud 快速試流程。
  • 若要客製，再研究 Frappe App、DocType、hooks、permission 與 API。
  • 若要接 AI Agent，優先走 REST API / Server Script / Custom App，不要直接改核心。
