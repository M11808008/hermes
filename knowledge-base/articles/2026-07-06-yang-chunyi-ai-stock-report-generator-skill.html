楊俊益｜AI Stock Report Generator Skill：台股財務模型、投資簡報與技術圖表生成器

一句話摘要

`ai-stock-report-generator` 是一個把「台股代號」轉成完整財務模型、投資分析簡報與技術圖表的 agent skill；它的價值不在自動喊買賣，而在把公開資料整理成可稽核、可調假設、可教學展示的 Excel / PPTX 分析素材。

  • 14 頁 PPTX 投資分析簡報。
  • Excel 10 張工作表。
  • MACD / KD 圖表。
  • 以貼文圖片示例，輸出可包含公司財務模型總覽、股價、估值、EPS、P/B、情境合理價、技術指標與風險提示。

Skill 要求 agent 完成：

  • 即時搜尋公開財務資料與近期日線資料。
  • 建立 Excel 財務模型。
  • 建立投資分析 PowerPoint 簡報。
  • 產出 KD / MACD 技術指標。
  • 產出 K 線型態圖表。
  • 製作成長預測儀表板。
  • 製作 DCF + 相對估值分析頁。
  • 保留資料來源與檢查結果。

Skill 指令設計亮點

1. 強制「即時搜尋」而不是靠模型記憶

`SKILL.md` 明確要求第 1 步用 web search 搜尋公開資料，不可依賴訓練記憶。建議查詢包含：

  • 股票代號 + 公司名 + 全年財報 EPS 營收
  • 公司名 + 股價 / 今日 / 每股淨值
  • 公司名 + 股價大漲 / 展望 / 產業
  • 最新季度財報分析

必蒐集清單包括最新收盤股價、市值、近三年營收與 EPS、近五季 EPS、月營收、BVPS、成長題材與風險觀察指標。

2. Excel 要可稽核、可調假設

Skill 要求：

  • 藍字為輸入，黑字為公式，綠字為跨表連結。
  • 三情境預測要引用假設表。
  • 敏感度表、風險值與估值計算要用 Excel 公式。
  • 每個硬編碼數字都要在備註或來源表標註來源與日期。
  • Excel 公式錯誤需檢查，避免只產出漂亮但不可驗證的檔案。

這點很重要：真正可用的財務模型不是一張圖片，而是能被使用者改假設、追資料源、檢查公式的工作簿。

3. 台股圖表色彩規範

Skill 特別寫出台股慣例：

  • 紅色 = 上漲 / 正值 / 偏多。
  • 綠色 = 下跌 / 負值 / 偏空。
  • K 線、成交量與 MACD 柱狀體不得做成美股常見的綠漲紅跌。

這是很實用的在地化細節，能避免 AI 直接套用英文金融圖表慣例造成錯誤解讀。

4. 技術指標只是短線補充

Skill 規定 KD、MACD、K 線型態只能作為短線位置補充，不取代基本面估值；若 K 線型態為規則式偵測，必須明確標註「適合教學與快速觀察，不等同投資建議」。

對 Luarn-Pin / AI 學習雷達的價值

1. 這是「Agent Skill 商品化」的好例子

這個 repo 不是完整 SaaS，而是一個高密度 `SKILL.md`。它示範了如何把一個專業任務拆成 agent 可執行流程：資料蒐集、模型建構、檔案產出、圖表規範、QA、交付話術與免責聲明。

對 AI 課程設計來說，它適合作為「如何寫一個可交付檔案的 agent skill」案例。

2. 財務分析任務很適合測 agent 是否真的能工作

台股分析報告需要同時處理：

  • 即時資料搜尋。
  • Excel 公式與多工作表連動。
  • PPTX 版面設計。
  • 技術指標計算。
  • 資料來源標註。
  • 免責與風險邊界。

這比單純聊天或摘要更能測出 agent 的工具使用、格式控制、資料誠信與交付能力。

3. 可延伸成「可稽核 AI 報告」課程模組

若之後做 AI 財務 / 商業分析課程，可以用這個 skill 的結構說明：

  • Prompt 不是只寫「幫我分析股票」，而是寫成完整作業規格。
  • 好的 agent skill 要明確規定輸出檔案、來源、檢查、色彩、免責。
  • 專業領域 agent 必須把「不能做什麼」寫清楚，例如不能給保證收益、不能把技術指標當買賣建議。

風險與限制

  • Repo 未標示 license。若要改作教材、商用模板或整合到服務，需先確認授權。
  • Skill 本身是指令規格，不代表每個 agent 執行都一定能產出正確檔案；實際品質仍取決於模型、工具環境、搜尋資料來源、Excel/PPTX 能力與 QA 流程。
  • 財務資料來源可能彼此矛盾或延遲，使用者需自行確認正確性。
  • 估值、DCF、相對估值、KD/MACD 都是分析工具，不構成投資建議。
  • 若 agent 能寫檔、抓網路與產生報告，需注意資料來源版權、錯誤引用、幻覺與報告被誤用為投資建議的風險。
