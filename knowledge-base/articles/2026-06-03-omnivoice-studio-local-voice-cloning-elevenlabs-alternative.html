OmniVoice Studio｜本地開源 ElevenLabs 替代方案，支援語音克隆、配音與聽寫

來源：Threads@alextathoho

一句話摘要

OmniVoice Studio 是一個開源、本地運行的 AI 語音工作室，主打 3 秒語音克隆、646 種語言、影片翻譯配音、全域聽寫與 MCP 整合，對需要保護音訊資料隱私、降低訂閱成本、建立多媒體內容流水線的人特別有價值。

重點整理

1. 定位：本地版 ElevenLabs 替代方案

Threads 貼文指出，ElevenLabs 的 AI 語音克隆方案有月費與企業方案成本，而 OmniVoice Studio 的定位是「open-source ElevenLabs alternative」。

GitHub README 也強調：

  • 開源
  • 不需要 API key
  • 完全本地運行
  • 支援 646 種語言
  • 可用於即時聽寫、zero-shot 語音克隆與影片配音

這代表它不是單一 TTS demo，而是試圖做成桌面端 AI 語音工作室。

2. 核心功能

| 功能 | 說明 |

|---|---|

| 語音克隆 | 3 秒音訊片段即可 zero-shot 克隆聲音 |

| 語音設計 | 可調整性別、年齡、口音、音高、速度、情緒、方言等特徵 |

| 影片配音 | 貼上 YouTube 連結或上傳檔案，進行轉錄、翻譯、重新配音並輸出 MP4 |

| 全域聽寫 | 在任何 App 使用快捷鍵呼叫，完成轉錄後自動貼上文字 |

| 人聲分離 | 透過 Demucs 分離語音與背景音樂，保留原始背景聲 |

| 說話人辨識 | 使用 Pyannote / WhisperX 自動辨識誰說了哪一句 |

| 批次佇列 | 可丟入多支影片批次處理 |

| MCP Server | 可從 Claude、Cursor 或其他 MCP Client 呼叫 OmniVoice 功能 |

| AI 浮水印 | 內建 Meta AudioSeal，用於隱形音訊浮水印 |

3. 與 ElevenLabs 的差異

OmniVoice Studio README 的比較重點包括：

  • ElevenLabs 是雲端服務；OmniVoice Studio 在本機硬體執行。
  • ElevenLabs 需要 API key；OmniVoice Studio 不需要帳號與 API key。
  • ElevenLabs 官方語言支援較少；OmniVoice Studio 標榜 646 語言。
  • ElevenLabs 是封閉式平台；OmniVoice Studio 可 fork、擴充、替換 TTS backend。
  • OmniVoice Studio 可支援 CUDA、Apple Silicon MPS、ROCm 與 CPU。

但也要注意：專案仍處於 active beta，README 明確提醒版本間可能會破壞相容性。若要使用最新功能，官方建議從 source 執行，而不是完全依賴預編譯安裝包。

4. 系統需求與使用門檻

README 提到的基本需求：

| 項目 | 最低需求 | 建議 |

|---|---|---|

| OS | Windows 10、macOS 12+、Ubuntu 20.04+ | 現代 64-bit OS |

| RAM | 8 GB | 16 GB+ |

| VRAM | 4 GB，可自動 offload | 8 GB+，例如 RTX 3060+ |

| 硬碟 | 10 GB free | 20 GB+ SSD |

| Python | 3.10+ | 3.11–3.12 |

| GPU | 可選，CPU 可跑但較慢 | CUDA / Apple Silicon MPS / AMD ROCm |

對 Mac 使用者來說，Apple Silicon MPS 支援是值得注意的亮點；對內容團隊來說，CPU 也能跑代表可以先低成本試用，但大量影片配音仍建議使用較好的 GPU。

為什麼值得收藏

1. AI 語音工具開始從雲端訂閱走向本地工作室

過去高品質語音生成多仰賴 ElevenLabs 這類雲端服務。OmniVoice Studio 代表另一種趨勢：把語音克隆、TTS、ASR、影片配音、說話人辨識與浮水印整合到本機桌面 App。

對創作者與企業內部內容團隊來說，這會影響：

  • 音訊資料是否能留在本地
  • 每月訂閱與用量成本
  • 是否能自訂模型與流程
  • 能否接進 Claude / Cursor / Agent 工作流

2. MCP Server 讓語音工作流可被 Agent 呼叫

它內建 MCP Server，因此不只是 GUI 工具，也可能成為 AI Agent 工具鏈的一部分。未來可想像：

  • Agent 讀腳本後自動生成配音
  • 將 YouTube 影片批次轉錄、翻譯、重新配音
  • 在內容生產流程中自動建立 voice profile
  • 讓 Claude / Cursor 直接呼叫本地語音生成與處理能力

這點對 Luarn-Pin 的 AI 學習雷達尤其重要，因為它連接了「本地 AI」、「多媒體內容自動化」與「Agent 工具整合」。

風險與注意事項

  • 語音克隆涉及聲音肖像權、授權與詐騙風險，應只使用取得授權的聲音素材。
  • 專案仍在 beta，正式工作流導入前需測試穩定性。
  • 本地運行雖保護隱私，但對硬體、模型下載與環境配置仍有要求。
  • 開源專案授權與商業使用條款需在正式商用前仔細確認。
