官方查核：哪些部分成立？

Claude Code

Claude Code 官方 Overview 說明，它是 agentic coding tool，能讀取 codebase、編輯檔案、執行命令，並整合 terminal、IDE、desktop app、browser 等開發工具。這支持「Claude Code 可協助搭建/修改前端網站」的方向。

但官方文件並沒有保證單一 prompt 就能產出可直接商用的完整網站。對真實專案來說，Claude Code 更像可迭代的工程代理：生成 scaffold、修改元件、安裝套件、跑測試、修錯、根據回饋調整，而不是一次性魔法輸出。

Framer Motion / Motion

Motion 官方網站指出：Motion（原 Framer Motion）是 JavaScript、React、Vue 的 production-grade animation library；Motion for React 文件也定位為 React animation library。這支持「用 Framer Motion / Motion 製作平滑前端動畫」的技術可行性。

需要注意：現在官方品牌與文件多使用 `Motion`，而 `Framer Motion` 是舊名與生態常用名稱。若要在新專案中實作，需依目前套件與文件版本確認 import 路徑與 API。

需要降級處理的社群主張

「AI can build one in minutes」

可作為 demo / landing page 原型敘事，但不應直接理解為任何商業網站都能幾分鐘完成。網站交付通常還包含：

  • 品牌定位、文案與素材授權。
  • RWD 與跨瀏覽器檢查。
  • SEO metadata、Open Graph、schema。
  • 表單、CRM、付款、會員、分析、隱私政策等整合。
  • 可近用性、效能、安全、部署與維運。

「one prompt = ready to launch」

這是社群短影音常見的強化說法。比較穩健的做法是把它視為「用一段高品質 prompt 生成第一版 animated landing page」，後續仍需人工審稿、測試與多輪迭代。

「NoCode」標籤

Claude Code 本質上是 coding agent，會產出與修改程式碼。使用者可能不必手寫每一行，但工作流仍是 code-based，而不是完全無程式碼平台。若交付需要工程品質，仍要懂得審查依賴、build output、錯誤訊息與部署流程。

實務可行工作流

若要把這類 Reel 轉成可落地流程，可拆成：

  • 建立 Vite / Next.js / Astro 等前端專案。
  • 用 Claude Code 產生 landing page 結構、品牌文案與 component tree。
  • 安裝 Motion / Framer Motion，讓 Claude Code 加入 hero animation、scroll reveal、micro-interaction。
  • 要求 Claude Code 跑 lint / build，修復型別與 bundler 錯誤。
  • 人工檢查手機版、可近用性、效能與轉換流程。
  • 部署到 Vercel / Netlify / Cloudflare Pages，設定網域與 analytics。

保存原因
