一句話摘要

這個 repo 的價值不在「求星」本身，而在示範如何把個人知識工作流程整理成可安裝、可驗證、可分享的 Codex Skills：從每日計畫、九宮格、FIRE 分析、Obsidian graph 到 EPUB / HyperCard，都被放進一個「Agent 高效率，人腦慢生活」的工作系統。

Repo 的核心定位

README 開宗明義寫道：

> 用 Desktop 概念幫助知識工作者：Agent 高效率，人腦慢生活。

這不是單純的 prompt collection，而是一套「Hermes HyperCard Loop」：

```text

人腦慢想

-> 卡片整理

-> Agent 執行

-> Skill 沉澱

-> GitHub 分享

-> 書籍連結

-> 回到 Desktop 日常

```

這個循環很接近欒帥目前 AI 學習雷達與 Hermes 的方向：把一次次工作流沉澱成可重複操作的模組，再用 GitHub / Markdown / 靜態頁面保留長期可引用的入口。

最值得觀察的 Featured Skill：今日的日計畫

README 將 `todays-daily-plan` 列為主打 Skill。它服務 Obsidian 的 Mandala Grid 日計畫檔，讓使用者用口語說：

```text

Use $todays-daily-plan 今日的日計畫，上午羽球，下午三點林君，日記 Miru 來訪，狀況不是很好，加油。

```

Agent 會把內容安全追加到今天的九宮時段或日記區塊，而不是手動尋找 Markdown section。這裡的設計重點包括：

  • 用自然語言當入口，而非要求使用者學複雜 CLI。
  • 將「今天」定位到年度日計畫中的 section。
  • 補齊固定九宮時段，例如 09-12、12-13、15-18、日記等。
  • 只追加，不重排全年檔案，避免破壞既有 Mandala Grid 結構。

對欒帥的可借鏡之處

  • **把工作流產品化**：每個 Skill 都不是單次 prompt，而是有用途、入口、文件、範例與安裝位置的微型產品。
  • **把個人系統公開成 repo**：repo 不是只放程式碼，也能放方法論、書籍連結、技能索引與日常操作面板。
  • **把 Agent 和慢思考分工**：快速轉檔、整理、驗證交給 Agent；判斷、選題、長期節奏留給人。
  • **用 GitHub 當長期入口**：書籍、課程或社群貼文都可連回固定 repo / docs / skill 路徑。
  • **把知識卡片與出版串起來**：FIRE 卡、九宮卡、Obsidian graph、EPUB、HyperCard 串成完整知識生產線。

對 AI 學習雷達的價值
