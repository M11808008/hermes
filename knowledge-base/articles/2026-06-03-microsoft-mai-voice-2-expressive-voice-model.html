Microsoft MAI-Voice-2｜支援情緒控制與短音訊語音提示的新一代 AI 語音模型

來源：Threads@kufutw；官方對照來源：Microsoft AI「Introducing MAI-Voice-2」

一句話摘要

微軟推出 MAI-Voice-2，主打更自然的文字轉語音、情緒標籤控制、5–60 秒 reference audio 的 zero-shot voice prompting，以及長內容中穩定維持說話者身份，適合客服、助理、內容朗讀與無障礙語音體驗。

貼文重點

Threads 貼文摘要指出：

  • 微軟 AI 語音大升級。
  • 只靠一小段錄音就能複製聲音。
  • 能直接演出喜怒哀樂等情緒。

貼文沒有完整列出模型名稱；對照 Microsoft AI 官方 2026-06-02 發布資訊，最接近且時間相符的是 MAI-Voice-2。

官方資訊整理

1. MAI-Voice-2 的定位

Microsoft AI 表示，MAI-Voice-2 是其目前最具表現力、最自然的 text-to-speech 模型，針對正式產品語音體驗設計，例如：

  • 品牌助理
  • 客服語音
  • 有聲書
  • 長時間內容朗讀
  • 無障礙語音介面
  • Dynamics 365 Contact Center 等企業場景

官方也提到模型已可在 Azure Foundry 使用，並正在整合進 VSCode 與 Dynamics 365 Contact Center。

2. 核心能力

| 能力 | 說明 |

|---|---|

| 多語言 | 從英文擴展到 15 種語言，維持自然度與表現力 |

| 情緒控制 | 可用 emotion tags 控制語氣，例如 sad、whispered、excited 等 |

| 短音訊提示 | 支援 5–60 秒 reference audio 的 zero-shot voice prompting |

| 說話者一致性 | 可在有聲書、podcast、講座等長內容中維持 speaker identity |

| Code-switching | 支援特定語言組合的自然語言混用，如 Hindi-English、Spanish-English |

| 偏好測試 | 官方稱 MAI-Voice-2 相對前代 MAI-Voice-1 有 72% 偏好率 |

為什麼值得收藏

1. 語音 AI 正從「像真人」進入「可導演」階段

早期 TTS 的重點是自然度；現在更重要的是可控性。MAI-Voice-2 強調 emotion tags 與 speaker consistency，代表語音模型開始具備類似導演指令的能力：同一段話可用不同情緒、角色與語言風格表演。

2. 對客服與品牌語音很關鍵

Kufu 酷服是 AI 智能客服平台，因此這類模型對客服場景特別重要。未來語音客服不只是「把文字唸出來」，而是要能依情境呈現：

  • 安撫
  • 興奮
  • 道歉
  • 專業說明
  • 低壓引導

這會直接影響使用者體驗與品牌一致性。

3. 和本地開源語音工具形成對照

同一天知識庫也收錄了 OmniVoice Studio 這類本地開源語音工作室。兩者代表不同路線：

  • Microsoft MAI-Voice-2：企業級、雲端平台、產品整合、治理與 consent guardrails。
  • OmniVoice Studio：本地運行、開源、可自架、重視資料不出機器。

對 Luarn-Pin 來說，這兩條線都值得追蹤：一條是大型平台產品化，一條是開源本地化。

風險與注意事項

  • 語音複製必須有明確同意與授權，否則容易涉及聲音肖像權與詐騙風險。
  • 官方提到 consent guardrails，表示微軟也將負責任部署視為核心議題。
  • 貼文用語「一小段錄音」較口語；官方規格是 5–60 秒 reference audio，而不是任意極短樣本都保證可用。
  • 實際可用語言、API、價格與區域可用性仍需以 Azure Foundry 實際上架狀態為準。
