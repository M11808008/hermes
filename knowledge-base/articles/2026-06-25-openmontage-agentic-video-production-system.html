OpenMontage｜開源 Agentic 影片生產系統，把 Claude / Cursor / Codex 變成影片工作室

开发者Hailey 在 X 分享 OpenMontage，稱它是「全球首個開源 Agentic 視頻生產系統」，可以把 Claude / Cursor / Copilot 變成完整 AI 影片工作室。這則貼文值得收錄，但不應只當作「一句話拍大片」的社群爆款文；真正有價值的是它把影片製作拆成 pipeline、skill、tool registry、checkpoint、cost governance 與 self-review 的 agent workflow 架構。

一句話摘要

OpenMontage 不是單一影片生成模型，而是一套讓 coding agent 按照 pipeline manifests 與 stage director skills 操作多種影音工具的「開源影片製作作業系統」。

它解決什麼問題

一般 AI 影片工具常卡在三個層級：

  • 只會產生單段影片或圖片，不會做完整內容企劃。
  • 每個 provider API 都要手動串接，成本與品質不可控。
  • 生成影片缺少腳本、素材授權、字幕、剪輯、配音、合成、自審與交付流程。

OpenMontage 的解法是把影片製作流程 agent 化：由 agent 讀取規則與 pipeline，選工具、估成本、產出素材、合成影片、跑品質檢查，並在關鍵節點要求人類批准。

核心架構

官方 `PROJECT_CONTEXT.md` 把 OpenMontage 定位為 instruction-driven / agent-first 架構：

```text

Agent reads pipeline manifest (YAML)

→ reads stage director skill (Markdown)

→ uses Python tools

→ self-reviews with meta skill

→ writes checkpoints

→ asks human approval when needed

```

也就是說，Python 主要負責工具與 persistence；真正的 orchestration、creative decisions、review logic 與 checkpoint policy 則在 YAML manifests 與 Markdown skills 裡。

主要組件

| 組件 | 作用 |

|---|---|

| `AGENT_GUIDE.md` | agent 操作契約，規範 pipeline、工具、決策溝通與 blocker escalation |

| `PROJECT_CONTEXT.md` | 架構總覽、source of truth、pipeline 與 tool conventions |

| `pipeline_defs/*.yaml` | 每種影片製作流程的 pipeline manifest |

| `skills/pipelines/<pipeline>/<stage>-director.md` | 每一階段的 director skill，教 agent 如何做該階段 |

| `tools/tool_registry.py` | 讓 agent discovery 目前可用工具、provider、成本與支援範圍 |

| `tools/cost_tracker.py` | 預估、保留、核銷影音生成成本 |

| `schemas/artifacts/` | 對 brief、script、scene plan、asset manifest、render report 等 artifact 做 schema 約束 |

| Remotion / HyperFrames / FFmpeg | 影片合成與 motion graphics runtime |

低成本說法如何判讀

X 貼文提到幾毛錢成本，README 也列出案例：

| 範例 | README 成本快照 | 可能原因 |

|---|---:|---|

| Ghibli-style Candyland animation | $0.15 | 使用 FLUX 圖像 + Remotion 動畫，而不是昂貴影片生成 API |

| Pixar-style banana short | $1.33 | 使用 Kling v3 motion clips + Google narration + music + captions |

| Product ad | $0.69 | OpenAI image/TTS + royalty-free music + subtitles + Remotion data visualizations |

這些成本是「特定工具組合與短片案例」的結果，不代表所有影片都能幾毛錢完成。若使用 Runway、Veo、HeyGen、長片、多角色、精細 reference video remake，成本會明顯上升。

OpenMontage 的關鍵設計：不是工具多，而是 governance 多

OpenMontage 的 `AGENT_GUIDE.md` 特別強調：

  • 所有影片製作都必須經過 pipeline system。
  • 重大 provider / model / runtime 選擇前要告知並取得批准。
  • paid 或 consequential generation call 前要說明工具、provider、model、理由、是 sample 還是 batch run。
  • 若 Remotion 與 HyperFrames 都可用，proposal 階段要同時呈現兩者選項，不能默默選預設。
  • blocker 必須清楚說明：嘗試了什麼、失敗原因、屬於 auth / provider / tool bug / prompt quality 哪一類、可選路徑與推薦方案。

這使它更像「可審計的 production agent」，不是單純把 API 串在一起。

1. 很適合做 AI 教學與課程素材 pipeline 範本

OpenMontage 的 pipeline 思維可直接轉成 AI 課程案例：

```text

主題 → research brief → script → scene plan → asset plan → narration → subtitles → composition → self-review → publish
```

這比單純教「輸入 prompt 生成影片」更能展現 AI agent 的實務價值。

採用建議

先做 sandbox，不要直接接主力內容生產

OpenMontage 是高活躍、依賴多 provider 的開源專案。建議：

  • 先 clone 到隔離測試目錄。
  • 只跑 `make setup` 與 tool discovery，不放入任何高權限 secrets。
  • 優先使用零 API key 路徑：Piper、Archive.org、NASA、Wikimedia、FFmpeg、Remotion。
  • 若要接 API key，先用低額度/可撤銷 key，並設定預算上限。
  • 先測 30–60 秒教育型 explainer，不直接做長片或商業素材。
  • 檢查 AGPL-3.0 授權對衍生服務、部署與商業使用的影響。

適合 / 不適合場景

| 適合 | 不適合 |

|---|---|

| 教學 demo、短影音 prototype、內容 pipeline 教材 | 立刻要求影視級穩定商用交付 |

| 有 coding agent 能讀檔/跑 code 的使用者 | 只想用網頁 UI 一鍵出片的人 |

| 願意做 provider 成本治理與品質檢查 | 不想管理 API key、FFmpeg、Node、Python 環境的人 |

| 需要可審計、可擴充的 agent workflow | 只需要單一模型生成幾秒影片的人 |

  • X 貼文是社群摘要，本文以 GitHub README、AGENT_GUIDE、PROJECT_CONTEXT、PROVIDERS 文件補強與校正。
  • 未在本機實際執行 OpenMontage；本文是架構與採用判讀，不是安裝成功報告。

  • 「成本低至幾毛錢」只適用於 README 中特定短片與工具組合；實際成本取決於 provider、長度、解析度、重試次數、模型與素材策略。
