林啟維｜Claude Fable 5 兩天窗口：AI Vault、知識庫更新與大型文件任務清單

一句話摘要

這則貼文值得收進 AI 學習雷達，因為它把「高能力模型短期開放」轉成可執行清單：不要只測模型聊天能力，而是把 Fable 5 用在 AI Vault、知識庫整理、官網重寫、模板化與大型計畫書這類需要長上下文、多步驟推理與文件輸出的任務。

貼文重點

林啟維在 Facebook 公開貼文中提醒，如果已購買 Claude Pro / Max，接下來兩天務必嘗試 Fable 模型。他的核心判斷是：Fable 是目前公開推出、能力與成本都很高的模型之一，短期被納入 Max 月訂用量，估算可等同數百到一千美元級別的 API fee。

他建議如果只有兩天時間，優先做五件事：

  • 建立個人知識庫：初次打造自己的 AI Vault / LLM Wiki，未來 Cowork、Claude Code 任務都能引用。
  • 更新知識庫：讀取全部資料，找出矛盾、過期內容，並更新 vault SOP 流程；貼文提到他用一晚更新 295 個文件。
  • 更新官網：用 Claude Code 完整閱讀官網、重寫每一頁介紹，強化主題聚焦與 SEO。
  • 建立 Templates：為日常作業建立合約、報價單、Excel 記帳表等模板，先用 Fable 打好高品質基本架構，再由 Opus 持續修改。
  • 撰寫大型計畫：產出商業計畫書 Business Plan 或標案企劃初稿，讓後續模型可承接修整。

貼文補充，他把一份國際商業文件包交給 Fable，主觀評估產出接近「中上水準顧問」等級；若以 API 計算，可能花費 300–500 美元 token fee，但目前包含在 200 美元 Claude Max 月費裡。貼文也提示 Fable 試用截止時間為台灣 7 月 8 日 15:00，並在留言提供開源 AI Vault repo：cwlin0131/Vault-for-Founders。

官方交叉驗證

Brave Search 與 Anthropic 官方頁面可確認幾個關鍵背景：

  • Anthropic 在「Redeploying Claude Fable 5」中表示，Fable 5 於 2026-07-01 起重新提供給 Claude Platform、Claude.ai、Claude Code、Claude Cowork 使用者。
  • 官方說明指出，Pro、Max、Team 與部分 Enterprise 方案中，Fable 5 到 2026-07-07 前可包含在每週用量上限的 50% 內；之後透過 usage credits 使用。
  • Anthropic 的 Fable 頁面把 Claude Fable 5 定位為可處理困難知識工作與 coding work 的高能力模型，適合長任務、複雜實作、多階段企業文件與大型遷移。
  • 官方也強調 Fable 5 對資安、生物、化學等高風險領域有 safeguards；部分查詢會被路由到 Opus 4.8，使用者不應把它視為無限制模型。
  • Claude Platform docs 稱 Fable 5 是 Anthropic 最強的廣泛發布模型之一，適合 demanding reasoning 與 long-horizon agentic work；整合時需處理 refusal、fallback 與新計費規則。

為什麼這則貼文重要

1. 它把「模型窗口期」轉成任務設計

很多人試用新模型時只會問幾個問題、跑幾個 benchmark。但 Fable 5 的價值不在日常聊天，而在高 token、高上下文、高文件產出的任務。貼文列出的五件事剛好符合高能力模型的經濟性：

  • 低頻但高價值。
  • 需要一次讀很多資料。
  • 產出可長期複用。
  • 未來可交給較便宜模型維護。

這比「測模型聰不聰明」更接近 AI 學習雷達應保存的實作觀點。

2. AI Vault 是 agent 工作流的基礎設施

建立 AI Vault / LLM Wiki 的重點不是漂亮筆記，而是讓未來的 Claude Code、Claude Cowork、Hermes Agent、Codex 類任務有可引用的長期記憶。若沒有結構化資料、SOP、決策背景與過期資訊清理，agent 只能在單次對話內臨時理解任務，難以累積組織知識。

對 Luarn-Pin 的知識庫而言，這則貼文可視為「把高能力模型用來整理知識底座」的案例：模型能力越強，越應該先用在改善資料環境，而不是只拿來多產幾篇內容。

3. 最值得做的是可複利資產

五個任務中，真正有複利的是：

  • 知識庫：讓未來所有任務降低理解成本。
  • SOP / Vault 流程：降低重複任務的記憶成本。
  • 官網：把定位、SEO 與對外敘事一次重構。
  • Templates：讓合約、報價、記帳等行政流程模板化。
  • 大型計畫：先生成可討論的完整初稿，再由人類與後續模型迭代。

這些都不是一次性輸出，而是未來工作流的底層資產。

可轉成 Luarn-Pin 工作流的地方

  • 高能力模型優先用在「知識底座整理」：例如每月讀取整個 AI 學習雷達、找出過期、矛盾、可合併與可升級成 skill 的內容。
  • 對每個常見任務建立模板：課程頁、社群貼文歸檔、PDF 教材整理、報名名單清理、LINE 群組回覆，都可以變成可複製模板。
  • 把大型文件任務切成兩段：先用最強模型產出完整骨架，再用較便宜或更穩定的模型做局部修訂與格式化。
  • 對高價模型設定「窗口期清單」：每當某模型短期免費或含在訂閱中，先跑高價值批次任務，而不是隨機試玩。
  • 保留 fallback：官方明確提到 Fable 5 可能 refusal 或路由，因此 agent 工作流需預設 Opus / Sonnet / 其他 provider 的替代路徑。

風險與判讀限制

  • Facebook 貼文中的 API fee 與外包價值屬作者估算與主觀體驗，應保存為使用者經驗，不當作官方定價或保證成效。
  • Fable 5 的可用窗口與計費規則以 Anthropic 官方頁面為準；貼文提到台灣 7 月 8 日 15:00，官方頁面則以 through July 7 與 usage credits 說明。
  • Fable 5 並非無限制模型。官方已說明高風險領域有 safeguards，且部分請求會 fallback 到 Opus 4.8。
  • 這則貼文的真正價值在「任務優先序」：把最昂貴模型用在可複利工作流，不在於鼓勵無條件課金。
