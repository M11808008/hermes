為什麼值得收進 AI 學習雷達？

MotionSites 代表一個很實用的趨勢：網頁設計 prompt 正在從「一句話描述」變成「可複製的設計系統片段」。

對非設計師而言，它提供：

  • 可直接複製的高質感網站 prompt。
  • 已分類的網站情境：SaaS、Agency、Portfolio、Landing Page。
  • 動畫、gradient、hero section 等視覺語彙。
  • 可餵給 Claude、Cursor、Lovable、Bolt 的生成規格。

對 AI 開發工作流而言，它提供一種「prompt-as-template」模式：先用高品質 prompt 生成第一版，再由 agent 接著改內容、串資料、部署。

適合應用場景

  • 課程頁、活動頁、招生頁快速原型。
  • SaaS landing page 與產品首頁。
  • 個人作品集與顧問服務頁。
  • Demo day / pitch deck 的互動網頁版。
  • 用 Claude / Cursor 快速產出視覺方向，再接 GitHub Pages 或 Vercel 部署。
  • 建立內部 prompt library，沉澱常用網站風格。

---

實務提醒

  • Prompt library 可加速第一版，但仍需要品牌文字、資訊架構與轉換目標。
  • 生成網站常見問題是動畫過重、可讀性不足、無障礙不足，需人工 QA。
  • 若使用第三方模板或 prompt，需檢查授權與可商用性。
  • 對正式網站，還需補 SEO、表單、安全、隱私政策、效能與手機版測試。
  • 生成後建議把可重複的 design tokens、components 與文案規格沉澱成專案模板。

對 AI 學習雷達的結論

MotionSites.ai 值得歸檔為「AI 網站生成 prompt library」案例。它不是單純的靈感站，而是把高質感網頁生成拆成可複製 prompt，讓 AI coding tools 更快產出 landing page 與 SaaS UI。

對欒帥的知識庫與課程頁工作流來說，這類資源可作為快速視覺提案來源：先生成 2-3 個高質感方向，再套入實際課程內容、報名流程與 GitHub Pages 部署。
